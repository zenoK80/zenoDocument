# Chart.js 설치하기

Chart.js를 사용하려면 먼저 설치가 필요합니다. 걱정하지 마세요! 설치는 생각보다 훨씬 간단합니다. 이 가이드에서는 여러 가지 설치 방법을 초보자도 쉽게 따라할 수 있도록 **아주 자세히** 설명하겠습니다.

## 설치 방법 선택하기

Chart.js를 설치하는 방법은 크게 세 가지가 있습니다. 여러분의 상황에 맞는 방법을 선택하세요:

### 방법 비교표

| 설치 방법 | 난이도 | 추천 대상 | 장점 |
|---------|--------|----------|------|
| **CDN 사용** | ⭐ 매우 쉬움 | HTML/CSS/JS만 배운 초보자 | 설치 불필요, 한 줄이면 끝 |
| **npm 설치** | ⭐⭐ 보통 | React, Vue 등 프레임워크 사용자 | 프로젝트 관리 용이, 버전 관리 |
| **GitHub 다운로드** | ⭐⭐⭐ 어려움 | 고급 개발자, 커스터마이징 필요 시 | 소스 코드 직접 수정 가능 |

**초보자라면 CDN 방법을 강력히 추천합니다!** 가장 쉽고 빠르게 시작할 수 있습니다.

## 방법 1: CDN 사용하기 (초보자 추천 ⭐)

### CDN이란 무엇인가요?

**CDN(Content Delivery Network)**은 인터넷 어딘가에 있는 파일 저장소입니다. Chart.js 파일이 이미 인터넷 서버에 올라가 있어서, 우리는 그 파일의 주소만 HTML에 적어주면 자동으로 불러와서 사용할 수 있습니다.

**장점**:
- 파일을 다운로드하거나 설치할 필요가 없습니다
- 한 줄만 추가하면 바로 사용 가능합니다
- 인터넷만 연결되어 있으면 언제든 최신 버전을 사용할 수 있습니다
- 프로젝트 파일 크기가 늘어나지 않습니다

**단점**:
- 인터넷이 연결되어 있어야 합니다
- CDN 서버에 문제가 생기면 차트가 작동하지 않을 수 있습니다 (매우 드뭅니다)

### jsDelivr CDN 사용하기 (가장 추천)

jsDelivr는 무료로 사용할 수 있는 안정적인 CDN 서비스입니다. 다음 한 줄을 HTML 파일에 추가하면 됩니다.

#### 단계별 설명

**1단계**: HTML 파일을 만듭니다 (이미 있다면 생략)

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Chart.js 연습</title>
</head>
<body>
  <h1>내 첫 번째 차트</h1>
  
  <!-- 여기에 차트가 들어갈 예정입니다 -->
  
</body>
</html>
```

**2단계**: `</body>` 태그 바로 위에 Chart.js CDN 링크를 추가합니다

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Chart.js 연습</title>
</head>
<body>
  <h1>내 첫 번째 차트</h1>
  
  <!-- 이 한 줄만 추가하면 끝! -->
  <script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
  
</body>
</html>
```

**해설**:
- `<script>`: JavaScript 파일을 불러오는 HTML 태그입니다
- `src="..."`: 파일의 주소(URL)를 지정합니다
- `https://cdn.jsdelivr.net/npm/chart.js`: jsDelivr CDN에 있는 Chart.js 파일 주소입니다

**3단계**: 이제 Chart.js를 사용할 수 있습니다!

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Chart.js 연습</title>
</head>
<body>
  <h1>내 첫 번째 차트</h1>
  
  <!-- 차트가 그려질 캔버스 -->
  <canvas id="myChart"></canvas>
  
  <!-- Chart.js 라이브러리 불러오기 -->
  <script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
  
  <!-- 차트 만들기 -->
  <script>
    const ctx = document.getElementById('myChart');
    new Chart(ctx, {
      type: 'bar',
      data: {
        labels: ['빨강', '파랑', '노랑', '초록', '보라', '주황'],
        datasets: [{
          label: '투표 수',
          data: [12, 19, 3, 5, 2, 3],
          borderWidth: 1
        }]
      },
      options: {
        scales: {
          y: {
            beginAtZero: true
          }
        }
      }
    });
  </script>
</body>
</html>
```

이 파일을 웹 브라우저에서 열면 막대 차트가 나타납니다!

### CDNJS 사용하기 (대안)

jsDelivr 대신 CDNJS를 사용할 수도 있습니다. 둘 다 안정적이고 무료입니다.

```html
<!-- CDNJS 버전 -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/Chart.js/4.4.1/chart.umd.js"></script>
```

**jsDelivr vs CDNJS 차이점**:
- jsDelivr: `npm`에 있는 최신 버전을 자동으로 가져옵니다
- CDNJS: 특정 버전을 명시해야 합니다 (위 예시는 4.4.1 버전)

초보자는 **jsDelivr를 추천**합니다. 버전 번호를 신경 쓰지 않아도 되기 때문입니다.

### 특정 버전 사용하기

때로는 특정 버전의 Chart.js를 사용하고 싶을 수 있습니다. 예를 들어, 버전 4.4.1을 사용하려면:

```html
<!-- 특정 버전 지정 -->
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.1"></script>
```

**왜 특정 버전을 사용할까요?**
- 최신 버전에서 발생할 수 있는 예상치 못한 변경을 피하기 위해
- 팀 프로젝트에서 모든 팀원이 같은 버전을 사용하도록 하기 위해
- 이전에 작성한 코드와의 호환성을 유지하기 위해

### CDN 작동 확인하기

Chart.js가 제대로 로드되었는지 확인하는 방법:

**방법 1: 브라우저 개발자 도구 사용**

1. 웹 브라우저에서 페이지를 엽니다
2. `F12` 키를 누르거나 우클릭 → "검사" 선택
3. "Console" 탭으로 이동
4. 다음 명령어를 입력하고 Enter:

```javascript
console.log(Chart);
```

5. 에러 없이 객체 정보가 나타나면 성공입니다!

**방법 2: 간단한 테스트 코드**

```html
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<script>
  if (typeof Chart !== 'undefined') {
    console.log('Chart.js 로드 성공! 버전:', Chart.version);
  } else {
    console.log('Chart.js 로드 실패...');
  }
</script>
```

## 방법 2: npm으로 설치하기

### npm이란 무엇인가요?

**npm(Node Package Manager)**은 JavaScript 라이브러리들을 관리하는 도구입니다. 마치 스마트폰의 앱 스토어처럼, 필요한 라이브러리를 설치하고 업데이트하고 삭제할 수 있습니다.

**이 방법을 사용해야 하는 경우**:
- React, Vue, Angular 같은 프레임워크를 사용할 때
- Node.js 환경에서 개발할 때
- 프로젝트를 다른 사람과 공유하거나 배포할 때
- 여러 라이브러리를 체계적으로 관리하고 싶을 때

### 사전 준비사항

npm을 사용하려면 먼저 **Node.js**가 설치되어 있어야 합니다.

#### Node.js 설치 확인하기

터미널(Windows에서는 명령 프롬프트 또는 PowerShell)을 열고 다음 명령어를 입력하세요:

```bash
node --version
```

버전 번호가 나타나면 (예: `v18.17.0`) Node.js가 이미 설치되어 있는 것입니다.

`'node'은(는) 내부 또는 외부 명령...` 같은 에러가 나타나면 Node.js가 설치되어 있지 않은 것입니다.

#### Node.js 설치하기

1. [https://nodejs.org](https://nodejs.org) 접속
2. 왼쪽의 "LTS" 버전 다운로드 (LTS는 Long Term Support의 약자로, 안정적인 버전입니다)
3. 다운로드한 파일을 실행하고 지시에 따라 설치
4. 설치 완료 후 컴퓨터 재부팅
5. 다시 `node --version` 명령어로 확인

### npm으로 Chart.js 설치하기

#### 단계별 설명

**1단계**: 프로젝트 폴더로 이동

터미널을 열고 프로젝트가 있는 폴더로 이동합니다.

```bash
# Windows
cd C:\Users\YourName\my-project

# Mac/Linux
cd /Users/YourName/my-project
```

**해설**:
- `cd`: "Change Directory"의 약자로, 폴더를 이동하는 명령어입니다
- 경로는 여러분의 실제 프로젝트 경로로 바꿔야 합니다

**2단계**: package.json 파일이 있는지 확인

```bash
# Windows
dir

# Mac/Linux
ls
```

`package.json` 파일이 목록에 없다면, 다음 명령어로 만듭니다:

```bash
npm init -y
```

**해설**:
- `npm init`: 새로운 npm 프로젝트를 시작합니다
- `-y`: 모든 질문에 기본값으로 자동 응답합니다 (처음이라면 이 옵션 사용 추천)

**3단계**: Chart.js 설치

```bash
npm install chart.js
```

**해설**:
- `npm install`: npm으로 패키지를 설치하는 명령어입니다
- `chart.js`: 설치할 패키지의 이름입니다

Enter를 누르고 기다리면... 설치가 진행됩니다! 

```
added 1 package, and audited 2 packages in 3s
found 0 vulnerabilities
```

이런 메시지가 나오면 성공입니다!

**4단계**: 설치 확인

프로젝트 폴더를 보면 다음과 같은 변화가 있습니다:

- `node_modules` 폴더가 생성됨 (Chart.js 파일들이 여기 있습니다)
- `package.json` 파일에 Chart.js가 추가됨
- `package-lock.json` 파일이 생성됨 (의존성 버전 관리용)

### npm으로 설치한 Chart.js 사용하기

#### 일반 JavaScript에서 사용하기

```javascript
// Chart.js 불러오기 (ES6 방식)
import { Chart } from 'chart.js';

// 또는 전체를 불러오기
import Chart from 'chart.js/auto';

// 이제 Chart를 사용할 수 있습니다
const ctx = document.getElementById('myChart');
new Chart(ctx, {
  type: 'bar',
  data: { /* ... */ }
});
```

**중요**: 이 방식은 **모듈 번들러(Webpack, Vite 등)**를 사용할 때만 작동합니다!

#### React에서 사용하기

```jsx
import { useEffect, useRef } from 'react';
import Chart from 'chart.js/auto';

function MyChart() {
  const chartRef = useRef(null);
  
  useEffect(() => {
    const ctx = chartRef.current;
    
    const myChart = new Chart(ctx, {
      type: 'bar',
      data: {
        labels: ['1월', '2월', '3월', '4월', '5월', '6월'],
        datasets: [{
          label: '판매량',
          data: [12, 19, 3, 5, 2, 3]
        }]
      }
    });
    
    // 컴포넌트가 사라질 때 차트 정리
    return () => {
      myChart.destroy();
    };
  }, []);
  
  return <canvas ref={chartRef} />;
}

export default MyChart;
```

#### Vue에서 사용하기

```vue
<template>
  <canvas ref="myChart"></canvas>
</template>

<script>
import { Chart } from 'chart.js/auto';
import { ref, onMounted, onUnmounted } from 'vue';

export default {
  setup() {
    const myChart = ref(null);
    let chartInstance = null;
    
    onMounted(() => {
      const ctx = myChart.value;
      
      chartInstance = new Chart(ctx, {
        type: 'bar',
        data: {
          labels: ['1월', '2월', '3월', '4월', '5월', '6월'],
          datasets: [{
            label: '판매량',
            data: [12, 19, 3, 5, 2, 3]
          }]
        }
      });
    });
    
    onUnmounted(() => {
      if (chartInstance) {
        chartInstance.destroy();
      }
    });
    
    return { myChart };
  }
}
</script>
```

### 특정 버전 설치하기

최신 버전 대신 특정 버전을 설치하고 싶다면:

```bash
# 정확한 버전 지정
npm install chart.js@4.4.1

# 4.x 버전 중 최신 버전
npm install chart.js@^4.0.0

# 4.4.x 버전 중 최신 버전
npm install chart.js@~4.4.0
```

**버전 표기법 설명**:
- `4.4.1`: 정확히 4.4.1 버전만 설치
- `^4.4.1`: 4.4.1 이상, 5.0.0 미만 (메이저 버전은 고정)
- `~4.4.1`: 4.4.1 이상, 4.5.0 미만 (마이너 버전까지 고정)

### 설치된 버전 확인하기

```bash
npm list chart.js
```

결과:

```
my-project@1.0.0
└── chart.js@4.4.1
```

### Chart.js 업데이트하기

최신 버전으로 업데이트하려면:

```bash
npm update chart.js
```

특정 버전으로 업데이트하려면:

```bash
npm install chart.js@latest
```

### Chart.js 제거하기

더 이상 필요 없다면:

```bash
npm uninstall chart.js
```

## 방법 3: GitHub에서 다운로드하기

### GitHub이란?

**GitHub**는 코드를 저장하고 공유하는 플랫폼입니다. Chart.js의 소스 코드가 여기에 공개되어 있습니다.

**이 방법을 사용해야 하는 경우**:
- Chart.js의 소스 코드를 직접 수정하고 싶을 때
- 개발 중인 최신 기능을 테스트하고 싶을 때
- Chart.js 개발에 기여하고 싶을 때

**주의**: 이 방법은 고급 개발자용입니다. 초보자는 권장하지 않습니다!

### 다운로드 방법

#### 방법 3-1: 릴리스 버전 다운로드 (권장)

1. [https://github.com/chartjs/Chart.js/releases](https://github.com/chartjs/Chart.js/releases) 접속
2. 원하는 버전 선택 (보통 가장 위의 최신 버전)
3. "Assets" 섹션에서 `Source code (zip)` 클릭
4. 다운로드한 zip 파일 압축 해제
5. `dist` 폴더 안의 `chart.js` 또는 `chart.min.js` 파일을 프로젝트에 복사

**중요**: 다운로드한 소스 코드는 **빌드(Build)**가 필요합니다! 

#### 방법 3-2: Git으로 클론하기

Git이 설치되어 있다면:

```bash
git clone https://github.com/chartjs/Chart.js.git
cd Chart.js
npm install
npm run build
```

**해설**:
- `git clone`: GitHub 저장소를 내 컴퓨터로 복사합니다
- `npm install`: 개발에 필요한 모든 패키지를 설치합니다
- `npm run build`: 소스 코드를 실제 사용 가능한 파일로 변환합니다

빌드가 완료되면 `dist` 폴더에 사용 가능한 파일들이 생성됩니다.

### 다운로드한 파일 사용하기

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>Chart.js 로컬 버전</title>
</head>
<body>
  <canvas id="myChart"></canvas>
  
  <!-- 다운로드한 파일 사용 -->
  <script src="./js/chart.min.js"></script>
  
  <script>
    // 차트 생성 코드
  </script>
</body>
</html>
```

**파일 경로 설명**:
- `./js/chart.min.js`: 현재 HTML 파일과 같은 위치의 `js` 폴더 안에 있는 파일
- 실제 경로는 여러분의 프로젝트 구조에 맞게 조정하세요

## 설치 문제 해결

### 문제 1: "Chart is not defined" 에러

**원인**: Chart.js 라이브러리가 제대로 로드되지 않았습니다.

**해결 방법**:

**CDN 사용 시**:
- 인터넷 연결 확인
- CDN URL이 정확한지 확인
- `<script>` 태그가 차트 생성 코드보다 먼저 있는지 확인

```html
<!-- 잘못된 순서 -->
<script>
  new Chart(ctx, { /* ... */ });  // 에러 발생!
</script>
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>

<!-- 올바른 순서 -->
<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>
<script>
  new Chart(ctx, { /* ... */ });  // 정상 작동
</script>
```

**npm 사용 시**:
- `import` 구문이 있는지 확인
- 번들러(Webpack, Vite 등)가 제대로 설정되었는지 확인

### 문제 2: npm install 시 에러

**에러 메시지**: `EACCES: permission denied`

**원인**: 권한 문제입니다.

**해결 방법 (Windows)**:
- 관리자 권한으로 터미널 실행

**해결 방법 (Mac/Linux)**:
```bash
sudo npm install chart.js
```

### 문제 3: 차트가 보이지 않음

**원인**: 캔버스 크기가 0이거나 보이지 않습니다.

**해결 방법**:

```html
<style>
  canvas {
    max-width: 600px;
    max-height: 400px;
  }
</style>

<div style="width: 80%; margin: 0 auto;">
  <canvas id="myChart"></canvas>
</div>
```

또는 JavaScript로 크기 지정:

```javascript
const canvas = document.getElementById('myChart');
canvas.width = 600;
canvas.height = 400;
```

### 문제 4: 버전 충돌

**원인**: 다른 라이브러리와 Chart.js 버전이 호환되지 않습니다.

**해결 방법**:
```bash
# 현재 설치된 버전 확인
npm list chart.js

# 특정 버전으로 재설치
npm uninstall chart.js
npm install chart.js@4.4.1
```

```html
<!DOCTYPE html>
<html lang="ko">

    new Chart(ctx, {
      type: 'bar',
      data: {
        labels: ['빨강', '파랑', '노랑', '초록', '보라', '주황'],
        datasets: [{
          label: '투표 수',

            'rgba(75, 192, 192, 0.7)',
            'rgba(153, 102, 255, 0.7)',

        }]

        responsive: true,
        scales: {
          y: {
            beginAtZero: true
          }
        }
      }
    });
  </script>

</html>
```
