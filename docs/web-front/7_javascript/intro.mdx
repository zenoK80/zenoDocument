# JavaScript란 무엇인가?

웹 개발의 세계에 오신 것을 환영합니다! 웹 개발을 시작할 때 가장 먼저,   
그리고 가장 많이 듣게 되는 단어가 바로 **JavaScript**(자바스크립트)입니다.

이 가이드에서는 JavaScript가 도대체 무엇인지, 어디서 왔는지,   
그리고 왜 전 세계 개발자들이 이 언어에 열광하는지 **아주 자세하고 친절하게** 설명해 드리겠습니다.

## JavaScript의 정체

간단히 말해, **JavaScript는 웹 페이지를 '살아 움직이게' 만드는 프로그래밍 언어**입니다.

웹 페이지를 구성하는 3대 요소로 비유해 볼까요?

### 웹의 3대 요소 비교

| 기술 | 역할 | 비유 (건축) | 비유 (인체) |
|:---:|:---:|:---:|:---:|
| **HTML** | **구조** 정의 | 건물의 뼈대와 기둥 | 뼈와 살 (구조) |
| **CSS** | **스타일** 지정 | 페인트, 인테리어, 장식 | 옷과 화장 (외관) |
| **JavaScript** | **동작** 구현 | 전기, 수도, 엘리베이터 | 근육과 뇌 (행동) |

**정리하자면**:
- HTML이 화면에 버튼을 *만들고*,
- CSS가 그 버튼을 예쁘게 *꾸미면*,
- **JavaScript는 사용자가 버튼을 클릭했을 때 무슨 일이 일어날지 *결정*합니다.**

## JavaScript의 탄생과 역사

JavaScript의 역사는 꽤 흥미롭고 극적입니다. 마치 실리콘밸리의 드라마 같습니다.

### 누가, 언제 만들었나요?

* **창시자**: 브렌던 아이크 (Brendan Eich)
* **회사**: 넷스케이프 (Netscape)
* **시기**: 1995년 5월
* **개발 기간**: 단 **10일**

놀랍게도, 초기 버전은 단 10일 만에 만들어졌습니다. 당시 넷스케이프는 웹 브라우저 시장을 장악하고 있었는데, 정적인 웹 페이지에 '동적인 요소'를 넣고 싶어 했습니다.

### 이름은 왜 JavaScript인가요?

이 부분이 많은 초보자를 헷갈리게 합니다.

> **주의**: **JavaScript와 Java는 아무런 관련이 없습니다!**
> 마치 '인도(India)'와 '인도네시아(Indonesia)'의 관계와 같습니다. 이름만 비슷할 뿐 전혀 다른 언어입니다.

원래 이름은 **Mocha(모카)**였고, 그 후 **LiveScript(라이브스크립트)**로 바뀌었습니다. 하지만 당시 'Java'라는 언어가 엄청난 인기를 끌고 있었습니다. 마케팅 효과를 노리기 위해 'Java'의 인기에 편승하고자 이름을 **JavaScript**로 바꾼 것입니다. (역사적인 마케팅 낚시였습니다!)

### 표준화와 ECMAScript

JavaScript가 인기를 끌자, 마이크로소프트 등 다른 회사들도 비슷한 언어를 만들기 시작했습니다. 표준이 필요해졌고, **ECMA**라는 국제 기구에서 표준을 정했습니다.

이 표준의 이름이 바로 **ECMAScript(이크마스크립트)**입니다.
* **JavaScript**: 언어의 이름 (우리가 부르는 말)
* **ECMAScript (ES)**: 언어의 사양과 표준 (설계도)

흔히 듣게 될 **ES6** (ECMAScript 2015)는 2015년에 발표된 버전으로, 현대 JavaScript의 가장 큰 전환점이 된 중요한 업데이트입니다.

## JavaScript는 어디에 사용되나요?

과거에는 단순히 브라우저에서 알림창을 띄우거나 메뉴를 펼치는 용도로만 쓰였습니다. 하지만 지금은 **거의 모든 곳**에 쓰입니다.

### 1. 웹 프론트엔드 (Web Frontend)
우리가 보는 웹 사이트의 모든 인터랙티브한 기능은 JavaScript로 만듭니다.
* React, Vue, Angular 같은 프레임워크가 모두 JavaScript 기반입니다.

### 2. 웹 백엔드 (Web Backend) - Node.js
2009년, **Node.js**의 등장으로 JavaScript가 브라우저 밖으로 나왔습니다. 이제 서버(Server)도 JavaScript로 만들 수 있습니다.
* Netflix, Uber, PayPal 등이 Node.js를 사용합니다.

### 3. 모바일 앱 (Mobile Apps)
**React Native** 등을 사용하면 JavaScript로 아이폰과 안드로이드 앱을 동시에 개발할 수 있습니다. (페이스북, 인스타그램 앱이 이렇게 만들어졌습니다.)

### 4. 데스크톱 프로그램
**Electron**을 사용하면 PC 프로그램도 만듭니다. 여러분이 쓰는 **VS Code**, **Slack**, **Discord**가 모두 JavaScript로 만들어졌습니다.

## 왜 JavaScript를 배워야 하나요?

개발자가 되고 싶다면 JavaScript는 **선택이 아닌 필수**에 가깝습니다.

### 이유 1: 압도적인 사용량
Stack Overflow 설문조사에 따르면, JavaScript는 **수년 연속 전 세계에서 가장 많이 사용되는 언어 1위**입니다. 웹이 존재하는 한 JavaScript는 사라지지 않습니다.

### 이유 2: 낮은 진입 장벽
다른 언어(C, Java, Python 등)는 개발 환경을 설정하는 것부터 복잡할 수 있습니다. 하지만 JavaScript는 여러분이 이미 가지고 있는 **웹 브라우저(크롬 등)**만 있으면 바로 실행할 수 있습니다.

### 이유 3: 활발한 커뮤니티
사용자가 많다는 것은 질문할 곳도 많고, 배울 자료도 넘쳐난다는 뜻입니다. 문제가 생겼을 때 구글링하면 99.9% 해결책이 나옵니다.

## 간단한 예제: Hello JavaScript!

백문이 불여일견! JavaScript가 어떻게 작동하는지 아주 간단한 코드로 확인해 봅시다.

### 코드 예시

```html
<!DOCTYPE html>
<html>
<body>

  <h1>내 첫 JavaScript</h1>
  <button type="button" onclick="changeText()">클릭해보세요!</button>

  <p id="demo">이 텍스트가 바뀔 것입니다.</p>

  <script>
    function changeText() {
      document.getElementById("demo").innerHTML = "와우! JavaScript가 작동했어요! 🎉";
      document.getElementById("demo").style.color = "blue";
    }
  </script>

</body>
</html>
