# Tailwind CSS 설치하기

이 가이드에서는 **Vite**와 함께 Tailwind CSS를 설치하는 방법을 단계별로 알아보겠습니다.

## Tailwind CSS의 작동 원리

먼저 Tailwind CSS가 어떻게 작동하는지 이해해봅시다.

### 작동 방식 개요

Tailwind CSS는 다음과 같은 과정으로 작동합니다:

1. **스캔 (Scan)**: 프로젝트의 모든 HTML, JavaScript, Vue, React 파일들을 스캔합니다
2. **감지 (Detect)**: 이 파일들에서 사용된 Tailwind 클래스 이름들을 찾아냅니다
3. **생성 (Generate)**: 실제로 사용된 클래스에 해당하는 CSS 스타일만 생성합니다
4. **출력 (Write)**: 생성된 스타일들을 하나의 CSS 파일로 만듭니다

**예를 들어:**

```html
<!-- 여러분의 HTML 파일 -->
<button class="bg-blue-500 text-white px-4 py-2">
  클릭
</button>
```

이 코드가 있으면 Tailwind는:
- `bg-blue-500` → 파란색 배경 스타일 생성
- `text-white` → 흰색 글자 스타일 생성
- `px-4` → 좌우 padding 스타일 생성
- `py-2` → 위아래 padding 스타일 생성

**사용하지 않은 수천 개의 다른 클래스들은 최종 CSS 파일에 포함되지 않습니다!**

이것이 Tailwind가 빠르고 효율적인 이유입니다. 필요한 스타일만 포함되기 때문에 최종 CSS 파일이 매우 작습니다 (보통 10KB 이하).

### 런타임 제로 (Zero-Runtime)

**런타임**(Runtime)이란 프로그램이 실행되는 동안을 의미합니다.

Tailwind는 "Zero-Runtime"입니다. 즉:
- 브라우저에서 실행될 때 JavaScript로 스타일을 계산하지 않습니다
- 미리 빌드 시점에 모든 CSS를 만들어 둡니다
- 웹사이트가 더 빠르게 로드됩니다

**비교:**
- **CSS-in-JS (런타임 있음)**: 브라우저에서 JavaScript가 실행되면서 스타일을 계산 → 느림
- **Tailwind (런타임 없음)**: 이미 준비된 CSS 파일만 로드 → 빠름

## 설치 방법 총정리

Tailwind CSS를 설치하는 방법은 여러 가지가 있습니다. 각 방법을 간단히 살펴보겠습니다.

### 1. Vite 플러그인 사용 (권장 ⭐)

**누구에게 적합한가요?**
- React, Vue, Svelte 등 모던 프레임워크를 사용하는 분
- 가장 빠르고 간단한 개발 환경을 원하는 분
- **초보자에게 가장 추천!**

**장점:**
- 설정이 가장 간단함
- 개발 서버가 매우 빠름
- 핫 리로드 (코드 수정하면 자동으로 화면 갱신)
- 최신 기술 스택

**이 가이드에서 다룰 방법입니다.**

### 2. PostCSS 플러그인 사용

**누구에게 적합한가요?**
- 이미 PostCSS를 사용 중인 프로젝트
- Next.js, Nuxt.js 같은 특정 프레임워크 사용자
- 더 세밀한 CSS 처리 제어가 필요한 분

**장점:**
- 다른 PostCSS 플러그인과 함께 사용 가능
- 기존 빌드 시스템에 통합하기 쉬움

**단점:**
- 설정이 약간 더 복잡함

### 3. Tailwind CLI 사용

**누구에게 적합한가요?**
- 프레임워크 없이 순수 HTML/CSS만 사용하는 분
- 간단한 정적 웹사이트를 만드는 분
- 빌드 도구를 최소화하고 싶은 분

**장점:**
- 별도의 빌드 도구 설치 불필요
- 매우 간단한 프로젝트에 적합

**단점:**
- 고급 기능 사용에 제한
- 개발 경험이 다소 불편할 수 있음

### 4. 프레임워크 가이드

**누구에게 적합한가요?**
- Next.js, SvelteKit, Astro 등 특정 프레임워크 사용자

각 프레임워크마다 최적화된 설치 방법이 별도로 제공됩니다.

### 5. Play CDN (개발/학습용만)

**누구에게 적합한가요?**
- Tailwind를 빠르게 체험해보고 싶은 분
- 간단한 실험이나 CodePen 같은 온라인 에디터 사용자

**사용 방법:**
```html
<!DOCTYPE html>
<html>
<head>
  <script src="https://cdn.tailwindcss.com"></script>
</head>
<body>
  <h1 class="text-3xl font-bold underline">
    Hello world!
  </h1>
</body>
</html>
```

**⚠️ 중요: 실제 프로젝트에서는 사용하지 마세요!**
- 파일 크기가 매우 큼 (전체 Tailwind CSS 로드)
- 성능이 매우 느림
- 커스터마이징 불가능
- **오직 학습과 프로토타이핑용으로만 사용!**

## Vite와 함께 Tailwind CSS 설치하기

이제 본격적으로 Vite를 사용한 설치 방법을 단계별로 알아보겠습니다.

### 사전 준비사항

설치를 시작하기 전에 다음이 필요합니다:

#### 1. Node.js 설치

**Node.js란?**
- JavaScript를 브라우저 밖에서도 실행할 수 있게 해주는 환경
- npm (Node Package Manager)이 함께 설치됩니다

**설치 방법:**
1. [nodejs.org](https://nodejs.org) 방문
2. LTS (Long Term Support) 버전 다운로드 (권장)
3. 설치 프로그램 실행
4. 기본 설정으로 설치

**설치 확인:**
터미널(Windows의 경우 명령 프롬프트 또는 PowerShell)을 열고 다음 명령어를 입력:

```bash
node --version
```

버전이 출력되면 (예: `v20.10.0`) 정상적으로 설치된 것입니다.

```bash
npm --version
```

npm 버전도 확인합니다 (예: `10.2.3`).

#### 2. 코드 에디터

**추천:**
- **Visual Studio Code (VS Code)**: 무료, 가장 인기 있는 에디터
- 다운로드: [code.visualstudio.com](https://code.visualstudio.com)

**유용한 VS Code 확장 프로그램:**
- **Tailwind CSS IntelliSense**: 자동완성 기능 제공
- **Prettier**: 코드 자동 정리

#### 3. 터미널 사용 기본 지식

**터미널이란?**
- 명령어로 컴퓨터를 제어하는 프로그램
- Windows: 명령 프롬프트, PowerShell, Git Bash
- Mac: Terminal
- VS Code에 내장된 터미널도 사용 가능 (단축키: Ctrl + `)

**기본 명령어:**
```bash
# 현재 디렉토리 확인
pwd

# 디렉토리 내용 보기
ls        # Mac/Linux
dir       # Windows

# 디렉토리 이동
cd 폴더이름

# 상위 디렉토리로 이동
cd ..
```

## 단계별 설치 가이드

### 1단계: Vite 프로젝트 생성

**Vite란?**
- 매우 빠른 프론트엔드 빌드 도구
- 프랑스어로 "빠르다"는 뜻
- 개발 서버가 즉시 시작됨
- 핫 모듈 리플레이스먼트 (HMR) 지원

#### 새 프로젝트 만들기

터미널을 열고 프로젝트를 만들고 싶은 폴더로 이동한 후:

```bash
npm create vite@latest my-project
```

**명령어 분석:**
- `npm`: Node Package Manager 실행
- `create`: 새로운 프로젝트 생성 명령
- `vite@latest`: Vite의 최신 버전 사용
- `my-project`: 프로젝트 이름 (원하는 이름으로 변경 가능)

**실행하면 나타나는 질문들:**

```bash
? Select a framework: › - Use arrow-keys. Return to submit.
❯   Vanilla      # 순수 JavaScript (프레임워크 없음)
    Vue          # Vue.js
    React        # React
    Preact       # 가벼운 React 대안
    Lit          # 웹 컴포넌트
    Svelte       # Svelte
    Solid        # SolidJS
    Qwik         # Qwik
    Others       # 기타
```

**화살표 키로 선택하고 Enter를 누릅니다.**

예를 들어 React를 선택하면:

```bash
? Select a variant: › - Use arrow-keys. Return to submit.
❯   TypeScript              # TypeScript 사용
    TypeScript + SWC        # TypeScript + 빠른 컴파일러
    JavaScript              # 순수 JavaScript
    JavaScript + SWC        # JavaScript + 빠른 컴파일러
```

**초보자는 JavaScript를 선택하는 것을 추천합니다.**

생성이 완료되면:

```bash
Scaffolding project in /경로/my-project...

Done. Now run:

  cd my-project
  npm install
  npm run dev
```

#### 프로젝트 폴더로 이동

```bash
cd my-project
```

**`cd`란?**
- "Change Directory"의 약자
- 폴더를 이동하는 명령어

이제 여러분은 `my-project` 폴더 안에 있습니다.

#### 생성된 프로젝트 구조

```
my-project/
├── node_modules/        # 설치된 패키지들 (아직 없음)
├── public/              # 정적 파일 (이미지, 폰트 등)
├── src/                 # 소스 코드
│   ├── App.jsx          # 메인 컴포넌트
│   ├── main.jsx         # 진입점
│   └── index.css        # 메인 CSS 파일
├── index.html           # HTML 파일
├── package.json         # 프로젝트 정보와 의존성
└── vite.config.js       # Vite 설정 파일
```

### 2단계: 의존성 패키지 설치

**의존성(Dependency)이란?**
- 프로젝트가 작동하기 위해 필요한 다른 패키지들
- 예: React, Vite, Tailwind CSS 등

먼저 Vite가 필요로 하는 기본 패키지들을 설치합니다:

```bash
npm install
```

**이 명령어가 하는 일:**
1. `package.json` 파일을 읽습니다
2. 필요한 모든 패키지를 다운로드합니다
3. `node_modules` 폴더에 저장합니다

**실행 결과:**
```bash
added 150 packages in 30s
```

숫자는 다를 수 있지만, 많은 패키지가 설치되는 것이 정상입니다.

이제 Tailwind CSS를 설치합니다:

```bash
npm install tailwindcss @tailwindcss/vite
```

**명령어 분석:**
- `npm install`: 패키지를 설치하는 명령어
- `tailwindcss`: Tailwind CSS 핵심 패키지
- `@tailwindcss/vite`: Vite와 통합하기 위한 플러그인

**`@` 기호가 붙은 패키지란?**
- 스코프 패키지(Scoped Package)라고 부릅니다
- `@조직이름/패키지이름` 형식
- 같은 이름의 패키지가 있을 수 있으므로 조직으로 구분

**설치 확인:**

`package.json` 파일을 열어보면:

```json
{
  "name": "my-project",
  "private": true,
  "version": "0.0.0",
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "vite build",
    "preview": "vite preview"
  },
  "dependencies": {
    "react": "^18.2.0",
    "react-dom": "^18.2.0",
    "tailwindcss": "^4.0.0",           // ← 추가됨!
    "@tailwindcss/vite": "^4.0.0"      // ← 추가됨!
  },
  "devDependencies": {
    "vite": "^5.0.0"
  }
}
```

`dependencies` 섹션에 Tailwind 관련 패키지가 추가된 것을 확인할 수 있습니다.

### 3단계: Vite 플러그인 설정

이제 Vite가 Tailwind CSS를 사용할 수 있도록 설정합니다.

#### vite.config.js 파일 수정

프로젝트 루트(최상위 폴더)에 있는 `vite.config.js` 파일을 엽니다.

**기존 내용 (React를 선택했다면):**

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [react()],
})
```

**수정된 내용:**

```javascript
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'
import tailwindcss from '@tailwindcss/vite'  // ← 이 줄 추가

// https://vitejs.dev/config/
export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),  // ← 이 줄 추가
  ],
})
```

**코드 설명:**

1. **import 문:**
```javascript
import tailwindcss from '@tailwindcss/vite'
```
- `import`: 다른 파일이나 패키지의 코드를 가져옵니다
- `tailwindcss`: 우리가 사용할 이름 (변수명)
- `from '@tailwindcss/vite'`: 어디서 가져올지 지정

2. **plugins 배열:**
```javascript
plugins: [
  react(),       // React 플러그인
  tailwindcss(), // Tailwind CSS 플러그인
],
```
- `plugins`: Vite가 사용할 플러그인들의 목록
- 배열(`[]`)에 여러 플러그인을 넣을 수 있습니다
- 각 플러그인은 함수 형태로 호출 (`()` 붙이기)

**순서가 중요한가요?**
- 일반적으로는 순서가 중요하지 않습니다
- 하지만 특정 프레임워크 플러그인(React, Vue 등)을 먼저 두는 것이 관례입니다

**다른 프레임워크의 경우:**

**Vue를 사용한다면:**
```javascript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [
    vue(),
    tailwindcss(),
  ],
})
```

**순수 JavaScript (Vanilla)를 사용한다면:**
```javascript
import { defineConfig } from 'vite'
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [
    tailwindcss(),
  ],
})
```

### 4단계: CSS 파일에 Tailwind 임포트

이제 Tailwind CSS를 프로젝트에서 사용할 수 있도록 CSS 파일에 임포트합니다.

#### src/index.css 파일 수정

`src` 폴더 안의 `index.css` (또는 `style.css`) 파일을 엽니다.

**기존 내용을 모두 삭제하고** 다음 한 줄만 추가합니다:

```css
@import "tailwindcss";
```

**코드 설명:**

- `@import`: CSS에서 다른 CSS 파일을 가져오는 명령어
- `"tailwindcss"`: Tailwind CSS의 모든 스타일을 의미
- 세미콜론(`;`)으로 끝나야 함

**이 한 줄이 하는 일:**

1. Tailwind CSS의 기본 스타일들을 가져옵니다
2. 프로젝트에서 사용된 유틸리티 클래스들을 포함합니다
3. 최종 CSS 파일을 생성합니다

**왜 기존 내용을 삭제하나요?**
- 기존 CSS는 Vite가 자동 생성한 샘플 스타일입니다
- Tailwind를 사용하면 대부분 필요 없습니다
- 필요한 커스텀 스타일이 있다면 `@import` 아래에 추가하면 됩니다

**커스텀 스타일 추가 예시:**

```css
@import "tailwindcss";

/* 여기에 커스텀 스타일 추가 가능 */
body {
  font-family: 'Noto Sans KR', sans-serif;
}

.custom-class {
  /* 특별한 스타일 */
}
```

#### CSS 파일이 임포트되는지 확인

**React를 사용하는 경우:**

`src/main.jsx` 파일을 열어봅니다:

```javascript
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'
import './index.css'  // ← 이 줄이 있어야 합니다!

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <App />
  </React.StrictMode>,
)
```

`import './index.css'` 줄이 있으면 정상입니다.

**순수 JavaScript를 사용하는 경우:**

`src/main.js` 파일에:

```javascript
import './style.css'  // ← CSS 파일 임포트

// 나머지 코드...
```

### 5단계: 개발 서버 실행

이제 모든 설정이 완료되었습니다! 개발 서버를 실행해봅시다.

터미널에서 다음 명령어를 입력합니다:

```bash
npm run dev
```

**명령어 분석:**
- `npm run`: npm 스크립트를 실행
- `dev`: `package.json`의 `scripts` 섹션에 정의된 명령어

**`package.json`에서 확인:**
```json
"scripts": {
  "dev": "vite",           // ← 이 명령어가 실행됨
  "build": "vite build",
  "preview": "vite preview"
}
```

**실행 결과:**

### 6단계: Tailwind 클래스 사용하기

이제 실제로 Tailwind CSS를 사용해봅시다!

#### React를 사용하는 경우

`src/App.jsx` 파일을 열고 수정합니다:

```jsx
function App() {
  return (
    <div className="min-h-screen bg-gray-100 flex items-center justify-center">
      <div className="bg-white p-8 rounded-lg shadow-lg">
        <h1 className="text-3xl font-bold text-blue-600 mb-4">
          Hello Tailwind CSS!
        </h1>
        <p className="text-gray-700 mb-4">
          Tailwind CSS가 정상적으로 작동하고 있습니다.
        </p>
        <button className="bg-blue-500 hover:bg-blue-600 text-white font-semibold py-2 px-4 rounded transition-colors">
          클릭해보세요
        </button>
      </div>
    </div>
  )
}

export default App
```

**코드 분석 (외부 컨테이너):**

```jsx
<div className="min-h-screen bg-gray-100 flex items-center justify-center">
```

- `min-h-screen`: 최소 높이를 화면 전체 높이(100vh)로 설정
- `bg-gray-100`: 배경색을 매우 연한 회색으로
- `flex`: Flexbox 레이아웃 활성화 (요소들을 정렬하기 위해)
- `items-center`: 세로 방향 가운데 정렬
- `justify-center`: 가로 방향 가운데 정렬

**결과:** 화면 전체를 연한 회색 배경으로 만들고, 내부 요소를 정중앙에 배치

**코드 분석 (카드):**

```jsx
<div className="bg-white p-8 rounded-lg shadow-lg">
```

- `bg-white`: 흰색 배경
- `p-8`: 모든 방향에 32px(8 × 4px)의 안쪽 여백(padding)
- `rounded-lg`: 큰(large) 크기의 둥근 모서리
- `shadow-lg`: 큰 그림자 효과

**결과:** 흰색 카드 모양의 박스

**코드 분석 (제목):**

```jsx
<h1 className="text-3xl font-bold text-blue-600 mb-4">
```

- `text-3xl`: 매우 큰 글자 크기 (30px)
- `font-bold`: 굵은 글자 (font-weight: 700)
- `text-blue-600`: 파란색 글자 (중간 정도 진하기)
- `mb-4`: 아래쪽 margin 16px (4 × 4px)

**코드 분석 (버튼):**

```jsx
<button className="bg-blue-500 hover:bg-blue-600 text-white font-semibold py-2 px-4 rounded transition-colors">
```

- `bg-blue-500`: 파란색 배경
- `hover:bg-blue-600`: 마우스 올리면 더 진한 파란색
- `text-white`: 흰색 글자
- `font-semibold`: 중간 굵기 글자 (font-weight: 600)
- `py-2`: 위아래 padding 8px
- `px-4`: 좌우 padding 16px
- `rounded`: 둥근 모서리
- `transition-colors`: 색상 변화를 부드럽게 애니메이션

#### 순수 HTML을 사용하는 경우

`index.html` 파일을 수정합니다:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Tailwind CSS 시작하기</title>
  <link rel="stylesheet" href="/src/style.css">
</head>
<body>
  <div class="min-h-screen bg-gray-100 flex items-center justify-center">
    <div class="bg-white p-8 rounded-lg shadow-lg">
      <h1 class="text-3xl font-bold text-blue-600 mb-4">
        Hello Tailwind CSS!
      </h1>
      <p class="text-gray-700 mb-4">
        Tailwind CSS가 정상적으로 작동하고 있습니다.
      </p>
      <button class="bg-blue-500 hover:bg-blue-600 text-white font-semibold py-2 px-4 rounded transition-colors">
        클릭해보세요
      </button>
    </div>
  </div>
</body>
</html>
```

**React와의 차이점:**
- `className` 대신 `class` 사용
- JSX가 아닌 일반 HTML

#### 파일 저장 후 확인

1. 파일을 저장합니다 (Ctrl + S 또는 Cmd + S)
2. 브라우저가 자동으로 새로고침됩니다 (핫 리로드!)
3. 스타일이 적용된 화면을 확인합니다

**스타일이 보이지 않는다면:**

1. 개발 서버가 실행 중인지 확인 (`npm run dev`)
2. 터미널에 에러 메시지가 없는지 확인
3. 브라우저 콘솔을 열어서 (F12) 에러 확인
4. CSS 파일에 `@import "tailwindcss";`가 있는지 확인
5. `vite.config.js`에 Tailwind 플러그인이 추가되었는지 확인

### 반응형 카드 만들기

```html
<div class="max-w-md mx-auto bg-white rounded-xl shadow-md overflow-hidden md:max-w-2xl">
  <div class="md:flex">
    <div class="md:shrink-0">
      <img class="h-48 w-full object-cover md:h-full md:w-48" 
           src="https://via.placeholder.com/300x200" 
           alt="이미지">
    </div>
    <div class="p-8">
      <div class="uppercase tracking-wide text-sm text-indigo-500 font-semibold">
        카테고리
      </div>
      <h2 class="block mt-1 text-lg leading-tight font-medium text-black">
        카드 제목
      </h2>
      <p class="mt-2 text-gray-500">
        카드에 대한 설명이 여기에 들어갑니다. Tailwind CSS를 사용하면 이렇게 쉽게 만들 수 있습니다.
      </p>
    </div>
  </div>
</div>
```

**반응형 클래스 설명:**

- `md:flex`: 중간 크기 화면(768px) 이상에서 Flexbox 활성화
- `md:max-w-2xl`: 중간 크기 이상에서 최대 너비 증가
- `md:h-full`: 중간 크기 이상에서 높이 100%
- `md:w-48`: 중간 크기 이상에서 너비 192px

**결과:**
- 작은 화면: 이미지가 위, 내용이 아래 (세로 배치)
- 큰 화면: 이미지가 왼쪽, 내용이 오른쪽 (가로 배치)


**새로운 클래스:**

- `space-x-7`: 가로 방향 자식 요소들 사이에 28px 간격
- `space-x-1`: 가로 방향 자식 요소들 사이에 4px 간격
- `border-b-4`: 아래쪽 테두리 4px
- `transition duration-300`: 300ms 애니메이션 효과

### 폼(Form) 만들기

```html
<div class="max-w-md mx-auto mt-8 bg-white p-6 rounded-lg shadow-lg">
  <h2 class="text-2xl font-bold mb-6 text-gray-800">로그인</h2>
  
  <form>
    <!-- 이메일 -->
    <div class="mb-4">
      <label class="block text-gray-700 text-sm font-bold mb-2" for="email">
        이메일
      </label>
      <input 
        class="shadow appearance-none border rounded w-full py-2 px-3 text-gray-700 leading-tight focus:outline-none focus:shadow-outline"
        id="email" 
        type="email" 
        placeholder="your@email.com">
    </div>
    
    <!-- 비밀번호 -->
    <div class="mb-6">
      <label class="block text-gray-700 text-sm font-bold mb-2" for="password">
        비밀번호
      </label>
      <input 
        class="shadow appearance-none border rounded w-full py-2 px-3 text-gray-700 mb-3 leading-tight focus:outline-none focus:shadow-outline"
        id="password" 
        type="password" 
        placeholder="••••••••">
    </div>
    
    <!-- 버튼 -->
    <div class="flex items-center justify-between">
      <button 
        class="bg-blue-500 hover:bg-blue-700 text-white font-bold py-2 px-4 rounded focus:outline-none focus:shadow-outline"
        type="button">
        로그인
      </button>
      <a class="inline-block align-baseline font-bold text-sm text-blue-500 hover:text-blue-800" href="#">
        비밀번호 찾기
      </a>
    </div>
  </form>
</div>
```

**폼 관련 클래스:**

- `appearance-none`: 브라우저 기본 스타일 제거
- `leading-tight`: 줄 간격을 좁게
- `focus:outline-none`: 포커스 시 기본 아웃라인 제거
- `focus:shadow-outline`: 포커스 시 커스텀 그림자
- `w-full`: 너비 100%

### 자주 묻는 질문

**Q: 개발 서버를 재시작해야 하나요?**  
A: 일반적으로 필요 없습니다. Vite의 핫 리로드가 자동으로 변경사항을 반영합니다.   
단, `vite.config.js`를 수정한 경우에는 재시작이 필요합니다.

**Q: 프로덕션 빌드는 어떻게 하나요?**  
A: `npm run build` 명령어를 실행하면 `dist` 폴더에 최적화된 파일들이 생성됩니다.

**Q: CSS 파일 크기가 너무 큰데요?**  
A: 개발 모드에서는 모든 클래스가 포함되어 클 수 있습니다.   
프로덕션 빌드(`npm run build`)를 하면 실제 사용한 클래스만 포함되어 매우 작아집니다.

**Q: VS Code에서 자동완성이 안 돼요**  
A: "Tailwind CSS IntelliSense" 확장 프로그램을 설치하세요.
