# GSAP 설치하기

GSAP을 사용하는 방법은 크게 세 가지가 있습니다. 각각의 방법은 프로젝트 상황에 따라 선택하시면 됩니다.

## 어떤 방법을 선택해야 할까요?

시작하기 전에, 자신의 상황에 맞는 설치 방법을 선택해보세요:

### 1. HTML 파일만으로 간단히 테스트하고 싶어요
→ **CDN 방식** 추천 (가장 쉽고 빠름)

### 2. React, Vue, Next.js 같은 프레임워크를 사용해요
→ **NPM 방식** 추천 (프로젝트 관리에 용이)

### 3. Yarn 패키지 매니저를 사용해요
→ **Yarn 방식** 추천

걱정하지 마세요! 모든 방법은 결국 같은 GSAP을 사용하는 것이고, 나중에 방법을 바꿀 수도 있습니다.

---

## 방법 1: CDN으로 설치하기 (가장 간단함!)

CDN(Content Delivery Network)은 인터넷에 올라와 있는 GSAP 파일을 바로 가져다 쓰는 방식입니다. 
별도로 다운로드하거나 설치할 필요 없이, HTML 파일에 한 줄만 추가하면 됩니다.

### 장점
- 설치 과정이 필요 없음
- 빠르게 테스트하고 배울 수 있음
- 간단한 프로젝트에 적합

### 단점
- 인터넷 연결이 필요함
- 대규모 프로젝트에는 적합하지 않음

### 사용 방법

HTML 파일의 `</body>` 태그 바로 위에 다음 코드를 추가하세요:

```html
<!DOCTYPE html>
<html lang="ko">
<head>
  <meta charset="UTF-8">
  <title>GSAP 테스트</title>
  <style>
    .box {
      width: 100px;
      height: 100px;
      background-color: #3498db;
      margin: 50px;
    }
  </style>
</head>
<body>
  
  <div class="box"></div>

  <!-- 1. GSAP 라이브러리 불러오기 -->
  <script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js"></script>
  
  <!-- 2. 여기에 애니메이션 코드 작성 -->
  <script>
    // 이제 GSAP을 사용할 수 있습니다!
    gsap.to(".box", {
      x: 200,
      rotation: 360,
      duration: 2
    });
  </script>

</body>
</html>
```

**코드 설명:**
- `<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js"></script>`
  - 이 한 줄이 GSAP을 불러오는 코드입니다
  - `@3.12.5`는 버전을 의미합니다 (최신 버전은 공식 사이트에서 확인하세요)
  - `gsap.min.js`는 압축된 파일로, 용량이 작아 빠르게 로딩됩니다

이제 파일을 저장하고 브라우저로 열어보세요. 파란 상자가 움직이는 것을 볼 수 있습니다!

### CDN으로 플러그인 추가하기

GSAP의 추가 기능(플러그인)을 사용하고 싶다면, 다음과 같이 추가할 수 있습니다:

```html
<!-- GSAP 메인 파일 -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js"></script>

<!-- ScrollTrigger 플러그인 (스크롤 애니메이션) -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/ScrollTrigger.min.js"></script>

<!-- 플러그인 사용하기 -->
<script>
  // 플러그인을 등록해야 사용할 수 있습니다
  gsap.registerPlugin(ScrollTrigger);
  
  // 이제 ScrollTrigger를 사용할 수 있습니다
  gsap.to(".box", {
    scrollTrigger: ".box",  // .box가 화면에 보이면
    x: 200,
    duration: 2
  });
</script>
```

**자주 사용하는 플러그인 CDN:**

```html
<!-- ScrollTrigger: 스크롤 기반 애니메이션 -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/ScrollTrigger.min.js"></script>

<!-- Draggable: 드래그 앤 드롭 -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/Draggable.min.js"></script>

<!-- MotionPath: 경로를 따라 움직이기 -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/MotionPathPlugin.min.js"></script>
```

## 방법 2: NPM으로 설치하기 (React, Vue 등)

NPM(Node Package Manager)은 JavaScript 패키지를 관리하는 도구입니다. 
React, Vue, Next.js 같은 현대적인 프레임워크를 사용한다면 이 방법을 추천합니다.

### 장점
- 프로젝트 의존성을 체계적으로 관리
- 버전 관리가 쉬움
- 빌드 도구와 잘 통합됨
- 인터넷 연결 없이도 개발 가능

### 단점
- Node.js와 npm이 설치되어 있어야 함
- 초보자에게는 다소 복잡할 수 있음

### 1단계: GSAP 설치하기

터미널(또는 명령 프롬프트)을 열고 프로젝트 폴더로 이동한 후 다음 명령어를 실행하세요:

```bash
npm install gsap
```

**명령어 설명:**
- `npm`: Node Package Manager를 실행하는 명령어
- `install`: 패키지를 설치하라는 명령
- `gsap`: 설치할 패키지 이름

이 명령어를 실행하면:
1. GSAP이 `node_modules` 폴더에 다운로드됩니다
2. `package.json` 파일에 GSAP이 의존성으로 추가됩니다

설치가 완료되면 다음과 같은 메시지를 볼 수 있습니다:

```bash
added 1 package, and audited 2 packages in 3s
```

### 2단계: GSAP 불러오기 (Import)

JavaScript 파일 상단에서 GSAP을 import해서 사용합니다:

```javascript
// GSAP 메인 라이브러리 불러오기
import { gsap } from "gsap";

// 이제 GSAP을 사용할 수 있습니다
gsap.to(".box", {
  x: 200,
  duration: 2
});
```

**import 문법 설명:**
- `import`: 다른 파일이나 패키지를 가져오는 키워드
- `{ gsap }`: gsap이라는 이름으로 가져옵니다 (중괄호는 "특정 것만 가져오기"를 의미)
- `from "gsap"`: gsap 패키지에서 가져옵니다

### 3단계: 플러그인 사용하기

플러그인을 사용하려면 함께 import하고 등록해야 합니다:

```javascript
// GSAP과 ScrollTrigger 플러그인 불러오기
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

// 플러그인 등록하기 (이 과정을 꼭 해야 사용 가능!)
gsap.registerPlugin(ScrollTrigger);

// 이제 ScrollTrigger를 사용할 수 있습니다
gsap.to(".box", {
  scrollTrigger: {
    trigger: ".box",
    start: "top center",
    end: "bottom center",
    markers: true  // 개발 중 위치 확인용 마커 표시
  },
  x: 200,
  duration: 2
});
```

**왜 registerPlugin을 해야 하나요?**

플러그인을 import만 하면 GSAP이 그 플러그인의 존재를 모릅니다. 
`gsap.registerPlugin()`을 호출하면 "이 플러그인을 사용할 거야!"라고 GSAP에게 알려주는 것입니다.

```javascript
// ❌ 잘못된 예 - 플러그인 등록 안 함
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

gsap.to(".box", {
  scrollTrigger: ".box",  // 작동하지 않음!
  x: 200
});

// ✅ 올바른 예 - 플러그인 등록함
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

gsap.registerPlugin(ScrollTrigger);  // 이 줄이 중요!

gsap.to(".box", {
  scrollTrigger: ".box",  // 정상 작동!
  x: 200
});
```

### 여러 플러그인 한 번에 불러오기

```javascript
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";
import { Draggable } from "gsap/Draggable";
import { MotionPathPlugin } from "gsap/MotionPathPlugin";

// 여러 플러그인을 한 번에 등록
gsap.registerPlugin(ScrollTrigger, Draggable, MotionPathPlugin);

// 이제 모든 플러그인 사용 가능
```

### React에서 사용하는 예제

React 컴포넌트에서 GSAP을 사용하는 방법:

```javascript
import { useEffect, useRef } from 'react';
import { gsap } from 'gsap';

function MyComponent() {
  // 애니메이션할 요소에 대한 참조 생성
  const boxRef = useRef(null);
  
  useEffect(() => {
    // 컴포넌트가 마운트되면 애니메이션 실행
    gsap.to(boxRef.current, {
      x: 200,
      rotation: 360,
      duration: 2
    });
  }, []); // 빈 배열 = 컴포넌트 마운트 시 한 번만 실행
  
  return (
    <div>
      <div 
        ref={boxRef}  // ref 연결
        className="box"
        style={{
          width: '100px',
          height: '100px',
          backgroundColor: '#3498db'
        }}
      />
    </div>
  );
}

export default MyComponent;
```

**코드 설명:**
1. `useRef(null)`: 애니메이션할 DOM 요소를 참조하기 위한 ref 생성
2. `ref={boxRef}`: div 요소에 ref 연결
3. `useEffect`: 컴포넌트가 화면에 나타난 후 실행될 코드
4. `boxRef.current`: ref가 가리키는 실제 DOM 요소
5. `[]`: 의존성 배열이 비어있어서 컴포넌트가 처음 렌더링될 때만 실행

### React 전용 훅: useGSAP

React에서 더 안전하고 편리하게 GSAP을 사용하려면 `@gsap/react` 패키지를 설치하세요:

```bash
npm install @gsap/react
```

사용 예제:

```javascript
import { useRef } from 'react';
import { gsap } from 'gsap';
import { useGSAP } from '@gsap/react';

function MyComponent() {
  const container = useRef(null);
  
  // useEffect 대신 useGSAP 사용
  useGSAP(() => {
    gsap.to(".box", {
      x: 200,
      rotation: 360,
      duration: 2
    });
  }, { scope: container }); // 이 컨테이너 내부에서만 .box 선택
  
  return (
    <div ref={container}>
      <div className="box" />
    </div>
  );
}
```

**useGSAP의 장점:**
- 컴포넌트가 언마운트될 때 애니메이션을 자동으로 정리
- 메모리 누수 방지
- 특정 컨테이너 안에서만 선택자 적용 가능

## 방법 3: Yarn으로 설치하기

Yarn은 npm의 대안 패키지 매니저입니다. npm과 거의 동일하게 작동하지만, 일부 개발자들이 선호합니다.

### GSAP 설치

```bash
yarn add gsap
```

### 사용 방법

npm과 동일합니다:

```javascript
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

gsap.registerPlugin(ScrollTrigger);

gsap.to(".box", { x: 200 });
```

## 자주 사용하는 플러그인 목록

GSAP에는 다양한 플러그인이 있습니다. 필요에 따라 선택해서 사용하세요:

### 무료 플러그인

```javascript
import { ScrollTrigger } from "gsap/ScrollTrigger";  // 스크롤 애니메이션
import { Draggable } from "gsap/Draggable";          // 드래그 앤 드롭
import { MotionPathPlugin } from "gsap/MotionPathPlugin";  // 경로 따라 이동
import { TextPlugin } from "gsap/TextPlugin";        // 텍스트 애니메이션
import { ScrollToPlugin } from "gsap/ScrollToPlugin";  // 스크롤 이동
```

### 유료 플러그인 (Club GSAP 회원 전용)

```javascript
import { DrawSVGPlugin } from "gsap/DrawSVGPlugin";  // SVG 선 그리기
import { MorphSVGPlugin } from "gsap/MorphSVGPlugin";  // SVG 모핑
import { SplitText } from "gsap/SplitText";          // 텍스트 분할
import { GSDevTools } from "gsap/GSDevTools";        // 개발 도구
```

**플러그인 사용 예제:**

```javascript
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";
import { TextPlugin } from "gsap/TextPlugin";

// 플러그인 등록
gsap.registerPlugin(ScrollTrigger, TextPlugin);

// ScrollTrigger 사용
gsap.to(".box", {
  scrollTrigger: {
    trigger: ".box",
    start: "top center",
    toggleActions: "play none none reverse"
  },
  x: 200
});

// TextPlugin 사용
gsap.to(".text", {
  duration: 2,
  text: "새로운 텍스트입니다!",
  ease: "none"
});
```

## 자주 묻는 질문 (FAQ)

### Q1. 플러그인을 매번 registerPlugin 해야 하나요?

**A:** 네, 각 JavaScript 파일에서 플러그인을 사용하려면 그 파일 안에서 한 번은 등록해야 합니다.

```javascript
// ComponentA.js
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";
gsap.registerPlugin(ScrollTrigger);  // 이 파일에서 사용하려면 등록 필요

// ComponentB.js
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";
gsap.registerPlugin(ScrollTrigger);  // 이 파일에서도 등록 필요
```

보통 메인 파일이나 전역 설정 파일에서 한 번 등록하면 전체 앱에서 사용 가능합니다.

### Q2. 같은 플러그인을 여러 번 등록하면 문제가 생기나요?

**A:** 아니요, 문제없습니다! GSAP은 똑똑하게 동작해서 이미 등록된 플러그인은 무시합니다.

```javascript
gsap.registerPlugin(ScrollTrigger);
gsap.registerPlugin(ScrollTrigger);  // 중복 등록이지만 문제없음
gsap.registerPlugin(ScrollTrigger);  // 이것도 괜찮음
```

### Q3. TypeScript 에러가 발생해요!

**A:** GSAP은 TypeScript를 완벽하게 지원합니다. 에러가 발생하면 다음을 확인하세요:

```typescript
// ✅ 올바른 import 방법
import { gsap } from "gsap";
import { ScrollTrigger } from "gsap/ScrollTrigger";

// 타입도 함께 import 가능
import type { GSAPTimeline } from "gsap";

gsap.registerPlugin(ScrollTrigger);

const tl: GSAPTimeline = gsap.timeline();
```

만약 `Cannot find module 'gsap'` 에러가 발생하면:

```bash
# 타입 정의가 포함된 최신 버전 재설치
npm install gsap@latest
```

### Q4. 웹팩(Webpack)이나 Create React App에서 빌드가 실패해요!

**A:** 대부분 플러그인 경로 문제입니다. 다음과 같이 정확한 경로를 사용하세요:

```javascript
// ✅ 올바른 방법
import { ScrollTrigger } from "gsap/ScrollTrigger";

// ❌ 잘못된 방법
import ScrollTrigger from "gsap/dist/ScrollTrigger";
```

### Q5. 옛날 버전의 GSAP(GSAP 2)을 사용할 수 있나요?

**A:** 가능하지만 권장하지 않습니다. GSAP 3은 더 작고, 빠르고, 사용하기 쉽습니다.

```bash
# GSAP 3 (권장)
npm install gsap

# GSAP 2 (구버전, 권장하지 않음)
npm install gsap@2
```

GSAP 2에서 3으로 업그레이드하는 것은 비교적 간단합니다. 대부분의 코드가 호환됩니다.
