# GSAP이란 무엇인가요?

**GSAP**(GreenSock Animation Platform)은 웹페이지에서 부드럽고 멋진 애니메이션을 만들 수 있게 도와주는 JavaScript 라이브러리입니다. 

"라이브러리"란 자주 사용하는 기능들을 미리 만들어 놓은 코드 모음집이라고 생각하시면 됩니다. 마치 레고 블록처럼, GSAP이 제공하는 코드 조각들을 조립해서 원하는 애니메이션을 쉽게 만들 수 있습니다.

2006년부터 개발되어 온 GSAP은 현재 전 세계 1,100만 개 이상의 웹사이트에서 사용되고 있으며, Google, Nike, Netflix 같은 세계적인 기업들도 GSAP을 신뢰하고 사용하고 있습니다.

## 애니메이션이 왜 필요한가요?

웹사이트에 애니메이션을 추가하면 다음과 같은 장점이 있습니다:

- **사용자의 시선을 끌 수 있습니다**: 움직이는 요소는 자연스럽게 눈길을 사로잡습니다
- **정보를 더 효과적으로 전달합니다**: 단계적으로 나타나는 정보는 이해하기 쉽습니다
- **전문적인 느낌을 줍니다**: 부드러운 애니메이션은 사이트를 더 세련되게 만듭니다
- **사용자 경험을 향상시킵니다**: 적절한 애니메이션은 사이트를 사용하기 더 즐겁게 만듭니다

## 왜 GSAP을 사용해야 할까요?

### 1. 초보자도 쉽게 시작할 수 있습니다

GSAP의 가장 큰 장점은 간단한 문법입니다. 복잡한 수학이나 어려운 개념 없이도 멋진 애니메이션을 만들 수 있습니다.

```javascript
// 단 한 줄로 애니메이션을 만들 수 있습니다
gsap.to(".box", { x: 100, duration: 1 });
```

위 코드를 한 줄씩 자세히 살펴볼까요?

- `gsap.to()`: "GSAP아, 이 요소를 ~로 이동시켜줘" 라는 명령입니다
- `".box"`: 애니메이션을 적용할 HTML 요소를 선택합니다 (CSS 선택자 문법과 동일)
- `{ x: 100 }`: x축(가로 방향)으로 100픽셀 이동하라는 의미입니다
- `duration: 1`: 애니메이션이 1초 동안 진행됩니다

이 코드만으로 `.box` 클래스를 가진 요소가 오른쪽으로 100픽셀 부드럽게 이동하는 애니메이션이 완성됩니다!

### 2. 뛰어난 성능 (빠르고 부드럽게 움직입니다)

GSAP은 CSS 애니메이션보다 최대 20배 빠른 성능을 제공합니다. 

**성능이 중요한 이유:**
- **60fps(frames per second)**: 1초에 60번 화면을 갱신하면 사람 눈에 매우 부드럽게 보입니다
- **GPU 가속**: 컴퓨터의 그래픽 카드를 활용하여 더 빠르게 처리합니다
- **모바일 최적화**: 성능이 낮은 스마트폰에서도 부드럽게 동작합니다

GSAP은 이 모든 것을 자동으로 처리해주기 때문에, 개발자는 성능 걱정 없이 창의적인 애니메이션에만 집중할 수 있습니다.

### 3. 모든 브라우저에서 동일하게 작동합니다

Chrome, Safari, Firefox, Edge는 물론이고 옛날 브라우저인 Internet Explorer에서도 작동합니다. GSAP이 브라우저별 차이를 알아서 처리해주기 때문에, 한 번 작성한 코드가 모든 환경에서 동일하게 동작합니다.

### 4. 다양한 플러그인으로 확장 가능합니다

GSAP은 기본 기능 외에도 특별한 기능을 추가할 수 있는 플러그인들을 제공합니다:

- **ScrollTrigger**: 스크롤할 때 애니메이션 실행 (스크롤하면 요소가 나타나는 효과 등)
- **Draggable**: 마우스로 요소를 드래그할 수 있게 만들기
- **MorphSVG**: 도형이 다른 도형으로 변하는 애니메이션
- **MotionPath**: 요소가 곡선 경로를 따라 움직이는 애니메이션

## 가장 기본적인 예제부터 시작해볼까요?

### 첫 번째 예제: 요소를 오른쪽으로 이동시키기

```javascript
// .box 클래스를 가진 요소를 오른쪽으로 200픽셀 이동
gsap.to(".box", { 
  x: 200,           // x축(가로)으로 200픽셀 이동
  duration: 2       // 2초 동안 천천히 이동
});
```

**코드 설명:**
- `gsap.to()`: "현재 위치에서 → 지정한 위치로" 애니메이션합니다
- `.box`: 움직일 요소의 CSS 클래스 이름
- `x: 200`: 오른쪽으로 200픽셀 (음수를 쓰면 왼쪽으로 이동)
- `duration: 2`: 애니메이션 지속 시간 (2초)

### 두 번째 예제: 여러 속성을 동시에 변경하기

```javascript
gsap.to(".box", { 
  x: 200,              // 오른쪽으로 200픽셀 이동
  y: 100,              // 아래로 100픽셀 이동
  rotation: 360,       // 360도 회전 (한 바퀴 회전)
  scale: 1.5,          // 1.5배 확대
  opacity: 0.5,        // 투명도 50%로 설정
  duration: 2          // 모든 변화가 2초 동안 진행
});
```

**새로운 속성들 설명:**
- `y`: 세로 방향 이동 (양수는 아래, 음수는 위)
- `rotation`: 회전 각도 (360도 = 한 바퀴)
- `scale`: 크기 조절 (1이 원본 크기, 1.5는 1.5배 확대, 0.5는 절반 축소)
- `opacity`: 투명도 (0은 완전 투명, 1은 불투명, 0.5는 반투명)

이 모든 변화가 동시에 부드럽게 일어납니다!

### 세 번째 예제: 애니메이션 방향 이해하기

GSAP에는 세 가지 기본 메서드가 있습니다:

```javascript
// 1. to(): 현재 상태 → 목표 상태
gsap.to(".box", { 
  x: 300,
  duration: 1 
});
// "현재 위치에서 x:300 위치로 이동해"

// 2. from(): 시작 상태 → 현재 상태
gsap.from(".box", { 
  x: -300,
  duration: 1 
});
// "x:-300 위치에서 시작해서 원래 위치로 이동해"

// 3. fromTo(): 시작 상태 → 끝 상태 (둘 다 명시)
gsap.fromTo(".box", 
  { x: -300 },          // 시작 위치
  { x: 300, duration: 1 }  // 끝 위치
);
// "x:-300에서 시작해서 x:300으로 이동해"
```

**언제 무엇을 사용할까요?**
- `to()`: 요소를 어디로 이동시킬지만 정하면 될 때 (가장 자주 사용)
- `from()`: 요소가 어디서 나타나는지 보여주고 싶을 때
- `fromTo()`: 시작과 끝을 정확하게 제어하고 싶을 때

## Timeline: 애니메이션을 순서대로 실행하기

Timeline은 여러 애니메이션을 마치 영화 장면처럼 순서대로 배치할 수 있게 해줍니다.

```javascript
// 1. Timeline 생성하기
const tl = gsap.timeline();

// 2. 애니메이션을 순서대로 추가하기
tl.to(".box1", { x: 100, duration: 1 })      // 첫 번째: box1 이동
  .to(".box2", { y: 100, duration: 1 })      // 두 번째: box2 이동
  .to(".box3", { rotation: 360, duration: 1 }); // 세 번째: box3 회전
```

**Timeline의 동작 방식:**
1. `.box1`이 오른쪽으로 이동합니다 (1초 동안)
2. `box1`의 애니메이션이 끝나면, `.box2`가 아래로 이동합니다 (1초 동안)
3. `box2`의 애니메이션이 끝나면, `.box3`이 회전합니다 (1초 동안)

**동시에 실행하고 싶다면?**

```javascript
const tl = gsap.timeline();

tl.to(".box1", { x: 100, duration: 1 })
  .to(".box2", { y: 100, duration: 1 }, "<")      // "<"는 "이전 애니메이션과 동시에"
  .to(".box3", { rotation: 360, duration: 1 }, "-=0.5"); // 이전 애니메이션이 끝나기 0.5초 전에 시작
```

**위치 제어 파라미터:**
- `"<"`: 이전 애니메이션 시작 시점과 동시에
- `">"`: 이전 애니메이션이 끝나고 (기본값)
- `"-=0.5"`: 이전 애니메이션이 끝나기 0.5초 전에
- `"+=0.5"`: 이전 애니메이션이 끝나고 0.5초 후에
- `0.5`: 타임라인 시작 후 0.5초 시점에

## Easing: 애니메이션에 생동감 더하기

Easing은 애니메이션의 "속도 변화"를 조절합니다. 현실 세계의 물체는 일정한 속도로 움직이지 않습니다. 가속했다가 감속하는 것이 자연스럽죠.

```javascript
// 기본 (linear): 일정한 속도
gsap.to(".box", { 
  x: 300, 
  duration: 2,
  ease: "none"  // 일정한 속도로 (로봇 같은 느낌)
});

// power1: 부드럽게 시작, 빠르게 끝 (자연스러운 느낌)
gsap.to(".box", { 
  x: 300, 
  duration: 2,
  ease: "power1.out"  
});

// elastic: 탄성 효과 (통통 튀는 느낌)
gsap.to(".box", { 
  x: 300, 
  duration: 2,
  ease: "elastic.out(1, 0.3)"  
});

// back: 살짝 뒤로 갔다가 앞으로 (당겼다 놓는 느낌)
gsap.to(".box", { 
  x: 300, 
  duration: 2,
  ease: "back.out(1.7)"  
});

// bounce: 바운스 효과 (공이 떨어지는 느낌)
gsap.to(".box", { 
  x: 300, 
  duration: 2,
  ease: "bounce.out"  
});
```

**Easing 타입 이해하기:**
- `.in`: 천천히 시작 → 빠르게 끝
- `.out`: 빠르게 시작 → 천천히 끝 (가장 자연스럽고 많이 사용)
- `.inOut`: 천천히 시작 → 빠르게 중간 → 천천히 끝

**자주 사용하는 Easing:**
- `"power1.out"`: 부드럽고 자연스러움 (대부분의 경우 사용)
- `"power2.out"`: power1보다 조금 더 역동적
- `"elastic.out"`: 재미있고 통통한 느낌
- `"back.out"`: 살짝 오버하는 느낌 (눈에 띄게 하고 싶을 때)

## 유용한 콜백 함수들

콜백 함수는 "애니메이션의 특정 시점에 실행되는 함수"입니다.

```javascript
gsap.to(".box", {
  x: 300,
  duration: 2,
  
  // 애니메이션 시작할 때 한 번 실행
  onStart: () => {
    console.log("애니메이션 시작!");
  },
  
  // 애니메이션 진행 중 계속 실행 (매 프레임마다)
  onUpdate: () => {
    console.log("애니메이션 진행 중...");
  },
  
  // 애니메이션이 완전히 끝났을 때 한 번 실행
  onComplete: () => {
    console.log("애니메이션 완료!");
    // 다음 애니메이션 실행하거나
    // 다른 작업 수행
  }
});
```

**실용적인 예제: 로딩 표시**

```javascript
const loadingBar = document.querySelector(".loading-bar");

gsap.to(".loading-bar", {
  width: "100%",      // 로딩바를 100%로 채우기
  duration: 3,
  
  onUpdate: function() {
    // 현재 진행률을 텍스트로 표시
    const progress = Math.round(this.progress() * 100);
    loadingBar.textContent = progress + "%";
  },
  
  onComplete: () => {
    console.log("로딩 완료!");
    // 메인 콘텐츠 표시
  }
});
```

## GSAP 설치하고 시작하기

### 방법 1: CDN 사용하기 (가장 간단함)

HTML 파일의 `</body>` 태그 바로 위에 이 코드를 추가하세요:

```html
<!DOCTYPE html>
<html>
<head>
  <title>GSAP 테스트</title>
  <style>
    .box {
      width: 100px;
      height: 100px;
      background-color: #3498db;
      border-radius: 10px;
    }
  </style>
</head>
<body>
  
  <div class="box"></div>

  <!-- GSAP 라이브러리 불러오기 -->
  <script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js"></script>
  
  <!-- 여기에 애니메이션 코드 작성 -->
  <script>
    gsap.to(".box", {
      x: 300,
      rotation: 360,
      duration: 2
    });
  </script>

</body>
</html>
```

이 HTML 파일을 저장하고 브라우저로 열면 파란 상자가 오른쪽으로 이동하면서 회전하는 것을 볼 수 있습니다!

### 방법 2: npm 사용하기 (React, Vue 등 프로젝트)

터미널에서 다음 명령어를 실행하세요:

```bash
npm install gsap
```

그리고 JavaScript 파일에서 import해서 사용하세요:

```javascript
// JavaScript 파일 상단에 작성
import gsap from "gsap";

// 사용 예제
gsap.to(".box", { 
  x: 100, 
  duration: 1 
});
```

## GSAP의 핵심 메서드 총정리

```javascript
// 1. gsap.to() - 가장 많이 사용
// "현재 상태에서 → 이 상태로 변경"
gsap.to(".element", { 
  x: 100,        // 목표 위치
  duration: 1 
});

// 2. gsap.from() - 등장 효과에 유용
// "이 상태에서 시작 → 원래 상태로"
gsap.from(".element", { 
  opacity: 0,    // 투명한 상태에서 시작
  y: -50,        // 위쪽에서 시작
  duration: 1 
});

// 3. gsap.fromTo() - 정확한 제어가 필요할 때
// "이 상태에서 → 저 상태로"
gsap.fromTo(".element",
  { x: -100, opacity: 0 },     // 시작 상태
  { x: 100, opacity: 1, duration: 1 }  // 끝 상태
);

// 4. gsap.set() - 즉시 적용 (애니메이션 없이)
// "애니메이션 없이 바로 이 상태로 설정"
gsap.set(".element", { 
  x: 100,
  opacity: 0 
});
// duration 없이 즉시 적용됨
```

## 자주 사용하는 속성들 정리

```javascript
gsap.to(".element", {
  // 위치 관련
  x: 100,              // 가로 이동 (오른쪽 +, 왼쪽 -)
  y: 100,              // 세로 이동 (아래 +, 위 -)
  
  // 크기 관련
  scale: 1.5,          // 전체 크기 (1.5배 확대)
  scaleX: 2,           // 가로만 확대
  scaleY: 0.5,         // 세로만 축소
  width: "200px",      // 너비 변경
  height: "100px",     // 높이 변경
  
  // 회전 관련
  rotation: 360,       // 회전 (360도 = 한 바퀴)
  rotationX: 180,      // X축 기준 회전 (3D 효과)
  rotationY: 180,      // Y축 기준 회전 (3D 효과)
  
  // 투명도
  opacity: 0.5,        // 투명도 (0~1)
  
  // 색상
  backgroundColor: "#ff0000",  // 배경색
  color: "#ffffff",            // 글자색
  
  // 기타
  borderRadius: "50%", // 모서리 둥글게
  
  // 애니메이션 제어
  duration: 2,         // 지속 시간 (초)
  delay: 0.5,          // 시작 전 대기 시간 (초)
  ease: "power2.out",  // 가속도 곡선
  repeat: 2,           // 반복 횟수 (2번 더 실행)
  repeatDelay: 1,      // 반복 사이 대기 시간
  yoyo: true,          // 왕복 운동 (true면 정방향→역방향 반복)
});
```

## 실전 예제: 간단한 인트로 애니메이션 만들기

```javascript
// Timeline 생성
const tl = gsap.timeline();

// 1. 로고가 위에서 떨어지면서 등장
tl.from(".logo", {
  y: -100,             // 위에서 시작
  opacity: 0,          // 투명한 상태로 시작
  duration: 1,
  ease: "bounce.out"   // 통통 튀는 효과
})

// 2. 제목이 왼쪽에서 슬라이드
.from(".title", {
  x: -200,
  opacity: 0,
  duration: 0.8,
  ease: "power2.out"
}, "-=0.3")  // 로고 애니메이션이 끝나기 0.3초 전에 시작

// 3. 설명 텍스트가 페이드인
.from(".description", {
  opacity: 0,
  y: 20,
  duration: 0.6
}, "-=0.2")  // 제목이 끝나기 0.2초 전에 시작

// 4. 버튼이 크기와 함께 등장
.from(".button", {
  scale: 0,
  opacity: 0,
  duration: 0.5,
  ease: "back.out(1.7)"  // 살짝 오버하는 효과
});

// 전체 애니메이션이 약 2~3초 동안 부드럽게 실행됩니다!
```

- **Google**: Google Fonts, Chrome 브라우저 프로모션 사이트
- **Nike**: 신제품 런칭 페이지의 화려한 애니메이션
- **Netflix**: 인터랙티브 스토리텔링 페이지
- **Samsung**: 갤럭시 제품 소개 페이지
- **Spotify**: 연말 결산 페이지 (Spotify Wrapped)

## 다음 단계로 나아가기

이제 GSAP이 무엇인지 이해하셨나요? 다음 단계로 넘어가볼까요:

1. **기본 애니메이션 마스터하기**
   - `gsap.to()`, `gsap.from()`, `gsap.fromTo()` 완벽하게 이해하기
   - 다양한 속성(x, y, scale, rotation, opacity 등) 실험해보기
   - 여러 easing 타입 비교해보기

2. **Timeline으로 복잡한 시퀀스 만들기**
   - 여러 애니메이션을 조합해서 스토리 만들기
   - 타이밍 조절하기 (동시, 순차, 겹치기)
   - Timeline 제어하기 (재생, 정지, 되감기)

3. **ScrollTrigger로 스크롤 애니메이션**
   - 스크롤하면 요소가 나타나는 효과
   - 패럴랙스 스크롤 효과
   - 스크롤 진행도에 따른 애니메이션

4. **고급 테크닉**
   - Stagger: 여러 요소를 순차적으로 애니메이션
   - 동적 값 사용하기
   - 반응형 애니메이션 만들기
