# TypeScript 설치하기

이 가이드에서는 TypeScript를 컴퓨터에 설치하는 전체 과정을 처음부터 끝까지 자세히 알려드립니다. 각 단계를 따라하시면 누구나 쉽게 TypeScript를 설치하고 사용할 수 있습니다.

## 설치 전 준비사항

TypeScript를 설치하기 전에 먼저 **Node.js**를 설치해야 합니다.

### Node.js가 무엇인가요?

**Node.js**는 JavaScript를 컴퓨터에서 실행할 수 있게 해주는 프로그램입니다. 원래 JavaScript는 브라우저에서만 실행되었지만, Node.js 덕분에 브라우저 없이도 JavaScript를 실행할 수 있게 되었습니다.

TypeScript는 Node.js의 **npm(Node Package Manager)**이라는 도구를 통해 설치됩니다. npm은 "Node.js 패키지 관리자"라는 뜻으로, 다양한 JavaScript 도구들을 쉽게 설치하고 관리할 수 있게 해주는 프로그램입니다.

### Node.js 설치 여부 확인하기

컴퓨터에 Node.js가 이미 설치되어 있는지 확인해봅시다.

**Windows 사용자:**
1. 시작 메뉴에서 "cmd" 또는 "명령 프롬프트"를 검색합니다
2. 명령 프롬프트를 실행합니다
3. 다음 명령어를 입력하고 Enter를 누릅니다:

```bash
node --version
```

**Mac 사용자:**
1. Spotlight 검색(⌘ + Space)에서 "터미널" 또는 "terminal"을 검색합니다
2. 터미널을 실행합니다
3. 다음 명령어를 입력하고 Enter를 누릅니다:

```bash
node --version
```

명령어를 입력했을 때:
- `v20.11.0` 같은 버전 번호가 나타나면 → Node.js가 이미 설치되어 있습니다 ✅
- `'node'은(는) 내부 또는 외부 명령...`이라는 에러가 나타나면 → Node.js를 설치해야 합니다 ❌

### Node.js 설치하기

Node.js가 설치되어 있지 않다면, 다음 단계를 따라 설치하세요:

**1단계: Node.js 다운로드**

1. 브라우저에서 [https://nodejs.org](https://nodejs.org) 접속합니다
2. 메인 페이지에 두 개의 다운로드 버튼이 보입니다
   - **LTS (Long Term Support)** - 추천! 안정적인 버전입니다
   - **Current** - 최신 기능이 포함된 버전입니다
3. **LTS** 버전을 다운로드합니다 (대부분의 사용자에게 적합합니다)

**2단계: Node.js 설치**

1. 다운로드한 설치 파일을 실행합니다
2. 설치 마법사가 나타나면 "Next"를 계속 클릭합니다
3. 라이선스 동의에 체크하고 "Next"를 클릭합니다
4. 설치 경로는 기본값을 사용하는 것을 권장합니다
5. "Install"을 클릭합니다
6. 설치가 완료되면 "Finish"를 클릭합니다

**3단계: 설치 확인**

명령 프롬프트(Windows) 또는 터미널(Mac)을 **다시 열고** 다음 명령어를 입력합니다:

```bash
node --version
```

그리고 npm도 확인합니다:

```bash
npm --version
```

두 명령어 모두 버전 번호가 나타나면 성공입니다!

:::tip 중요!
Node.js를 설치한 후에는 **반드시 명령 프롬프트나 터미널을 다시 열어야** 합니다. 이미 열려있던 창에서는 새로 설치된 프로그램을 인식하지 못할 수 있습니다.
:::

## TypeScript 설치하기

이제 본격적으로 TypeScript를 설치해봅시다. TypeScript를 설치하는 방법은 크게 두 가지가 있습니다:

1. **전역 설치 (Global Installation)** - 컴퓨터 전체에서 사용
2. **로컬 설치 (Local Installation)** - 특정 프로젝트에서만 사용

처음 배우는 분들은 먼저 **전역 설치**를 추천합니다. 이후 실제 프로젝트를 진행할 때는 **로컬 설치**를 사용하는 것이 좋습니다.

### 방법 1: 전역 설치 (초보자 추천)

전역 설치는 컴퓨터의 모든 폴더에서 TypeScript를 사용할 수 있게 해줍니다.

**1단계: 명령 프롬프트/터미널 열기**

- Windows: 시작 메뉴에서 "cmd" 검색
- Mac: Spotlight에서 "터미널" 검색

**2단계: 설치 명령어 입력**

다음 명령어를 입력하고 Enter를 누릅니다:

```bash
npm install -g typescript
```

명령어를 하나씩 이해해봅시다:

- `npm` - Node Package Manager를 실행합니다
- `install` - 무언가를 설치하겠다는 명령어입니다
- `-g` - "global"의 약자로, 전역으로 설치한다는 의미입니다
- `typescript` - 설치할 패키지의 이름입니다

**3단계: 설치 진행 과정 확인**

명령어를 입력하면 다음과 같은 화면이 나타납니다:

```
npm WARN deprecated ...
added 1 package in 3s
```

몇 가지 경고 메시지가 나올 수 있지만, 대부분 문제없습니다. 마지막에 `added 1 package`라는 메시지가 나오면 설치가 완료된 것입니다.

:::note 설치 시간
인터넷 속도에 따라 10초에서 1분 정도 소요될 수 있습니다. 설치가 진행되는 동안 기다려주세요!
:::

**4단계: 설치 확인**

TypeScript가 제대로 설치되었는지 확인해봅시다:

```bash
tsc --version
```

또는 짧게:

```bash
tsc -v
```

명령어 설명:
- `tsc` - TypeScript Compiler의 약자입니다
- `--version` 또는 `-v` - 설치된 버전을 보여달라는 명령어입니다

`Version 5.7.2` 같은 버전 번호가 나타나면 성공입니다! 🎉

:::tip 에러가 발생한다면?
만약 `'tsc'은(는) 내부 또는 외부 명령...` 에러가 나타난다면:
1. 명령 프롬프트/터미널을 완전히 닫았다가 다시 열어보세요
2. 그래도 안 되면 컴퓨터를 재시작해보세요
3. 여전히 문제가 있다면 설치 과정을 처음부터 다시 시도해보세요
:::

### 방법 2: 로컬 설치 (프로젝트용)

로컬 설치는 특정 프로젝트 폴더 안에만 TypeScript를 설치하는 방법입니다. 이 방법의 장점은:

- 프로젝트마다 다른 TypeScript 버전을 사용할 수 있습니다
- 팀원들이 모두 같은 버전을 사용하게 됩니다
- 프로젝트를 다른 컴퓨터로 옮겨도 동일하게 작동합니다

**1단계: 프로젝트 폴더 만들기**

먼저 프로젝트를 위한 폴더를 만듭니다:

```bash
mkdir my-typescript-project
cd my-typescript-project
```

명령어 설명:
- `mkdir my-typescript-project` - "my-typescript-project"라는 이름의 폴더를 만듭니다
  - `mkdir`은 "make directory"의 약자입니다
- `cd my-typescript-project` - 만든 폴더로 이동합니다
  - `cd`는 "change directory"의 약자입니다

:::tip 폴더 이름
폴더 이름은 원하는 대로 지정할 수 있습니다. 단, 공백 대신 하이픈(-)이나 언더스코어(_)를 사용하는 것이 좋습니다.
:::

**2단계: npm 프로젝트 초기화**

프로젝트 폴더에서 npm을 초기화합니다:

```bash
npm init -y
```

명령어 설명:
- `npm init` - 새로운 npm 프로젝트를 시작합니다
- `-y` - 모든 질문에 자동으로 "yes"라고 답합니다 (빠른 초기화)

이 명령어를 실행하면 `package.json` 파일이 생성됩니다. 이 파일은 프로젝트의 정보와 사용하는 패키지 목록을 담고 있는 일종의 "프로젝트 설명서"입니다.

**3단계: TypeScript를 개발 의존성으로 설치**

```bash
npm install --save-dev typescript
```

또는 짧게:

```bash
npm i -D typescript
```

명령어 설명:
- `npm install` 또는 `npm i` - 패키지를 설치합니다
- `--save-dev` 또는 `-D` - 개발할 때만 필요한 패키지로 저장합니다
  - "개발 의존성"이라고 부르며, 실제 서비스 운영시에는 필요 없고 개발 중에만 필요한 도구입니다
- `typescript` - 설치할 패키지 이름입니다

**4단계: 설치 확인**

프로젝트에 TypeScript가 설치되었는지 확인합니다:

```bash
npx tsc --version
```

명령어 설명:
- `npx` - 프로젝트 로컬에 설치된 패키지를 실행하는 명령어입니다
- 전역 설치와 달리 `npx`를 앞에 붙여야 합니다

버전 번호가 나타나면 성공입니다!

:::note 전역 설치 vs 로컬 설치
- 전역 설치: `tsc` 명령어로 바로 사용
- 로컬 설치: `npx tsc` 명령어로 사용 (npx를 앞에 붙임)
:::

## TypeScript 설정 파일 만들기

TypeScript를 제대로 사용하려면 설정 파일이 필요합니다. 이 파일은 TypeScript 컴파일러에게 "이렇게 작동해주세요"라고 알려주는 역할을 합니다.

### tsconfig.json 파일 생성하기

프로젝트 폴더에서 다음 명령어를 실행합니다:

```bash
tsc --init
```

로컬 설치를 했다면:

```bash
npx tsc --init
```

이 명령어를 실행하면:

```
Created a new tsconfig.json with:
  target: ES2016
  module: commonjs
  strict: true
  esModuleInterop: true
  skipLibCheck: true
  forceConsistentCasingInFileNames: true
```

이런 메시지와 함께 `tsconfig.json` 파일이 생성됩니다.

### tsconfig.json 파일 이해하기

생성된 파일을 메모장이나 에디터로 열어보면 매우 많은 내용이 들어있습니다. 처음에는 대부분 주석(//로 시작하는 줄)으로 되어 있고, 실제 설정은 몇 줄만 활성화되어 있습니다.

초보자가 꼭 알아야 할 중요한 설정들만 살펴보겠습니다:

```json
{
  "compilerOptions": {
    /* 기본 옵션 */
    "target": "ES2016",                    // 어떤 JavaScript 버전으로 변환할지
    "module": "commonjs",                  // 모듈 시스템 방식
    
    /* 엄격한 타입 체크 */
    "strict": true,                        // 엄격 모드 활성화
    
    /* 프로젝트 구조 */
    "rootDir": "./src",                    // TypeScript 파일이 있는 폴더
    "outDir": "./dist",                    // 컴파일된 JavaScript 파일을 저장할 폴더
    
    /* 기타 */
    "esModuleInterop": true,               // CommonJS와 ES Module 호환성
    "forceConsistentCasingInFileNames": true,  // 파일 이름 대소문자 일관성 체크
    "skipLibCheck": true                   // 라이브러리 타입 체크 건너뛰기
  }
}
```

각 옵션을 자세히 알아봅시다:

#### target (대상 JavaScript 버전)

TypeScript 코드를 어떤 버전의 JavaScript로 변환할지 결정합니다.

- `"ES2016"` (또는 `"ES7"`) - 2016년 JavaScript 표준
- `"ES2020"` - 2020년 JavaScript 표준 (최신 브라우저용)
- `"ES5"` - 구형 브라우저 지원 (Internet Explorer 등)

**어떤 걸 선택해야 하나요?**
- 최신 브라우저만 지원한다면: `"ES2020"` 이상
- 구형 브라우저도 지원해야 한다면: `"ES5"`
- 잘 모르겠다면: 기본값인 `"ES2016"` 사용

#### module (모듈 시스템)

JavaScript 파일들을 어떻게 import/export 할지 결정합니다.

- `"commonjs"` - Node.js에서 사용하는 방식
- `"esnext"` - 최신 ES Module 방식

**어떤 걸 선택해야 하나요?**
- Node.js 프로젝트라면: `"commonjs"`
- 브라우저 프로젝트라면: `"esnext"`
- 잘 모르겠다면: `"commonjs"` (기본값)

#### strict (엄격 모드)

TypeScript의 타입 체크를 얼마나 엄격하게 할지 결정합니다.

- `true` - 엄격하게 체크 (추천!)
- `false` - 느슨하게 체크

처음에는 `true`가 어렵게 느껴질 수 있지만, 버그를 미리 잡아주므로 `true`를 사용하는 것을 강력히 추천합니다.

#### rootDir와 outDir (폴더 구조)

프로젝트 파일들을 어떻게 구성할지 결정합니다.

```
my-project/
├── src/              ← TypeScript 파일들 (.ts)
│   ├── index.ts
│   └── utils.ts
├── dist/             ← 컴파일된 JavaScript 파일들 (.js)
│   ├── index.js
│   └── utils.js
└── tsconfig.json
```

- `"rootDir": "./src"` - TypeScript 소스 파일은 src 폴더에 작성
- `"outDir": "./dist"` - 컴파일된 JavaScript 파일은 dist 폴더에 저장

이렇게 하면 소스 코드와 컴파일된 코드를 깔끔하게 분리할 수 있습니다.

### 초보자를 위한 추천 설정

처음 시작하는 분들을 위한 간단한 `tsconfig.json` 설정입니다:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist",
    "rootDir": "./src"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}
```

추가 옵션 설명:
- `"include": ["src/**/*"]` - src 폴더 안의 모든 TypeScript 파일을 컴파일합니다
  - `**/*`는 "모든 하위 폴더의 모든 파일"을 의미합니다
- `"exclude": ["node_modules"]` - node_modules 폴더는 컴파일에서 제외합니다

## 첫 번째 TypeScript 파일 만들기

이제 실제로 TypeScript 파일을 만들어서 컴파일해봅시다!

### 1단계: 프로젝트 구조 만들기

프로젝트 폴더에서:

```bash
mkdir src
```

이제 폴더 구조는 이렇게 됩니다:

```
my-typescript-project/
├── src/              ← 여기에 TypeScript 파일 작성
├── package.json
└── tsconfig.json
```

### 2단계: TypeScript 파일 작성하기

`src` 폴더 안에 `hello.ts` 파일을 만들고 다음 코드를 작성합니다:

```typescript
// hello.ts
function greet(name: string): string {
  return `안녕하세요, ${name}님!`;
}

const message = greet("TypeScript");
console.log(message);

// 타입 에러 테스트 (주석을 풀어보세요!)
// const wrongMessage = greet(123);  // 에러 발생!
```

코드 설명:
- `function greet(name: string): string` - greet 함수를 정의합니다
  - `name: string` - name 매개변수는 문자열이어야 합니다
  - `: string` - 함수는 문자열을 반환합니다
- 백틱(`)을 사용한 템플릿 리터럴로 문자열을 만듭니다
- `greet(123)` - 숫자를 전달하면 TypeScript가 에러를 발생시킵니다!

### 3단계: TypeScript 컴파일하기

이제 TypeScript 파일을 JavaScript로 변환(컴파일)해봅시다.

전역 설치를 했다면:

```bash
tsc
```

로컬 설치를 했다면:

```bash
npx tsc
```

명령어를 실행하면 아무 메시지도 나타나지 않을 수 있습니다. 이것은 정상입니다! 에러가 없다면 조용히 컴파일됩니다.

이제 폴더 구조를 확인해보면:

```
my-typescript-project/
├── src/
│   └── hello.ts      ← TypeScript 원본 파일
├── dist/
│   └── hello.js      ← 컴파일된 JavaScript 파일 (새로 생성됨!)
├── package.json
└── tsconfig.json
```

### 4단계: 컴파일된 JavaScript 실행하기

생성된 JavaScript 파일을 실행해봅시다:

```bash
node dist/hello.js
```

결과:

```
안녕하세요, TypeScript님!
```

성공입니다! 🎉

### 5단계: Watch 모드로 자동 컴파일하기

매번 파일을 수정할 때마다 `tsc`를 입력하는 것은 번거롭습니다. **Watch 모드**를 사용하면 파일을 저장할 때마다 자동으로 컴파일됩니다.

```bash
tsc --watch
```

또는 짧게:

```bash
tsc -w
```

로컬 설치는:

```bash
npx tsc -w
```

이 명령어를 실행하면:

```
[오후 3:24:00] Starting compilation in watch mode...
[오후 3:24:01] Found 0 errors. Watching for file changes.
```

이제 `hello.ts` 파일을 수정하고 저장하면 자동으로 컴파일됩니다!

Watch 모드를 종료하려면 `Ctrl + C`를 누르세요.

:::tip 개발할 때 유용한 팁
개발할 때는 Watch 모드를 켜놓고 코드를 작성하세요. 파일을 저장할 때마다 자동으로 컴파일되어 매우 편리합니다!
:::

## 자주 묻는 질문 (FAQ)

### Q: Node.js와 npm은 왜 필요한가요?

A: TypeScript는 npm을 통해 설치되고, Node.js 환경에서 실행되기 때문입니다. TypeScript 컴파일러 자체가 JavaScript로 작성되어 있어 Node.js가 필요합니다.

### Q: 전역 설치와 로컬 설치 중 어떤 것을 사용해야 하나요?

A: 처음 배울 때는 전역 설치가 편리하지만, 실제 프로젝트를 진행할 때는 로컬 설치를 사용하는 것이 좋습니다. 로컬 설치는 프로젝트마다 다른 TypeScript 버전을 사용할 수 있고, 팀 협업 시 버전 충돌을 방지할 수 있습니다.

### Q: tsconfig.json 파일은 반드시 필요한가요?

A: 간단한 테스트라면 없어도 되지만, 실제 프로젝트에서는 반드시 필요합니다. 컴파일 옵션, 파일 경로, 타입 체크 수준 등을 설정할 수 있어 프로젝트 관리가 훨씬 쉬워집니다.

### Q: .ts 파일과 .js 파일의 차이는 무엇인가요?

A: `.ts`는 TypeScript 파일이고, `.js`는 JavaScript 파일입니다. TypeScript 파일(`.ts`)은 타입 정보가 포함되어 있으며, 컴파일 과정을 거쳐 일반 JavaScript 파일(`.js`)로 변환됩니다.

### Q: 컴파일 에러가 발생하면 어떻게 하나요?

A: 에러 메시지를 천천히 읽어보세요. TypeScript의 에러 메시지는 매우 친절하게 무엇이 문제인지 알려줍니다. 에러가 발생한 파일 이름, 줄 번호, 그리고 문제의 원인을 명확히 보여줍니다.

### Q: TypeScript Playground가 무엇인가요?

A: [https://www.typescriptlang.org/play](https://www.typescriptlang.org/play)에서 제공하는 온라인 에디터입니다. 설치 없이 브라우저에서 바로 TypeScript 코드를 작성하고 테스트할 수 있어 매우 편리합니다.
