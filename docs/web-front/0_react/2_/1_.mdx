# React 빠른 시작

React 문서에 오신 것을 환영합니다! 이 페이지에서는 일상적으로 사용하게 될 React 개념의 80%를 소개합니다.

:::info React란?
React는 Facebook(현 Meta)에서 만든 사용자 인터페이스(UI)를 만들기 위한 JavaScript 라이브러리입니다. 웹 페이지를 작은 조각(컴포넌트)으로 나누어 관리하고, 데이터가 변경되면 자동으로 화면을 업데이트해줍니다.
:::

## 학습 내용

이 문서에서 배울 내용:

- **컴포넌트 생성 및 중첩하기** - UI를 작은 조각으로 나누는 방법
- **마크업과 스타일 추가하기** - 화면에 보이는 요소와 디자인 만들기
- **데이터 표시하기** - JavaScript 변수를 화면에 보여주는 방법
- **조건부 렌더링과 리스트 렌더링** - 상황에 따라 다르게 보여주기
- **이벤트에 응답하고 화면 업데이트하기** - 클릭 등의 사용자 동작 처리
- **컴포넌트 간 데이터 공유하기** - 여러 컴포넌트가 데이터를 주고받는 방법

## 컴포넌트 생성 및 중첩하기

### 컴포넌트란?

React 앱은 **컴포넌트(Component)**로 구성됩니다. 컴포넌트는 레고 블록처럼 UI를 구성하는 하나의 조각입니다.

- **컴포넌트**는 자체 로직(동작)과 외형(모양)을 가진 UI(사용자 인터페이스)의 일부입니다
- 컴포넌트는 버튼처럼 **작을 수도** 있고, 전체 페이지만큼 **클 수도** 있습니다
- 컴포넌트를 조합해서 더 복잡한 화면을 만들 수 있습니다

### 첫 번째 컴포넌트 만들기

React 컴포넌트는 **마크업(HTML 같은 것)을 반환하는 JavaScript 함수**입니다:

```jsx
function MyButton() {  // 1. 함수 이름은 대문자로 시작 (MyButton)
  return (             // 2. return 키워드로 JSX를 반환
    <button>I'm a button</button>  // 3. HTML처럼 보이는 JSX 코드
  );
}
```

**코드 한 줄씩 설명:**
1. `function MyButton()` - "MyButton"이라는 이름의 함수를 만듭니다 (반드시 대문자로 시작!)
2. `return` - 이 함수가 무엇을 돌려줄지 정의합니다
3. `<button>I'm a button</button>` - 실제 화면에 보일 버튼 요소입니다

### 컴포넌트 중첩하기

이제 `MyButton`을 선언했으니, 다른 컴포넌트 안에 **중첩(nesting)**할 수 있습니다:

```jsx
export default function MyApp() {  // 1. 메인 앱 컴포넌트
  return (                         // 2. 여러 요소를 반환
    <div>                          // 3. div로 감싸기 (필수!)
      <h1>Welcome to my app</h1>   // 4. 제목
      <MyButton />                 // 5. 위에서 만든 MyButton 컴포넌트 사용
    </div>
  );
}
```

**코드 한 줄씩 설명:**
1. `export default function MyApp()` 
   - `export default`: 이 파일의 "주인공" 컴포넌트를 지정 (다른 파일에서 가져다 쓸 수 있음)
   - `function MyApp()`: MyApp이라는 컴포넌트 함수
2. `return` - 이 컴포넌트가 화면에 보여줄 내용
3. `<div>` - 여러 요소를 묶는 컨테이너 (React는 하나의 부모 요소만 반환 가능)
4. `<h1>Welcome to my app</h1>` - 일반 HTML 태그 (소문자)
5. `<MyButton />` - 우리가 만든 React 컴포넌트 (대문자, 자기 자신을 닫는 태그)

:::tip 컴포넌트 네이밍 규칙 (매우 중요!)
- **React 컴포넌트**: `<MyButton />` - 대문자로 시작 ✅
- **HTML 태그**: `<button>`, `<div>` - 소문자 ✅
- **잘못된 예**: `<myButton />` - React가 HTML 태그로 인식해버림 ❌

이렇게 구분해야 React가 "이게 내가 만든 컴포넌트인지, 일반 HTML 태그인지" 알 수 있습니다!
:::

### export default란?

```jsx
export default function MyApp() { ... }
```

- `export` - 이 함수를 다른 파일에서도 쓸 수 있게 내보냅니다
- `default` - 이 파일의 "기본(메인)" 컴포넌트라는 뜻
- 한 파일에 여러 컴포넌트가 있어도, `export default`는 하나만 가능합니다

**실제 사용 예시:**
```jsx
// MyApp.jsx 파일
export default function MyApp() { ... }

// 다른 파일에서
import MyApp from './MyApp';  // default로 export된 것을 가져옴
```

## JSX로 마크업 작성하기

### JSX란 무엇인가요?

위에서 본 마크업 문법을 **JSX (JavaScript XML)**라고 합니다.

```jsx
<button>I'm a button</button>  // 이게 JSX입니다!
```

- **JSX는 JavaScript 안에서 HTML을 쓸 수 있게 해주는 문법**입니다
- HTML처럼 보이지만, 실제로는 JavaScript 코드입니다
- React에서 필수는 아니지만, 99%의 프로젝트가 JSX를 사용합니다 (편하거든요!)

### JSX와 HTML의 차이점

JSX는 HTML보다 **더 엄격**합니다. 몇 가지 중요한 규칙이 있습니다:

#### 1. 모든 태그는 닫아야 합니다

```jsx
// ❌ HTML에서는 가능하지만 JSX에서는 안됨
<br>
<img src="photo.jpg">

// ✅ JSX에서는 이렇게 해야 함
<br />
<img src="photo.jpg" />
```

#### 2. 반드시 하나의 부모 요소만 반환

```jsx
// ❌ 잘못된 예 - 두 개의 형제 요소를 바로 반환
function AboutPage() {
  return (
    <h1>About</h1>
    <p>Hello there.</p>  // 에러 발생!
  );
}

// ✅ 방법 1: <div>로 감싸기
function AboutPage() {
  return (
    <div>              {/* 부모 div로 감쌌습니다 */}
      <h1>About</h1>
      <p>Hello there.</p>
    </div>
  );
}

// ✅ 방법 2: Fragment <> 사용 (추천!)
function AboutPage() {
  return (
    <>                 {/* 빈 태그 = Fragment (화면에 안 보임) */}
      <h1>About</h1>
      <p>Hello there.<br />How do you do?</p>
    </>
  );
}
```

**Fragment(`<>...</>`)를 사용하는 이유:**
- 불필요한 `<div>`를 만들지 않아도 됩니다
- HTML 구조가 깔끔해집니다
- 화면에는 아무것도 렌더링되지 않고, 단지 요소들을 묶어주기만 합니다

```jsx
// <div>로 감싼 경우 실제 HTML
<div>
  <h1>About</h1>
  <p>Hello there.</p>
</div>

// <>로 감싼 경우 실제 HTML (div 없음!)
<h1>About</h1>
<p>Hello there.</p>
```

:::info HTML을 JSX로 변환하기
많은 HTML 코드를 JSX로 변환해야 한다면 [온라인 변환기](https://transform.tools/html-to-jsx)를 사용하세요. 복사-붙여넣기만 하면 자동으로 변환해줍니다!
:::

### JSX 문법 요약

| HTML | JSX | 이유 |
|------|-----|------|
| `<br>` | `<br />` | 모든 태그를 닫아야 함 |
| `<img src="">` | `<img src="" />` | 자기 자신을 닫는 태그 |
| `class="..."` | `className="..."` | class는 JavaScript 예약어라서 |
| `for="..."` | `htmlFor="..."` | for도 JavaScript 예약어 |
| 여러 요소 | `<>...</>` 또는 `<div>...</div>` | 하나의 부모 필요 |

## 스타일 추가하기

React에서 스타일을 추가하는 방법은 여러 가지가 있습니다. 가장 기본적인 방법부터 알아봅시다.

### className 사용하기 (CSS 파일)

React에서는 HTML의 `class` 대신 `className`을 사용합니다.

```jsx
// ❌ HTML에서는 이렇게
<img class="avatar" />

// ✅ React/JSX에서는 이렇게
<img className="avatar" />
```

**왜 className을 쓰나요?**
- JavaScript에서 `class`는 예약어입니다 (클래스를 만들 때 쓰는 키워드)
- JSX는 JavaScript이기 때문에 `className`을 사용합니다

그런 다음 별도의 CSS 파일에 스타일을 작성합니다:

```css
/* styles.css 파일 */
.avatar {
  border-radius: 50%;    /* 둥근 모서리 */
  width: 100px;          /* 너비 */
  height: 100px;         /* 높이 */
  object-fit: cover;     /* 이미지 비율 유지 */
}
```

**CSS 파일 연결하기:**

```jsx
// App.jsx 파일 맨 위에
import './styles.css';  // CSS 파일을 불러옵니다

function App() {
  return <img className="avatar" src="profile.jpg" />;
}
```

### 인라인 스타일 사용하기

CSS 파일 없이 JavaScript 객체로 직접 스타일을 줄 수도 있습니다:

```jsx
// 방법 1: 직접 style 객체 작성
<div style={{ 
  color: 'blue',           // 문자열로 작성
  fontSize: '20px',        // 숫자+단위는 문자열
  backgroundColor: 'red'   // kebab-case 대신 camelCase!
}}>
  Hello
</div>

// 방법 2: 변수로 분리 (더 깔끔함)
const divStyle = {
  color: 'blue',
  fontSize: '20px',
  backgroundColor: 'red'
};

<div style={divStyle}>Hello</div>
```

**주의사항:**
- CSS: `background-color` (케밥-케이스)
- JSX: `backgroundColor` (카멜케이스)
- 중괄호가 2개인 이유: 바깥 `{}`는 "JavaScript 시작", 안쪽 `{}`는 "객체"

```jsx
style={{ color: 'red' }}
//     ↑              ↑
//   JS 시작        JS 객체
```

## 데이터 표시하기

### 중괄호 {} - JavaScript의 세계로!

JSX 안에서 **중괄호 `{}`**를 사용하면 JavaScript 코드를 실행할 수 있습니다.

```jsx
const name = "철수";

return <h1>안녕하세요, {name}님!</h1>;
// 화면에 표시: 안녕하세요, 철수님!
```

**중괄호의 역할:**
- JSX(HTML 같은 것) 안에서 JavaScript 변수나 표현식을 사용하고 싶을 때
- 중괄호 안에는 **모든 JavaScript 표현식**을 쓸 수 있습니다

### 변수 표시하기

```jsx
const user = {
  name: 'Hedy Lamarr',
  age: 30
};

return (
  <div>
    <h1>{user.name}</h1>           {/* 화면: Hedy Lamarr */}
    <p>나이: {user.age}세</p>       {/* 화면: 나이: 30세 */}
    <p>{user.age + 10}년 후</p>     {/* 화면: 40년 후 (계산 가능!) */}
  </div>
);
```

**코드 설명:**
- `{user.name}` - user 객체의 name 속성 값을 표시
- `{user.age}` - user 객체의 age 속성 값을 표시
- `{user.age + 10}` - 계산도 가능! (30 + 10 = 40)

### 속성(attribute)에서도 중괄호 사용

HTML 속성 값으로도 JavaScript 변수를 사용할 수 있습니다:

```jsx
const user = {
  name: 'Hedy Lamarr',
  imageUrl: 'https://example.com/photo.jpg'
};

return (
  <img
    className="avatar"              // 문자열은 따옴표
    src={user.imageUrl}             // 변수는 중괄호!
    alt={'Photo of ' + user.name}   // 문자열 합치기도 가능
  />
);
```

**주의할 점:**
```jsx
// ✅ 올바른 방법
<img src={user.imageUrl} />

// ❌ 잘못된 방법 (따옴표와 중괄호를 같이 쓰면 안됨!)
<img src="{user.imageUrl}" />
```

### 복잡한 표현식도 가능!

중괄호 안에서 더 복잡한 JavaScript 표현식도 사용할 수 있습니다:

```jsx
const user = {
  name: 'Hedy Lamarr',
  imageUrl: 'https://i.imgur.com/yXOvdOSs.jpg',
  imageSize: 90,
};

export default function Profile() {
  return (
    <>
      {/* 1. 변수 표시 */}
      <h1>{user.name}</h1>
      
      {/* 2. 여러 속성에서 변수 사용 */}
      <img
        className="avatar"
        src={user.imageUrl}
        alt={'Photo of ' + user.name}    // 문자열 연결
        style={{                          // 객체를 직접 만들어서 전달
          width: user.imageSize,          // 변수 값 사용
          height: user.imageSize
        }}
      />
    </>
  );
}
```

**코드 한 줄씩 설명:**

1. `<h1>{user.name}</h1>` 
   - user.name의 값인 "Hedy Lamarr"를 제목으로 표시

2. `src={user.imageUrl}`
   - 이미지 주소를 변수에서 가져옴

3. `alt={'Photo of ' + user.name}`
   - 문자열 "Photo of "와 user.name을 합침
   - 결과: "Photo of Hedy Lamarr"

4. `style={{ width: user.imageSize, height: user.imageSize }}`
   - 바깥쪽 `{}`: JSX에서 JavaScript 시작
   - 안쪽 `{}`: JavaScript 객체
   - `width: user.imageSize`: 너비를 90으로 설정
   - `height: user.imageSize`: 높이를 90으로 설정

:::tip 중괄호 사용 규칙 정리
- **JSX 태그 사이**: `<h1>{변수}</h1>` ✅
- **속성 값으로**: `src={변수}` ✅
- **따옴표 안에서**: `src="{변수}"` ❌ (작동 안 함!)
- **계산식**: `{2 + 2}` ✅ (결과: 4)
- **함수 호출**: `{getName()}` ✅
- **삼항 연산자**: `{isLoggedIn ? '로그인됨' : '로그아웃'}` ✅
:::

### style 속성의 이중 중괄호

```jsx
<div style={{ color: 'red', fontSize: '20px' }}>Hello</div>
```

**왜 중괄호가 2개일까요?**

```jsx
// 단계별로 이해하기
const styleObject = { color: 'red', fontSize: '20px' };  // 1. 먼저 객체 만듦
<div style={styleObject}>Hello</div>                     // 2. 객체를 전달

// 위 두 줄을 한 줄로 합치면
<div style={{ color: 'red', fontSize: '20px' }}>Hello</div>
//         ↑ JSX 문법  ↑ JavaScript 객체
```

- 첫 번째 `{}`: "여기서부터 JavaScript야!"
- 두 번째 `{}`: JavaScript 객체 `{ key: value }`

## 조건부 렌더링

React에는 조건문을 작성하는 특별한 문법이 없습니다. 대신 일반 JavaScript 코드를 작성할 때 사용하는 기법을 사용합니다. 예를 들어, `if` 문을 사용하여 조건부로 JSX를 포함할 수 있습니다:

```jsx
let content;
if (isLoggedIn) {
  content = <AdminPanel />;
} else {
  content = <LoginForm />;
}
return (
  <div>
    {content}
  </div>
);
```

더 간결한 코드를 선호한다면 삼항 연산자 `?`를 사용할 수 있습니다. `if`와 달리 JSX 내부에서 작동합니다:

```jsx
<div>
  {isLoggedIn ? (
    <AdminPanel />
  ) : (
    <LoginForm />
  )}
</div>
```

`else` 분기가 필요하지 않은 경우 더 짧은 논리 `&&` 문법을 사용할 수도 있습니다:

```jsx
<div>
  {isLoggedIn && <AdminPanel />}
</div>
```

:::tip 조건부 속성
이러한 모든 접근 방식은 속성을 조건부로 지정하는 데도 사용할 수 있습니다. JavaScript 문법에 익숙하지 않다면 항상 `if...else`를 사용하는 것부터 시작할 수 있습니다.
:::

## 리스트 렌더링

컴포넌트 리스트를 렌더링하려면 `for` 루프와 배열 `map()` 함수 같은 JavaScript 기능을 사용합니다.

예를 들어, 다음과 같은 제품 배열이 있다고 가정해봅시다:

```jsx
const products = [
  { title: 'Cabbage', id: 1 },
  { title: 'Garlic', id: 2 },
  { title: 'Apple', id: 3 },
];
```

컴포넌트 내부에서 `map()` 함수를 사용하여 제품 배열을 `<li>` 항목 배열로 변환합니다:

```jsx
const listItems = products.map(product =>
  <li key={product.id}>
    {product.title}
  </li>
);

return (
  <ul>{listItems}</ul>
);
```

:::warning key 속성의 중요성
`<li>`에 `key` 속성이 있는 것을 주목하세요. 리스트의 각 항목에 대해 형제 항목 중에서 해당 항목을 고유하게 식별하는 문자열이나 숫자를 전달해야 합니다. 일반적으로 key는 데이터베이스 ID와 같이 데이터에서 가져와야 합니다. React는 나중에 항목을 삽입, 삭제 또는 재정렬할 때 무슨 일이 일어났는지 알기 위해 key를 사용합니다.
:::

```jsx
const products = [
  { title: 'Cabbage', isFruit: false, id: 1 },
  { title: 'Garlic', isFruit: false, id: 2 },
  { title: 'Apple', isFruit: true, id: 3 },
];

export default function ShoppingList() {
  const listItems = products.map(product =>
    <li
      key={product.id}
      style={{
        color: product.isFruit ? 'magenta' : 'darkgreen'
      }}
    >
      {product.title}
    </li>
  );

  return (
    <ul>{listItems}</ul>
  );
}
```

## 이벤트에 응답하기

컴포넌트 내부에 이벤트 핸들러 함수를 선언하여 이벤트에 응답할 수 있습니다:

```jsx
function MyButton() {
  function handleClick() {
    alert('You clicked me!');
  }

  return (
    <button onClick={handleClick}>
      Click me
    </button>
  );
}
```

:::caution 이벤트 핸들러 전달 시 주의
`onClick={handleClick}`의 끝에 괄호가 없다는 점을 주목하세요! 이벤트 핸들러 함수를 *호출*하지 마세요. 단지 *전달*하기만 하면 됩니다. React는 사용자가 버튼을 클릭할 때 이벤트 핸들러를 호출합니다.
:::

## 화면 업데이트하기

종종 컴포넌트가 일부 정보를 "기억"하고 표시하기를 원할 것입니다. 예를 들어, 버튼이 클릭된 횟수를 세고 싶을 수 있습니다. 이를 위해 컴포넌트에 **state**를 추가합니다.

먼저 React에서 `useState`를 import합니다:

```jsx
import { useState } from 'react';
```

이제 컴포넌트 내부에 state 변수를 선언할 수 있습니다:

```jsx
function MyButton() {
  const [count, setCount] = useState(0);
  // ...
}
```

`useState`에서 두 가지를 얻습니다: 현재 state(`count`)와 이를 업데이트할 수 있는 함수(`setCount`). 이름은 원하는 대로 지을 수 있지만, 관례는 `[something, setSomething]`으로 작성하는 것입니다.

버튼이 처음 표시될 때 `count`는 `0`입니다. `useState()`에 `0`을 전달했기 때문입니다. state를 변경하려면 `setCount()`를 호출하고 새 값을 전달합니다. 이 버튼을 클릭하면 카운터가 증가합니다:

```jsx
function MyButton() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <button onClick={handleClick}>
      Clicked {count} times
    </button>
  );
}
```

React는 컴포넌트 함수를 다시 호출합니다. 이번에는 `count`가 `1`이 됩니다. 그다음에는 `2`가 됩니다. 계속 이어집니다.

동일한 컴포넌트를 여러 번 렌더링하면 각각 자체 state를 갖습니다:

```jsx
import { useState } from 'react';

export default function MyApp() {
  return (
    <div>
      <h1>Counters that update separately</h1>
      <MyButton />
      <MyButton />
    </div>
  );
}

function MyButton() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <button onClick={handleClick}>
      Clicked {count} times
    </button>
  );
}
```

각 버튼이 자체 `count` state를 "기억"하고 다른 버튼에 영향을 주지 않는 것을 주목하세요.

## Hooks 사용하기

`use`로 시작하는 함수를 **Hooks**라고 합니다. `useState`는 React에서 제공하는 내장 Hook입니다. [API 레퍼런스](https://react.dev/reference/react)에서 다른 내장 Hook을 찾을 수 있습니다. 기존 Hook을 결합하여 자신만의 Hook을 작성할 수도 있습니다.

Hooks는 다른 함수보다 제한적입니다. 컴포넌트(또는 다른 Hook)의 **최상위**에서만 Hook을 호출할 수 있습니다. 조건문이나 루프에서 `useState`를 사용하려면 새 컴포넌트를 추출하고 거기에 넣으세요.

:::danger Hooks 사용 규칙
- ✅ 컴포넌트의 최상위 레벨에서만 Hook 호출
- ✅ 다른 Hook의 최상위 레벨에서만 Hook 호출
- ❌ 조건문, 반복문 안에서 Hook 호출 금지
- ❌ 일반 JavaScript 함수에서 Hook 호출 금지
:::

## 컴포넌트 간 데이터 공유하기

이전 예제에서 각 `MyButton`은 독립적인 `count`를 가졌고, 각 버튼을 클릭했을 때 클릭된 버튼의 `count`만 변경되었습니다.

하지만 종종 컴포넌트가 **데이터를 공유하고 항상 함께 업데이트**되어야 할 필요가 있습니다.

두 `MyButton` 컴포넌트가 동일한 `count`를 표시하고 함께 업데이트되도록 하려면 개별 버튼에서 모든 버튼을 포함하는 가장 가까운 컴포넌트로 state를 "위로" 이동해야 합니다.

이 예제에서는 `MyApp`입니다:

```jsx
export default function MyApp() {
  const [count, setCount] = useState(0);

  function handleClick() {
    setCount(count + 1);
  }

  return (
    <div>
      <h1>Counters that update together</h1>
      <MyButton count={count} onClick={handleClick} />
      <MyButton count={count} onClick={handleClick} />
    </div>
  );
}

function MyButton({ count, onClick }) {
  return (
    <button onClick={onClick}>
      Clicked {count} times
    </button>
  );
}
```

이렇게 전달하는 정보를 **props**라고 합니다. 이제 `MyApp` 컴포넌트는 `count` state와 `handleClick` 이벤트 핸들러를 포함하고, 이 둘을 각 버튼에 props로 전달합니다.

마지막으로 `MyButton`을 변경하여 부모 컴포넌트에서 전달한 props를 읽도록 합니다:

```jsx
function MyButton({ count, onClick }) {
  return (
    <button onClick={onClick}>
      Clicked {count} times
    </button>
  );
}
```

버튼을 클릭하면 `onClick` 핸들러가 실행됩니다. 각 버튼의 `onClick` prop은 `MyApp` 내부의 `handleClick` 함수로 설정되었으므로, 그 안의 코드가 실행됩니다. 해당 코드는 `setCount(count + 1)`를 호출하여 `count` state 변수를 증가시킵니다. 새로운 `count` 값이 각 버튼에 prop으로 전달되므로 모두 새 값을 표시합니다. 이를 **"state 끌어올리기(lifting state up)"**라고 합니다. state를 위로 이동함으로써 컴포넌트 간에 공유했습니다.

:::tip State 끌어올리기
여러 컴포넌트가 동일한 데이터를 반영해야 할 때는 공통 부모 컴포넌트로 state를 끌어올리는 것이 좋습니다. 이는 React의 핵심 패턴 중 하나입니다.
:::
