# CodeIgniter란?

## 개요

CodeIgniter는 **웹 애플리케이션 개발을 위한 경량 PHP 프레임워크**입니다.
2006년 EllisLab에서 개발했으며, 현재는 BCIT(British Columbia Institute of Technology)에서 유지보수하고 있습니다.

:::tip 핵심 정의
CodeIgniter = 작은 설치 공간 + 뛰어난 성능 + 최소한의 설정 + 자유로운 코딩 스타일
:::

---

## CodeIgniter의 탄생 배경

### 당시의 문제 상황 (2006년)

PHP 프레임워크 시장은 복잡하고 무거운 프레임워크들이 주를 이루고 있었습니다:

**기존 프레임워크들의 문제점**:
- 복잡한 설정과 높은 학습 곡선
- 무거운 라이브러리로 인한 성능 저하
- 엄격한 코딩 규칙 강요
- 커맨드 라인 사용 필수
- PEAR 같은 거대한 의존성

**개발자들의 불편함**:
- 간단한 프로젝트에도 과도한 설정
- 빠른 개발이 어려움
- 공유 호스팅 환경에서 사용 제한

### EllisLab의 비전
```
"간단하고 빠르며, 성능이 뛰어난
가벼운 프레임워크가 필요하다"
```

이 비전에서 CodeIgniter가 탄생했습니다.

## CodeIgniter의 개발 철학

CodeIgniter는 다음과 같은 원칙을 바탕으로 설계되었습니다:

### 1. 작은 설치 공간 (Small Footprint)

**핵심 가치**: "필요한 것만 로드한다"
```php
// 필요한 라이브러리만 선택적 로드
$this->load->library('email');
$this->load->helper('url');
```

**장점**:
- 빠른 실행 속도
- 적은 메모리 사용
- 공유 호스팅에서도 쾌적한 동작

---

### 2. 뛰어난 성능 (Exceptional Performance)

**측정 결과**: 다른 주요 프레임워크 대비 2-3배 빠른 처리 속도
```php
// 간결한 코드로 빠른 처리
$query = $this->db->get('users');
foreach ($query->result() as $row) {
    echo $row->name;
}
```

**성능 최적화 전략**:
- 느슨한 결합 (Loose Coupling)
- 필요할 때만 컴포넌트 로드
- 가벼운 코어 시스템

---

### 3. 최소한의 설정 (Nearly Zero Configuration)

**철학**: "설정보다 실행"
```php
// config/database.php에서 DB 설정만 하면 끝
$db['default'] = array(
    'hostname' => 'localhost',
    'username' => 'root',
    'password' => '',
    'database' => 'mydb'
);
```

**장점**:
- 즉시 개발 시작 가능
- 복잡한 설정 파일 불필요
- 빠른 프로토타이핑

---

### 4. 자유로운 코딩 스타일 (Flexibility)

**원칙**: "강요하지 않는다"
```php
// 프레임워크 사용 선택적
// MVC를 원하지 않으면 사용하지 않아도 됨
// 템플릿 엔진도 선택사항
echo "Hello, World!"; // 이것도 가능
```

**자유**:
- 엄격한 코딩 규칙 없음
- 원하는 구조로 개발 가능
- 라이브러리 선택적 사용

---

## CodeIgniter의 주요 개념

### MVC 아키텍처 (선택적)

CodeIgniter는 MVC 패턴을 **권장**하지만 **강요하지 않습니다**:
```
Request
  ↓
Router → Controller → Model (Database)
           ↓
        View
           ↓
        Response
```

**특징**: MVC를 사용하지 않고 직접 로직을 작성할 수도 있습니다.

---

### Controller (컨트롤러)

요청을 처리하는 클래스:
```php
class Welcome extends CI_Controller {
    public function index() {
        $this->load->view('welcome_message');
    }
    
    public function hello($name = 'World') {
        echo "Hello, {$name}!";
    }
}
```

**URL 매핑**: `example.com/welcome/hello/John`

---

### Model (모델)

데이터베이스 작업을 처리:
```php
class User_model extends CI_Model {
    public function get_users() {
        $query = $this->db->get('users');
        return $query->result_array();
    }
    
    public function insert_user($data) {
        return $this->db->insert('users', $data);
    }
}
```

**사용법**:
```php
$this->load->model('user_model');
$users = $this->user_model->get_users();
```

---

### View (뷰)

HTML 출력을 담당:
```php
<!-- application/views/user_list.php -->
<h1>사용자 목록</h1>
<ul>
    <?php foreach($users as $user): ?>
        <li><?= $user['name'] ?></li>
    <?php endforeach; ?>
</ul>
```

**컨트롤러에서 로드**:
```php
$data['users'] = $this->user_model->get_users();
$this->load->view('user_list', $data);
```

---

### Database Query Builder

직관적인 데이터베이스 작업:
```php
// SELECT
$query = $this->db->get('users');

// WHERE
$this->db->where('status', 'active');
$query = $this->db->get('users');

// JOIN
$this->db->select('users.name, posts.title');
$this->db->from('users');
$this->db->join('posts', 'posts.user_id = users.id');
$query = $this->db->get();

// INSERT
$data = array(
    'name' => '홍길동',
    'email' => 'hong@example.com'
);
$this->db->insert('users', $data);
```

**장점**: SQL을 직접 작성하지 않아도 됨

---

### Libraries (라이브러리)

다양한 기능을 제공하는 라이브러리:
```php
// 이메일 전송
$this->load->library('email');
$this->email->from('your@example.com');
$this->email->to('someone@example.com');
$this->email->subject('제목');
$this->email->message('내용');
$this->email->send();

// 세션 관리
$this->load->library('session');
$this->session->set_userdata('username', 'hong');
$username = $this->session->userdata('username');

// 파일 업로드
$this->load->library('upload', $config);
$this->upload->do_upload('userfile');
```

---

### Helpers (헬퍼)

자주 사용하는 함수 모음:
```php
// URL 헬퍼
$this->load->helper('url');
echo site_url('welcome/hello');
echo base_url('assets/css/style.css');

// Form 헬퍼
$this->load->helper('form');
echo form_open('user/submit');
echo form_input('username', 'John');
echo form_close();

// 텍스트 헬퍼
$this->load->helper('text');
echo word_limiter($text, 100);
```

---

### Routing (라우팅)

URL과 컨트롤러 매핑:
```php
// config/routes.php
$route['default_controller'] = 'welcome';
$route['product/(:num)'] = 'catalog/product_lookup/$1';
$route['products/([a-z]+)/(\d+)'] = 'catalog/product_lookup_by_name/$1/$2';
```

**예시**:
- `example.com/product/123` → `Catalog::product_lookup(123)`
- `example.com/products/shoes/42` → `Catalog::product_lookup_by_name('shoes', 42)`

## CodeIgniter가 해결한 문제들

### 1. 복잡한 프레임워크

**문제**: 대형 프레임워크의 복잡성  
**해결**: 간단한 구조와 직관적인 사용법

---

### 2. 느린 성능

**문제**: 무거운 프레임워크의 성능 저하  
**해결**: 가벼운 코어와 선택적 컴포넌트 로딩

---

### 3. 가파른 학습 곡선

**문제**: 복잡한 개념과 많은 학습량  
**해결**: 간단한 API와 명확한 문서
```php
// 배우기 쉬운 문법
$this->load->model('user_model');
$users = $this->user_model->get_all();
$this->load->view('users', ['users' => $users]);
```

---

### 4. 공유 호스팅 제약

**문제**: 커맨드 라인 접근 불가  
**해결**: 웹 기반 설치와 관리

---

### 5. 과도한 의존성

**문제**: PEAR 같은 거대한 라이브러리  
**해결**: 필요한 기능만 포함된 경량 라이브러리

## CodeIgniter의 강점과 약점

### 강점 ✅

1. **뛰어난 성능**: 가장 빠른 PHP 프레임워크 중 하나
2. **작은 설치 공간**: 다운로드 용량 약 1.2MB
3. **쉬운 학습**: 초보자도 하루면 기본 학습 가능
4. **명확한 문서**: 한국어 번역 포함, 잘 정리된 문서
5. **낮은 서버 요구사항**: 저사양 환경에서도 동작
6. **자유로운 구조**: 원하는 방식으로 코딩 가능
7. **커맨드 라인 불필요**: 웹 기반으로 모든 작업 가능
8. **PHP 기본 문법**: 특별한 템플릿 언어 학습 불필요

---

### 약점 ❌

1. **작은 생태계**: Laravel 대비 적은 패키지
2. **모던한 기능 부족**: 최신 PHP 기능 활용 제한 (CI3)
3. **ORM 미제공**: 기본 Query Builder만 제공
4. **커뮤니티 규모**: Laravel 대비 작은 커뮤니티
5. **인재 풀**: 구인/구직 시장이 작음
6. **CI3와 CI4 단절**: 버전 간 호환성 없음

---

## CodeIgniter vs Laravel

### 비교표

| 항목 | CodeIgniter | Laravel |
|------|------------|---------|
| **성능** | 매우 빠름 ⚡ | 중간 |
| **학습 난이도** | 매우 쉬움 📚 | 중간 |
| **설치 공간** | 작음 (1.2MB) | 큼 (10MB+) |
| **생태계** | 작음 | 매우 큼 🌟 |
| **문서 품질** | 좋음 | 최고 |
| **커뮤니티** | 중간 | 매우 큼 |
| **ORM** | 없음 | Eloquent |
| **CLI 도구** | 제한적 | 풍부 (Artisan) |
| **최신 트렌드** | 보수적 | 최신 기술 적극 도입 |

---

### CodeIgniter를 선택해야 하는 경우

**✅ CodeIgniter 추천**:

- 작은 규모의 웹사이트

## CodeIgniter가 적합한 경우

CodeIgniter는 다음과 같은 상황에 최적입니다:

### ✅ 적합한 경우

1. **작은 설치 공간이 필요할 때**
   - 공유 호스팅 환경
   - 저사양 서버
   - 빠른 배포 필요

2. **뛰어난 성능이 필요할 때**
   - 트래픽이 많은 사이트
   - 응답 속도가 중요한 API
   - 리소스 제약이 있는 환경

3. **최소한의 설정으로 시작하고 싶을 때**
   - 빠른 프로토타이핑
   - MVP 개발
   - 학습용 프로젝트

4. **커맨드 라인을 사용할 수 없을 때**
   - 제한된 호스팅 환경
   - Windows 환경에서의 개발
   - GUI만 사용 가능한 환경

5. **엄격한 코딩 규칙을 원하지 않을 때**
   - 자유로운 구조 선호
   - 기존 PHP 코드와 통합
   - 점진적 마이그레이션

6. **템플릿 언어를 배우고 싶지 않을 때**
   - 순수 PHP 선호
   - 기존 HTML 활용
   - 디자이너와 협업

7. **간단한 솔루션을 선호할 때**
   - 복잡성 최소화
   - 빠른 이해와 수정
   - 작은 팀 운영

8. **명확한 문서를 원할 때**
   - 한국어 문서 필요
   - 예제 코드 중심 학습
   - 빠른 문제 해결

## CodeIgniter의 철학

### 1. 단순함 (Simplicity)
```php
// 단순하고 명확한 코드
$this->db->where('id', $id);
$this->db->update('users', $data);
```

---

### 2. 성능 (Performance)
```php
// 필요한 것만 로드
$this->load->database();
$this->load->helper('url');
```

---

### 3. 자유 (Freedom)
```php
// 원하는 방식으로 작성
// MVC를 따르지 않아도 됨
class Simple extends CI_Controller {
    public function index() {
        echo "Hello!";
    }
}
```

---

### 4. 명확성 (Clarity)
```php
// 무슨 일을 하는지 명확
$this->email->from('me@example.com');
$this->email->to('you@example.com');
$this->email->subject('안녕하세요');
$this->email->message('메시지 내용');
$this->email->send();
```

## CodeIgniter 3 vs 4

### 주요 차이점

| 특징 | CodeIgniter 3 | CodeIgniter 4 |
|------|---------------|---------------|
| **PHP 버전** | 5.6+ | 7.4+ |
| **네임스페이스** | 없음 | 사용 |
| **PSR-4** | 미지원 | 지원 |
| **CLI** | 제한적 | 강력한 spark 도구 |
| **성능** | 빠름 | 더 빠름 |
| **구조** | 레거시 | 모던 |

### CodeIgniter 4의 개선사항
```php
// CI4: 네임스페이스 사용
namespace App\Controllers;

class Home extends BaseController {
    public function index() {
        return view('welcome_message');
    }
}
```
```bash
# CI4: 강력한 CLI 도구
php spark make:controller User
php spark make:model User
php spark serve
```
