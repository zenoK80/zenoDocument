### 2️⃣ 그래프의 정의와 핵심 용어
**파일**: `02-graph-definition-terms.mdx`

#### 목표
- 그래프 용어를 정확하게 이해하기
- 정점, 간선, 차수 등의 개념 습득
- 경로와 사이클의 차이 파악

#### 주요 내용
- **정점(Vertex)**: 그래프의 기본 단위
- **간선(Edge)**: 정점 간의 연결
- **인접 정점**: 직접 연결된 정점들
- **차수(Degree)**: 연결된 간선 개수
  - 무방향: deg(v)
  - 방향: in-deg, out-deg
- **경로(Path)**: 정점들을 잇는 수열
- **사이클(Cycle)**: 시작과 끝이 같은 경로
- **연결 그래프**: 모든 정점이 연결됨

#### 배우는 핵심 용어
```
V(정점), E(간선), deg(차수), Path(경로), Cycle(사이클)
```

#### ⏱️ 예상 시간
- 읽기: 25-30분
- 이해 + 연습: 30-40분
- **합계: 55-70분**

### 3️⃣ 그래프의 다양한 종류
**파일**: `03-graph-types.mdx`

#### 목표
- 무방향/방향 그래프 구분하기
- 가중치 그래프 이해하기
- 부분 그래프 개념 습득

#### 주요 내용
- **무방향 그래프**: 양방향 이동 가능 (SNS 친구)
- **방향 그래프**: 한쪽 방향만 (팔로우)
- **가중치 그래프**: 간선에 비용/거리 등 추가 정보
- **부분 그래프**: 원본의 일부만 선택
- **특수 그래프**:
  - 완전 그래프: 모든 정점 연결
  - 사이클 그래프: 원형 구조
  - 이분 그래프: 두 그룹으로 나뉨

#### 선택 기준
```
상황에 따라 적절한 그래프 선택하기
→ SNS 친구관계 = 무방향
→ 팔로우 = 방향
→ 네비게이션 = 가중치
```

#### ⏱️ 예상 시간
- 읽기: 20-25분
- 이해 + 비교: 25-35분
- **합계: 45-60분**

### 4️⃣ 그래프를 프로그래밍으로 표현하는 방법
**파일**: `04-graph-representation.mdx`

#### 목표
- 인접 행렬이 무엇인지 이해하기
- 인접 리스트가 무엇인지 이해하기
- 상황에 따른 선택 기준 학습

#### 주요 내용
- **인접 행렬**: 2D 배열로 표현
  - 장점: O(1) 조회 빠름
  - 단점: O(N²) 메모리 낭비
  - 추천: 정점 적음, 간선 많음
  
- **인접 리스트**: 연결 리스트로 표현
  - 장점: O(N+E) 효율적
  - 단점: 조회가 느림
  - 추천: 정점 많음, 간선 적음

#### 비교표
```
        인접 행렬    인접 리스트
메모리  O(N²)      O(N+E) ⭐
조회    O(1) ⭐    O(degree)
순회    O(N²)      O(N+E) ⭐
```

#### ⏱️ 예상 시간
- 읽기: 20-25분
- 개념 이해: 25-30분
- **합계: 45-55분**

## 🔑 꼭 기억해야 할 것

### 1. 용어는 정확하게
```
"정점"과 "간선"은 다릅니다!
정점 = 노드 = 데이터
간선 = 링크 = 연결
```

### 2. 구조체 필드 이해
```c
typedef struct {
    int n;                    // ⭐ 정점 개수
    int adj_matrix[50][50];   // ⭐ 인접 행렬
} Graph;
```

### 3. malloc과 free
```c
할당: g = (Graph *)malloc(sizeof(Graph));
해제: free(g);
→ 반드시 쌍을 이루어야 함!
```

### 4. 무방향 vs 방향
```
무방향: (u,v) = (v,u)    → 양쪽을 모두 1로
방향:   (u,v) ≠ (v,u)    → 한쪽만 1로
```
