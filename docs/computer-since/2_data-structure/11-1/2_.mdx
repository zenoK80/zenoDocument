## 📚 들어가기 전에

지금까지 여러분이 배운 **선형 자료구조**(배열, 링크드 리스트) 에서 나아가 이제 **비선형 자료구조**의 끝판왕인 **그래프**를 배웁니다. 이것이 마지막 단계이면서 가장 강력한 자료구조입니다!

---

## 🎯 그래프란?

**그래프(Graph)**는 다음 세 가지를 포함하는 자료구조입니다:

1. **객체(Object)** - 여러 개의 의미 있는 데이터들
2. **관계(Relationship)** - 객체들 사이의 연결 관계
3. **연결 구조** - 객체들이 연결되어 있는 형태

### 쉬운 비유로 이해하기 🌍

생각해봅시다. 다음의 예시들이 모두 그래프로 표현될 수 있습니다:

- **도시 네트워크**: 서울, 부산, 대구 같은 도시들이 도로로 연결되어 있음
- **지하철 노선**: 각 역(정점)이 선로(간선)로 연결되어 있음
- **전자회로**: 전자부품들이 전선으로 연결되어 있음
- **소셜 네트워크**: 사람들이 친구 관계로 연결되어 있음
- **웹페이지**: 페이지들이 하이퍼링크로 연결되어 있음

### 트리와의 차이점 ⚡

여러분이 배운 **트리(Tree)**와 그래프의 차이는 무엇일까요?

| 특징 | 트리 | 그래프 |
|------|------|--------|
| 연결 관계 | 1대1 (부모-자식) | **다대다 (n대n)** |
| 사이클 | 없음 (항상 트리) | 있을 수 있음 |
| 계층 구조 | 있음 (위/아래) | **없음 (동등한 관계)** |
| 구조 | 선형 | **비선형** |

✨ **핵심**: 그래프는 **모든 노드가 동등한 관계**를 가지는 구조입니다!

## 🌟 역사 속의 그래프: 오일러와 다리 문제

그래프 이론은 **1800년대에 오일러(Euler)**라는 수학자가 만들었습니다. 

### 📖 유명한 문제: 쾨니히스베르크의 다리 문제

#### 문제 상황
어떤 도시에는 4개의 지역과 7개의 다리가 있습니다. 각 지역을 연결하는 다리가 있는데...

**질문**: 모든 다리를 **정확히 한 번씩만** 건너서 원래 출발했던 지역으로 돌아올 수 있을까요?

#### 지도로 표현
```
   지역 A ─── 다리 ─── 지역 B
    │  ╲       │        ╱  │
    │   ╲      │       ╱   │
   다리  다리  다리  다리  다리
    │     ╲    │    ╱     │
    │      ╲   │   ╱      │
   지역 D ─── 다리 ─── 지역 C
```

#### 오일러의 해결책
이 복잡한 도시 지도를 **그래프로 단순화**했습니다:

```
    A ─── B
    │  ╲  │ ╱
    │   ╲ │╱
    D ─── C
```

이렇게 그래프로 표현하니 문제가 훨씬 쉬워졌고, 오일러는 **수학적 정리**를 만들었습니다!

### 📌 오일러 정리

> 모든 정점에 연결된 간선의 수가 **짝수**일 경우, 모든 간선을 **한 번씩만** 거쳐서 원래대로 돌아올 수 있다.

#### 위의 예시에서:
- 지역 A: 3개의 다리 연결 (홀수) ❌
- 지역 B: 3개의 다리 연결 (홀수) ❌
- 지역 C: 3개의 다리 연결 (홀수) ❌
- 지역 D: 3개의 다리 연결 (홀수) ❌

→ **답: 불가능합니다!** 모든 정점이 홀수 차수를 가지므로 불가능하죠.

## 💡 그래프를 배우는 이유

### 실제 활용 예시

#### 1️⃣ **네비게이션**
- 지도의 도시/교차로: 정점
- 도로: 간선
- 도로 길이/소요 시간: 가중치
→ 최단 경로를 찾아줍니다!

#### 2️⃣ **대학 선수 과목**
```
CS1 ──→ CS2 ──→ CS5
           └──→ CS3 ──→ CS4
```
- CS5를 들으려면 CS2, CS3를 먼저 들어야 함
- 이런 선수 관계를 그래프로 표현

#### 3️⃣ **소셜 네트워크**
- 사람들 간의 친구 관계를 그래프로 모델링
- "친구의 친구" 찾기 등에 사용

#### 4️⃣ **컴퓨터 네트워크**
- 각 컴퓨터: 정점
- 네트워크 연결: 간선
- 데이터 전송 경로 설계
