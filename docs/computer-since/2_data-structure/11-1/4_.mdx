## 🎨 그래프는 다양한 모습을 가져요

그래프는 문제의 특성에 따라 여러 종류로 나뉩니다. 마치 다양한 종류의 차가 있듯이요!

---

## 🔄 무방향 그래프 (Undirected Graph)

### 특징

간선에 **방향이 없는** 그래프입니다.

**간선 표현**: `(u, v) = (v, u)` → 순서가 같음

### 시각으로 보기

```
A ─── B
│     │
C ─── D
```

- A↔B : 양쪽 방향 모두 가능
- A-B 또는 B-A : 똑같은 간선

### 실제 예시 📍

- **SNS의 친구 관계**: A가 B의 친구면, B도 A의 친구
- **도시의 도로**: 서울↔부산 양쪽으로 이동 가능
- **협력 관계**: A가 B와 협력하면, B도 A와 협력

### 인접 행렬 표현

```
  A B C D
A 0 1 1 0
B 1 0 1 1
C 1 1 0 1
D 0 1 1 0

대칭 구조! (1,0)과 (0,1)이 같음)
```

---

## ➡️ 방향 그래프 (Directed Graph)

### 특징

간선에 **방향이 있는** 그래프입니다.

**간선 표현**: `(u, v) ≠ (v, u)` → 순서가 중요!

### 시각으로 보기

```
A ──→ B
│     │
│     ↓
└──→ C
```

- A→B : A에서 B로만 이동 가능
- B→A : 불가능!

### 실제 예시 🎯

- **대학 선수 과목**: CS1→CS2→CS5 (순서 중요!)
- **페이스북 팔로우**: A가 B를 팔로우해도, B가 A를 팔로우할 필요 없음
- **링크드인 팔로잉**: 일방적 관계
- **웹페이지 링크**: A→B 링크가 있어도 B→A 링크는 없을 수 있음

### 진입/진출 차수 복습

```
  1 ──→ 2
  ↑     ↓
  └──── 3
```

| 정점 | 진입 차수 | 진출 차수 |
|------|---------|---------|
| 1 | 1 | 1 |
| 2 | 1 | 1 |
| 3 | 1 | 1 |

### 인접 행렬 표현

```
  1 2 3
1 0 1 0
2 0 0 1
3 1 0 0

비대칭 구조! (1,0)과 (0,1)이 다름)
```

## ⚖️ 가중치 그래프 / 네트워크 (Weighted Graph / Network)

### 특징

간선에 **가중치(비용, 거리, 시간 등)가 있는** 그래프입니다.

```
일반 그래프:
서울 ─── 부산

가중치 그래프:
서울 ─30km─ 부산    (또는 2시간, 5000원 등)
```

### 가중치의 의미

| 적용 분야 | 정점 | 간선 | 가중치 |
|---------|------|------|--------|
| 네비게이션 | 도시 | 도로 | 거리/시간 |
| 항공편 | 공항 | 항공로 | 항공료/시간 |
| 우정도 | 사람 | 관계 | 친절도 |
| 통신 | 컴퓨터 | 회선 | 대역폭/비용 |

### 가중치 그래프의 예시 🌍

```
    10      15
서울 ──── 대전 ──── 부산
 │              │
 │ 8km         │ 20km
 │              │
 └──── 대구 ────┘
    12

가중치의 의미: 거리(km)
최단 경로: 서울 → 대구(12km) → 부산(20km) = 32km
```

### 왜 가중치가 필요할까? 🤔

```
네비게이션을 생각해봅시다.

시나리오 1) 가중치 = 거리
서울 → 대전(167km) → 부산 vs 서울 → 대구(309km) → 부산
→ 거리가 짧은 경로 추천

시나리오 2) 가중치 = 혼잡도
서울 → 대전(1) → 부산 vs 서울 → 대구(0) → 부산
→ 덜 혼잡한 경로 추천 (실시간 교통 정보 활용)
```

## 🌳 부분 그래프 (Subgraph)

### 정의

**원본 그래프에서 일부 정점과 간선을 선택한 그래프**입니다.

### 수학적 표현

원본 그래프: $G = (V, E)$

부분 그래프: $G' = (V', E')$ 

조건: $V' \subseteq V$ 이고 $E' \subseteq E$

(부분 그래프의 정점과 간선은 모두 원본에 포함되어야 함)

### 시각적 예시

```
원본 그래프 G:
    0 ─── 1
    │ ╲   │
    2 ─── 3

부분 그래프 G₁:          부분 그래프 G₂:
0 ─── 1                  1 ─── 3
│           (정점 2,3 제거)

0 ─── 2                  0 ─── 1
      │    (정점 1,3 제거)      │
      3                        3
```

### 실제 응용 📋

```
전체 소셜 네트워크에서:
- "서울 지역 사람들만의 친구 관계"
- "2024년 데이터만 분석"
- "특정 주제 관심사 그룹"
→ 이런 것들이 모두 부분 그래프!
```

## 특수한 그래프들

### 완전 그래프 (Complete Graph) 🌟

**모든 정점이 서로 직접 연결된 그래프**

```
n개 정점의 완전 그래프: K_n

K₃ (삼각형):           K₄ (사면체):
  0                      0
 / \                     /|\
1---2                   1-+-2
                         \|/
                          3

정점 개수: 3            정점 개수: 4
간선 개수: 3            간선 개수: 6
(3-1)/2 × 3             (4-1)/2 × 4
```

### 사이클 그래프 (Cycle Graph) ⭕

**정점들이 원형으로 연결된 그래프**

```
C₄ (4-사이클):
  0
 / \
3   1
 \ /
  2

경로: 0→1→2→3→0
```

### 이분 그래프 (Bipartite Graph) 🔄

**정점을 두 그룹으로 나눈 후, 같은 그룹 내 연결이 없는 그래프**

```
그룹 A: {0, 1}
그룹 B: {2, 3}

  0 ─── 2
  │ ╱   │
  1 ─── 3

특징: 같은 색끼리 연결 안 됨
```

## 📊 그래프 종류 비교표

| 특징 | 무방향 | 방향 | 가중치 | 부분 |
|------|--------|------|--------|------|
| 간선 방향 | ❌ | ✅ | 상관없음 | 상관없음 |
| 간선 가중치 | 없음 | 없음 | ✅ | 부모에 따름 |
| 차수 계산 | deg(v) | in-deg, out-deg | 차수 + 가중치 | 범위 내에서만 |
| 사용 예 | SNS친구 | 팔로우 | 네비게이션 | 데이터 필터링 |
| 인접행렬 | 대칭 | 비대칭 | 숫자값 | 부분집합 |

## 🎯 그래프 선택하기

다음 상황에서 **어떤 그래프**를 사용할까요?

### 상황 1: 지구 위의 도시들
```
지도상 도시 네트워크
→ 무방향 그래프 (또는 가중치 그래프 추가 가능)
   왜냐하면 도로는 양방향이니까!
```

### 상황 2: 대학원의 학과 간 협력
```
A학과 → B학과 → C학과 → A학과
→ 방향 그래프
   왜냐하면 "공동 연구"는 방향이 있을 수 있음!
```

### 상황 3: 페이스북 친구 추천
```
당신과 친구 관계들
→ 무방향 그래프
   왜냐하면 친구는 상호 관계!
```

### 상황 4: 트위터 팔로우
```
팔로잉 관계들
→ 방향 그래프
   왜냐하면 팔로우는 일방적일 수 있음!
```

---

## 🔑 핵심 정리

1. **무방향 그래프**: 간선에 방향 없음 → 인접행렬은 대칭
2. **방향 그래프**: 간선에 방향 있음 → 인접행렬은 비대칭
3. **가중치 그래프**: 간선에 비용/거리 등 추가 정보
4. **부분 그래프**: 원본의 일부만 선택한 그래프

---

## ❓ 생각해보기

1. 당신의 가족관계는 무방향인가 방향인가?
   → **무방향** (아버지-아들은 상호 관계)

2. 지메일 수신함은?
   → **방향** (A가 B에게 메일 보냈어도, B가 A에게 보냈을 필요 없음)

3. 비행기 항로는?
   → **가중치 그래프** (거리, 시간, 항공료 등이 중요)

4. 자신과 1촌인 친구들의 관계만 보면?
   → **부분 그래프** (전체 SNS에서 일부만 추출)
