## 🎯 그래프 탐색이란?

**그래프 탐색**은 하나의 정점에서 시작하여 **이어진 모든 정점을 차례대로 한 번씩 방문**하는 과정입니다.

### 핵심 포인트

```
✓ 한 정점에서 시작
✓ 모든 정점을 한 번씩 방문 (중복 방문 금지!)
✓ 체계적인 순서로 진행
```

---

## 💡 왜 그래프 탐색이 필요할까?

### 실제 응용 사례

#### 도로망에서의 연결성 확인
```
서울 ─── 부산
  │        │
대전 ─── 대구

질문: 서울에서 부산으로 갈 수 있나?
→ 탐색으로 경로 확인 가능
```

#### 전자회로의 연결성 확인
```
단자 A ─── 단자 B
  │          │
단자 C ─── 단자 D

질문: 단자 A와 B가 연결되어 있나?
→ 탐색으로 연결 상태 파악
```

#### SNS의 친구 관계 탐색
```
내가 아는 사람 → ? → ? → 원하는 사람
→ 최단 경로 찾기 등에 사용
```

### 탐색의 중요성

많은 그래프 관련 문제들이 **단순히 노드를 탐색하는 것만으로 해결**됩니다.

## 🔀 두 가지 탐색 방식

### 깊이 우선 탐색 (DFS: Depth-First Search)

**한 방향으로 계속 깊게 들어갔다가, 더 이상 갈 수 없으면 되돌아가서 다른 경로 탐색**

```
시작 정점: 0

단계:
0 → 1 → 2 → 3 → 4 (막힘)
     ↑
     되돌아가서 다른 경로 탐색
```

#### 특징
- **깊이 우선**: 깊게 들어가기
- **되돌아가기**: 가장 가까운 갈림길로 복귀
- **자료구조**: 스택(Stack) 사용
- **구현**: 재귀함수 또는 명시적 스택

### 너비 우선 탐색 (BFS: Breadth-First Search)

**시작 정점에서 가까운 정점들을 먼저 방문, 멀리 떨어진 정점은 나중에 방문**

```
시작 정점: A

단계:
Level 1: A 방문
Level 2: A의 이웃들 (B, C) 방문
Level 3: B, C의 이웃들 (D, E, F) 방문
...
```

#### 특징
- **너비 우선**: 한 단계씩 확장
- **가까운 것부터**: 거리가 가까운 정점부터
- **자료구조**: 큐(Queue) 사용
- **구현**: 반복문 + 큐

---

## 📊 DFS vs BFS 한눈에 보기

| 특성 | DFS | BFS |
|------|-----|-----|
| 탐색 순서 | 깊게 들어가기 | 넓게 퍼지기 |
| 자료구조 | 스택 | 큐 |
| 구현 방법 | 재귀함수 | 반복문 |
| 메모리 | 스택 메모리 | 큐 메모리 |
| 최단 경로 | 보장 안 함 | 보장함 |
| 구현 난이도 | 쉬움 | 중간 |

## 🔍 DFS의 원리

### 기본 개념

```
깊이 우선 탐색은 다음과 같이 동작합니다:

1. 현재 정점을 방문했다고 표시
2. 현재 정점의 이웃 정점들 중에서
3. 아직 방문하지 않은 정점을 찾음
4. 찾은 정점으로 이동 (재귀 호출)
5. 더 이상 갈 곳이 없으면 돌아가기 (백트래킹)
```

### 예시: 미로 찾기

```
마우스가 미로를 헤맬 때:

갈림길 1 → 막힘 → 되돌아가기
         → 갈림길 2 → 막힘 → 되돌아가기
                    → 갈림길 3 → 출구!

가장 최근 갈림길로 돌아가기 = 스택 구조
```

### 스택과의 관계

```c
// 재귀 호출할 때
dfs(v);
  dfs(u1);  // u1의 복귀주소를 스택에 저장
    dfs(u2); // u2의 복귀주소를 스택에 저장
    // 역순으로 복귀 (가장 최근 주소부터)
```

**재귀함수는 자동으로 스택을 사용합니다!** (묵시적 스택)

## 🎯 BFS의 원리

### 기본 개념

```
너비 우선 탐색은 다음과 같이 동작합니다:

1. 시작 정점을 큐에 넣고 방문 표시
2. 큐가 비워질 때까지:
   a) 큐에서 정점 꺼내기
   b) 그 정점의 이웃들 중 미방문 정점 찾기
   c) 미방문 정점을 큐에 넣고 방문 표시
   d) 반복
```

### 예시: 층별 탐색

```
시작: A

1단계: [A 방문]
2단계: [A의 이웃 B, C 방문]
3단계: [B의 이웃 D, E 방문]
       [C의 이웃 F 방문]
4단계: ...

큐에 먼저 들어온 것부터 처리 = FIFO (First In First Out)
```

## 🔑 핵심 개념 정리

### 탐색이 필요한 이유

| 문제 | 해결 방법 |
|------|---------|
| 연결성 확인 | 탐색으로 도달 가능한지 확인 |
| 모든 정점 방문 | 탐색으로 체계적 방문 |
| 경로 찾기 | BFS로 최단 경로 찾기 |
| 사이클 감지 | DFS로 사이클 찾기 |

### DFS 특징

```
깊이 우선 탐색 (DFS)
├─ 스택 사용 (명시적 또는 묵시적)
├─ 한 방향으로 계속 진행
├─ 막혔을 때 되돌아가기
├─ 재귀함수로 구현 가능
└─ 구현이 간단
```

### BFS 특징

```
너비 우선 탐색 (BFS)
├─ 큐 사용
├─ 한 단계씩 확장
├─ 가까운 것부터 먼저
├─ 반복문으로 구현
├─ 최단 경로 보장
└─ 구현이 다소 복잡
```

## 📋 주의사항

### 탐색 순서는 보장되지 않음

```
문제에서 특별히 지정하지 않으면,
어느 순서로 탐색해도 상관없습니다.

하지만 시험 문제에서는 보통:
"오름차순으로 방문한다" 등의 조건이 있습니다.
→ 이 경우 반드시 그 순서대로 진행해야 합니다!
```

### 중복 방문 금지

```
✓ 올바른 탐색: 0 → 1 → 2 → 3 (각 1번씩)
✗ 잘못된 탐색: 0 → 1 → 0 → 2 (0을 2번 방문)

방문 여부를 기록하는 배열이 필수입니다!
```
