# 함수 사용의 실제 장점

## 🎯 함수의 3가지 주요 장점

함수를 배우는 것이 왜 중요한지 구체적으로 살펴보겠습니다.

## 1️⃣ 코드 반복을 줄인다

### ❌ 함수를 사용하지 않은 경우

학생 정보를 3명분 출력해야 한다고 하겠습니다.

```c
#include <stdio.h>

int main() {
    // 첫 번째 학생
    printf("=== 학생 정보 ===\n");
    printf("이름: 김철수\n");
    printf("학번: 001\n");
    printf("학점: A+\n");
    printf("\n");
    
    // 두 번째 학생 (같은 코드 반복!)
    printf("=== 학생 정보 ===\n");
    printf("이름: 이영희\n");
    printf("학번: 002\n");
    printf("학점: A\n");
    printf("\n");
    
    // 세 번째 학생 (또 같은 코드 반복!)
    printf("=== 학생 정보 ===\n");
    printf("이름: 박민준\n");
    printf("학번: 003\n");
    printf("학점: B+\n");
    printf("\n");
    
    return 0;
}
```

**문제점:**
- 💀 코드가 너무 길어짐 (12줄)
- 🔄 같은 코드가 3번 반복됨
- 😫 4번째 학생을 추가하려면? 또 12줄을 더 작성해야 함!
- 🐛 출력 형식을 바꾸려면? 3곳 모두 수정해야 함

### ✅ 함수를 사용한 경우

```c
#include <stdio.h>

// 함수 원형
void print_student(char name[], char student_id[], char grade[]);

// main 함수
int main() {
    // 함수 호출만으로 끝!
    print_student("김철수", "001", "A+");
    print_student("이영희", "002", "A");
    print_student("박민준", "003", "B+");
    
    return 0;
}

// 함수 정의
void print_student(char name[], char student_id[], char grade[]) {
    printf("=== 학생 정보 ===\n");
    printf("이름: %s\n", name);
    printf("학번: %s\n", student_id);
    printf("학점: %s\n", grade);
    printf("\n");
}
```

**장점:**
- ✨ main 함수가 매우 깔끔함 (3줄)
- 🚀 같은 함수를 여러 번 호출하기만 하면 됨
- 📝 4번째 학생 추가? `print_student("이름", "학번", "학점");` 한 줄 추가
- 🔧 출력 형식 변경? 함수 내부만 수정하면 자동으로 모든 곳에 반영됨

**비교:**
```
함수 없이: 12줄의 반복 코드
함수 사용: 1줄의 함수 호출
```

## 2️⃣ 코드의 가독성이 좋아진다

### ❌ 함수를 사용하지 않은 경우 - 모든 코드가 main에

```c
#include <stdio.h>

int main() {
    // ... 계산 코드 20줄
    // ... 검증 코드 10줄
    // ... 변환 코드 15줄
    // ... 저장 코드 20줄
    // ... 출력 코드 10줄
    // ... (총 75줄의 코드!)
    
    return 0;
}
```

**문제점:**
- 📖 코드를 읽기 어려움
- 🔍 어디서 뭘 하는지 찾기 힘듦
- 🧠 전체 흐름을 이해하기 복잡함

### ✅ 함수를 사용한 경우 - 각 기능별로 분리

```c
#include <stdio.h>

// 함수 원형
void calculate_result();
void validate_input();
void convert_data();
void save_to_file();
void display_output();

int main() {
    // 프로그램이 뭘 하는지 한눈에 보임!
    validate_input();      // 입력값 검증
    calculate_result();    // 계산 수행
    convert_data();        // 데이터 변환
    save_to_file();        // 파일에 저장
    display_output();      // 결과 표시
    
    return 0;
}

void validate_input() {
    // 검증 관련 코드 10줄
}

void calculate_result() {
    // 계산 관련 코드 20줄
}

// ... (나머지 함수들)
```

**장점:**
- 🎯 main만 봐도 프로그램이 뭘 하는지 명확함
- 🔍 특정 기능을 찾기 쉬움 (예: 데이터 변환은 convert_data 함수)
- 🧩 각 함수가 독립적으로 이해됨
- 📚 함수 이름이 코드의 설명서 역할을 함

## 3️⃣ 유지보수와 수정이 쉬워진다

### ❌ 함수를 사용하지 않은 경우

만약 프로그램에 버그가 있다면?

```c
#include <stdio.h>

int main() {
    // 버그가 어디에 있을까? 찾기 어려움!
    
    // 가능성 1: 여기 있나?
    int a = 10;
    int b = 20;
    int result = a + b;
    
    // 가능성 2: 여기 있나?
    if (result > 50) {
        printf("큽니다\n");
    }
    
    // 가능성 3: 여기 있나?
    printf("결과: %d\n", result);
    
    // 가능성 4: 여기 있나?
    // ... (계속 찾음)
    
    return 0;
}
```

### ✅ 함수를 사용한 경우

```c
#include <stdio.h>

// 함수 원형
int calculate(int a, int b);
void display_result(int value);

int main() {
    int result = calculate(10, 20);
    display_result(result);
    
    return 0;
}

// 버그가 계산에 있다면? calculate 함수만 확인!
int calculate(int a, int b) {
    return a + b;  // 여기만 보면 됨
}

// 버그가 출력에 있다면? display_result 함수만 확인!
void display_result(int value) {
    if (value > 50) {
        printf("큽니다\n");
    }
    printf("결과: %d\n", value);  // 여기만 보면 됨
}
```

**장점:**
- 🎯 버그가 어디에 있는지 쉽게 파악
- ⚡ 수정 속도가 빨라짐
- 🔒 다른 부분에 영향을 주지 않음

## 🔄 실제 예제: 반복 작업을 함수로 줄이기

### 상황: 특정 문장을 여러 번 출력

우리가 다음을 100번 출력해야 한다면?

```
배움 C언어
배움 시어너 수강자
유상현
```

### ❌ 함수를 사용하지 않은 경우

```c
#include <stdio.h>

int main() {
    // 방법 1: 직접 작성 (100번!)
    printf("배움 C언어\n");
    printf("배움 시어너 수강자\n");
    printf("유상현\n");
    printf("\n");
    
    // ... (99번 더 반복)
    
    return 0;
}
```

**코드 줄 수:** 4 × 100 = **400줄**! 😱

### ✅ 함수를 사용한 경우 (방법 1: 여러 번 호출)

```c
#include <stdio.h>

void print_message() {
    printf("배움 C언어\n");
    printf("배움 시어너 수강자\n");
    printf("유상현\n");
    printf("\n");
}

int main() {
    // 100번 호출
    for (int i = 0; i < 100; i++) {
        print_message();  // 한 줄로 끝!
    }
    
    return 0;
}
```

**코드 줄 수:** 약 **20줄** ✨

**비교:**
```
함수 없이: 400줄 😫
함수 사용: 20줄 😊
```

## 🧩 함수 사용으로 얻을 수 있는 것들

### 1. 생산성 향상

```
코드 작성 시간: 기존 50% 이하로 감소
```

### 2. 버그 감소

```
반복 코드가 적으므로 → 버그 발생 확률 ↓
```

### 3. 코드 품질 향상

```
가독성 ↑ / 유지보수성 ↑ / 확장성 ↑
```

### 4. 프로젝트 규모 확장 가능

```
함수 없이: 1000줄짜리 프로젝트도 어려움
함수 사용: 10,000줄짜리 프로젝트도 가능
```

## 🎬 완성된 프로젝트 예제

### 학점 관리 프로그램

```c
#include <stdio.h>

// 함수 원형
void enter_grades();
void calculate_average();
void display_result();

// main - 프로그램의 전체 흐름이 명확!
int main() {
    printf("=== 학점 관리 프로그램 ===\n\n");
    
    enter_grades();        // 학점 입력
    calculate_average();   // 평균 계산
    display_result();      // 결과 표시
    
    return 0;
}

// 학점 입력
void enter_grades() {
    printf("학생들의 학점을 입력하세요.\n");
    // ... 입력 관련 코드
}

// 평균 계산
void calculate_average() {
    printf("평균을 계산합니다.\n");
    // ... 계산 관련 코드
}

// 결과 표시
void display_result() {
    printf("결과를 표시합니다.\n");
    // ... 출력 관련 코드
}
```

**특징:**
- 프로그램이 뭘 하는지 한눈에 보임
- 각 단계가 함수로 구분됨
- 수정하기 쉬움
- 새로운 기능 추가가 간단함

## 📌 핵심 정리

:::note 함수의 3가지 장점
1. **반복되는 코드를 줄인다**
   - 같은 코드를 여러 번 작성할 필요 없음
   - 100번 호출할 때도 함수 정의는 1번만

2. **코드의 가독성이 좋아진다**
   - main 함수의 흐름이 명확함
   - 각 함수의 역할이 분명함

3. **유지보수가 쉬워진다**
   - 버그 수정이 간단함
   - 기능 수정 시 한 곳만 수정하면 됨
   - 다른 부분에 영향 없음
:::

## 💡 함수를 사용할 때의 원칙

```c
하나의 함수 = 하나의 기능만 담아라!
```

여러 기능을 한 함수에 넣으면 복잡해지므로, 기능별로 함수를 나누는 것이 좋습니다.
