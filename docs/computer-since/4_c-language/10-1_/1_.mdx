# 스택(Stack) 개념 기초

## 스택이란 무엇인가요?

스택(Stack)은 **차곡차곡 쌓아놓은 모습**을 의미하는 자료구조입니다.

### 일상 생활 속 스택의 예시

#### 1. 책을 쌓아놓는 모습
```
    [책 3] ← 가장 나중에 놓음
    [책 2]
    [책 1] ← 가장 먼저 놓음
   --------
```

책을 한 권씩 쌓아놓으면, 가장 위에 있는 책부터 빼게 됩니다.

#### 2. 급식실 식판 쌓기
급식실에서 식판이 쌓여있을 때를 생각해보세요:
- 가장 **위에 있는 식판**을 먼저 집습니다
- 아래에 있는 식판을 빼려면 위의 모든 식판을 들어야 합니다
- 따라서 **가장 위의 것부터** 하나씩 빼게 됩니다

```
 [식판 4] ← 이것을 먼저 가져감
 [식판 3]
 [식판 2]
 [식판 1] ← 이것은 나중에 가져감
==========
```

## 스택의 핵심 개념: 후입선출 (LIFO)

### LIFO란?
**L**ast **I**n **F**irst **O**ut (후입선출)
- **마지막에 들어간 것이 가장 먼저 나온다**
- **먼저 들어간 것은 나중에 나온다**

### 스택의 동작 방식

```
1. 처음 상태 (비어있음)
   |     |
   |     |
   |     |
   -------

2. 데이터 3 삽입 (PUSH)
   |     |
   |     |
   |  3  | ← 가장 먼저 들어옴
   -------

3. 데이터 4 삽입 (PUSH)
   |     |
   |  4  | ← 3 위에 쌓임
   |  3  |
   -------

4. 데이터 1 삽입 (PUSH)
   |  1  | ← 4 위에 쌓임
   |  4  |
   |  3  |
   -------

5. 데이터 8 삽입 (PUSH)
   |  8  | ← TOP (가장 위)
   |  1  |
   |  4  |
   |  3  |
   -------

6. 데이터 꺼내기 (POP)
   |  1  | ← 8이 나가고 TOP이 여기로 이동
   |  4  |
   |  3  |
   -------
   8이 나옴 ← 가장 마지막에 들어간 8이 먼저 나옴!
```

## 스택의 주요 용어

### 1. PUSH (푸시)
**데이터를 스택에 넣는 작업**
- 영어로 "밀어넣다"는 의미
- 스택의 맨 위에 새로운 데이터를 추가합니다

```c
// PUSH 작업의 개념
stack[top] = data;  // 현재 top 위치에 데이터 저장
top = top + 1;      // top을 하나 증가 (다음 위치로 이동)
```

### 2. POP (팝)
**데이터를 스택에서 꺼내는 작업**
- 영어로 "튀어나오다"는 의미
- 스택의 맨 위에 있는 데이터를 제거하고 반환합니다

```c
// POP 작업의 개념
top = top - 1;            // top을 하나 감소 (이전 위치로 이동)
data = stack[top];        // 그 위치의 데이터를 꺼냄
```

### 3. TOP (탑)
**스택의 맨 꼭대기를 가리키는 위치**
- "제일 꼭대기"라는 뜻
- **다음에 데이터가 들어갈 위치**를 가리킵니다
- PUSH를 하면 top이 **증가**합니다
- POP을 하면 top이 **감소**합니다

```
현재 스택 상태:
   |  8  | ← TOP이 여기를 가리킴 (top = 4)
   |  1  |
   |  4  |
   |  3  |
   -------
```

## 스택이 사용되는 곳

### 1. 웹브라우저의 뒤로 가기
웹브라우저에서 페이지를 방문할 때마다 스택에 저장됩니다:

```
페이지 A 방문 → 페이지 B 방문 → 페이지 C 방문

스택 상태:
   [페이지 C] ← 현재 페이지
   [페이지 B]
   [페이지 A]

뒤로 가기 버튼 클릭 시:
   [페이지 B] ← C가 POP 되고 B로 이동
   [페이지 A]
```

### 2. 실행 취소 (Undo) 기능
문서 편집 프로그램에서 작업 내역을 스택에 저장:

```
작업1 수행 → 작업2 수행 → 작업3 수행

Ctrl+Z (실행 취소) 누르면:
가장 마지막 작업(작업3)부터 취소됩니다
```

### 3. 함수 호출
프로그램에서 함수를 호출할 때 스택을 사용합니다 (다음 강의에서 자세히 배웁니다)

## 스택의 특징 정리

### ✅ 장점
- 구조가 **단순**하여 이해하기 쉽습니다
- 데이터 삽입/삭제가 **빠릅니다** (맨 위에서만 일어나므로)
- 구현이 **간단**합니다

### ⚠️ 제약사항
- 중간에 있는 데이터에 **직접 접근할 수 없습니다**
- 맨 위의 데이터만 접근 가능합니다
- 크기가 고정되어 있으면 **오버플로우**(스택이 꽉 참)가 발생할 수 있습니다

---

## 핵심 요약

```
스택(Stack) = 후입선출(LIFO) 자료구조

핵심 동작:
1. PUSH: 데이터를 스택에 넣기 (위로 쌓기)
2. POP:  데이터를 스택에서 빼기 (위에서 꺼내기)
3. TOP:  스택의 맨 위 위치 가리키기

특징:
- 한쪽 끝에서만 삽입/삭제가 일어남
- 가장 나중에 들어간 것이 가장 먼저 나옴
- 식판 쌓기, 웹브라우저 뒤로가기 등에 사용
```
