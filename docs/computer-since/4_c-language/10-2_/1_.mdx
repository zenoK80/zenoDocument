# 메모리와 변수의 기초

## 학습 목표
- 변수가 메모리에 저장되는 방식을 이해합니다
- 메모리 주소의 개념을 학습합니다
- 다양한 자료형의 메모리 할당 방식을 파악합니다

## 1. 메모리란 무엇인가요?

컴퓨터의 메모리는 데이터를 저장하는 공간입니다. 메모리는 **1바이트(byte) 단위**로 나뉘어져 있으며, 각 바이트마다 고유한 **주소(address)**가 있습니다.

### 메모리의 특징
```
메모리 구조:
┌─────────┬─────────┬─────────┬─────────┐
│ 0x01    │ 0x02    │ 0x03    │ 0x04    │  ← 주소 (16진수)
├─────────┼─────────┼─────────┼─────────┤
│ 1바이트 │ 1바이트 │ 1바이트 │ 1바이트 │  ← 각 칸은 1바이트
└─────────┴─────────┴─────────┴─────────┘
```

:::info 왜 16진수를 사용하나요?
메모리 주소는 **16진수(0x로 시작)**로 표현됩니다. 이는 주소를 간결하게 표현하기 위함입니다.
- 10진수: 255 → 16진수: 0xFF
- 10진수: 1024 → 16진수: 0x400
:::

## 2. 정수형 변수의 메모리 저장

### 예제 코드
```c
int n = 10;  // 정수형 변수 선언 및 초기화
```

### 코드 해설 (한 줄씩)
```c
int n = 10;
// int: 정수형 자료형 (4바이트 크기)
// n: 변수의 이름
// = 10: 변수 n에 10이라는 값을 저장
```

### 메모리에 저장되는 모습

```
주소:      0x05    0x06    0x07    0x08
        ┌────────┬────────┬────────┬────────┐
값:     │        10이라는 값이 저장됨        │
        └────────┴────────┴────────┴────────┘
         ←──────── 4바이트 (int 크기) ────────→
         
변수 n은 0x05 주소에서 시작합니다.
```

### 핵심 개념 정리

1. **int형 변수는 4바이트를 차지**합니다
2. **시작 주소만 기억**합니다 (예: 0x05)
3. 시작 주소부터 자료형 크기만큼 공간을 할당합니다

:::tip 메모리 할당의 핵심
- 변수를 만들면 → 메모리의 어딘가에 공간이 할당됩니다
- 시작 주소만 알면 → 자료형 크기로 전체 공간을 파악할 수 있습니다
- 예: 0x05에서 시작 + 4바이트 = 0x05~0x08 영역 사용
:::

## 3. 실수형 변수의 메모리 저장

### 예제 코드
```c
double d = 3.141592;  // 실수형 변수 선언
```

### 코드 해설
```c
double d = 3.141592;
// double: 실수형 자료형 (8바이트 크기)
// d: 변수의 이름
// = 3.141592: 변수 d에 원주율 값을 저장
```

### 메모리에 저장되는 모습

```
주소:      0x05    0x06    0x07    0x08    0x09    0x0A    0x0B    0x0C
        ┌────────┬────────┬────────┬────────┬────────┬────────┬────────┬────────┐
값:     │                     3.141592가 저장됨                            │
        └────────┴────────┴────────┴────────┴────────┴────────┴────────┴────────┘
         ←────────────────────── 8바이트 (double 크기) ──────────────────────→
```

### 자료형별 크기 정리

| 자료형 | 크기 | 저장 가능한 값 |
|--------|------|----------------|
| `char` | 1바이트 | 문자 1개 |
| `int` | 4바이트 | 정수 |
| `double` | 8바이트 | 실수 |

## 4. 문자형 배열의 메모리 저장

### 예제 코드
```c
char str[5] = "abc";  // 문자형 배열 선언
```

### 코드 해설
```c
char str[5] = "abc";
// char: 문자형 자료형 (1바이트)
// str[5]: 크기가 5인 배열 선언
// = "abc": 문자열 "abc"로 초기화
```

### 메모리에 저장되는 모습

```
배열:      str[0]  str[1]  str[2]  str[3]  str[4]
주소:      0x01    0x02    0x03    0x04    0x05
        ┌────────┬────────┬────────┬────────┬────────┐
값:     │   'a'  │   'b'  │   'c'  │  '\0'  │  '\0'  │
        └────────┴────────┴────────┴────────┴────────┘
         1바이트  1바이트  1바이트  1바이트  1바이트
```

### 중요 포인트

:::warning 널 문자(Null Character)
- 문자열의 끝을 표시하기 위해 **'\0' (널 문자)**가 자동으로 추가됩니다
- "abc"는 3글자지만, 배열 크기는 **최소 4 이상**이어야 합니다
- 널 문자를 위한 공간이 필요하기 때문입니다
:::

```c
// 올바른 예시
char str1[4] = "abc";   // 3글자 + 널문자 1개 = 4바이트 ✅

// 잘못된 예시
char str2[3] = "abc";   // 널문자 공간 부족 ❌
```

## 5. 변수 이름과 메모리 주소의 관계

### 왜 변수 이름을 사용하나요?

```c
// 주소를 직접 사용한다면? (이렇게 하지 않습니다)
0x05 = 10;        // ❌ 매우 불편하고 어려움!

// 변수 이름을 사용하면 (실제로 사용하는 방법)
int n = 10;       // ✅ 간단하고 이해하기 쉬움!
```

### 변수 이름의 장점

1. **기억하기 쉽습니다**: "n" vs "0x05번지"
2. **의미를 표현할 수 있습니다**: "age", "score" 등
3. **주소를 몰라도 됩니다**: 컴파일러가 자동으로 연결해줍니다

```c
int age = 20;      // "나이"라는 의미를 담은 변수명
int score = 95;    // "점수"라는 의미를 담은 변수명

// 컴퓨터는 내부적으로 주소로 관리하지만,
// 우리는 의미 있는 이름으로 사용할 수 있습니다!
```

## 6. 메모리 할당의 특징

### 실행할 때마다 주소가 바뀝니다

```c
// 첫 번째 실행
int n = 10;  // n의 주소: 0x1000

// 두 번째 실행
int n = 10;  // n의 주소: 0x2500 (다를 수 있음!)
```

:::info 왜 주소가 바뀌나요?
- 다른 프로그램이 메모리를 사용하고 있을 수 있습니다
- 운영체제가 **사용 가능한 메모리 공간**을 찾아서 할당합니다
- 매번 실행할 때마다 **사용 가능한 공간이 다르기** 때문입니다
:::

## 학습 정리

### 핵심 개념 체크리스트

✅ **메모리는 1바이트 단위로 구성**되어 있습니다  
✅ **각 바이트는 고유한 주소**를 가집니다 (16진수로 표현)  
✅ **변수는 시작 주소 + 자료형 크기**만큼 메모리를 차지합니다  
✅ **변수 이름은 메모리 주소와 연결**되어 있습니다  
✅ **실행할 때마다 메모리 주소는 달라질 수** 있습니다  

### 자료형별 크기 요약

```c
char    c;     // 1바이트
int     n;     // 4바이트
double  d;     // 8바이트
char    s[10]; // 1바이트 × 10 = 10바이트
```
