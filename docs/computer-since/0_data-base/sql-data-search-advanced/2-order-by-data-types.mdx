# 📊 데이터 타입별 정렬 기준 — 숫자, 문자, 날짜, NULL 완전 정복

SQL을 처음 배우면 데이터를 화면에 출력하는 것만으로도 벅차게 느껴집니다. 그런데 실무에서는 단순히 데이터를 꺼내는 것에서 끝나지 않습니다. **"급여가 많은 사람 순으로 보여줘"**, **"가입일이 가장 오래된 고객부터 정렬해줘"**, **"이름 가나다순으로 출력해줘"** 처럼 데이터를 **원하는 순서로 정렬**해서 보고 싶은 경우가 훨씬 더 많습니다.

이번 문서에서는 SQL의 `ORDER BY` 구문을 중심으로, **숫자·문자·날짜·NULL** 각 데이터 타입이 정렬될 때 어떤 기준으로 순서가 결정되는지를 초보자도 이해할 수 있도록 아주 꼼꼼하게 설명합니다. 또한 정렬 기준을 여러 개 지정하는 **다중 정렬** 방법까지 함께 다룹니다.

---

## 🔍 ORDER BY란 무엇인가?

### ORDER BY의 기본 개념

`ORDER BY`는 SQL 문장으로 조회된 데이터를 **특정 컬럼(속성)을 기준으로 정렬**해서 출력하는 명령어입니다. 마치 도서관에서 책을 제목 가나다순으로 꽂아두거나, 도서 번호 순서대로 정리하는 것과 같습니다. 데이터 자체는 바뀌지 않지만, 우리가 **보는 순서**를 바꿔주는 것이죠.

> **ORDER BY = 조회 결과의 출력 순서를 사용자가 원하는 대로 설정하는 명령어**

예를 들어 쇼핑몰을 운영하는 담당자가 고객 데이터를 분석하려 한다고 생각해봅시다.

- 우리 사이트에서 **구매 금액이 가장 많은 고객**부터 순서대로 보고 싶다
- 상품 목록을 **가격이 저렴한 것부터** 비싼 순서로 보여주고 싶다
- 신규 가입 고객을 **가입일이 최근인 사람**부터 정렬해서 보고 싶다

이 모든 요구사항이 `ORDER BY` 하나로 해결됩니다.

### ORDER BY 기본 문법

```sql
SELECT 컬럼1, 컬럼2, ...
FROM 테이블명
WHERE 조건
ORDER BY 정렬기준_컬럼 [ASC | DESC];
```

- `ASC` (Ascending, 오름차순): 작은 값 → 큰 값 순서. **생략하면 기본값으로 적용됩니다.**
- `DESC` (Descending, 내림차순): 큰 값 → 작은 값 순서. 반드시 명시해야 합니다.

:::info 오름차순과 내림차순이 헷갈릴 때
- **오름차순(ASC)**: 계단을 올라가듯 1 → 2 → 3 → 4 → 5 처럼 값이 점점 커짐
- **내림차순(DESC)**: 계단을 내려오듯 5 → 4 → 3 → 2 → 1 처럼 값이 점점 작아짐

"ascending"은 "상승하다"는 영어 단어에서 왔고, "descending"은 "하강하다"에서 왔다고 기억하면 쉽습니다.
:::

## 🔢 숫자 타입의 정렬 기준

### 숫자 정렬은 왜 필요한가?

숫자 데이터는 우리가 가장 직관적으로 이해할 수 있는 정렬 기준입니다. **급여가 많은 사람부터**, **점수가 낮은 사람부터**, **재고 수량이 많은 상품부터** 등 실무에서 가장 빈번하게 사용되는 정렬 방식입니다.

숫자 타입(`INT`, `FLOAT`, `DECIMAL` 등)을 `ORDER BY`로 정렬하면, 컴퓨터는 우리가 수학 시간에 배운 것과 똑같이 수의 크기를 비교해서 순서를 결정합니다.

### 숫자 정렬의 동작 방식

숫자 오름차순 정렬은 **가장 작은 값이 먼저** 출력됩니다. 마치 달리기 경기에서 1등(가장 빠른 사람)이 맨 앞에 서는 것처럼, 가장 작은 숫자가 맨 앞으로 옵니다.

```sql
-- ✅ 올바른 예시: 2번 부서 사원을 급여 오름차순으로 정렬
SELECT emp_name,   -- 사원 이름 출력
       title,      -- 직급 출력
       salary      -- 급여 출력
FROM employee      -- employee(사원) 테이블에서
WHERE dno = 2      -- 부서 번호가 2번인 사원만 필터링
ORDER BY salary;   -- 급여를 기준으로 오름차순 정렬 (ASC 생략 가능)
```

위 쿼리의 결과는 이렇게 됩니다.

| emp_name | title | salary |
|---|---|---|
| 김대리 | 대리 | 2500000 |
| 이과장 | 과장 | 3000000 |
| 박사장 | 사장 | 5000000 |

급여가 250만원 → 300만원 → 500만원으로 **점점 커지는 오름차순** 정렬입니다.

```sql
-- ✅ 올바른 예시: 급여 내림차순 정렬 (급여 많은 사람 순)
SELECT emp_name,
       title,
       salary
FROM employee
WHERE dno = 2
ORDER BY salary DESC;  -- DESC를 명시해야 내림차순 적용
```

| emp_name | title | salary |
|---|---|---|
| 박사장 | 사장 | 5000000 |
| 이과장 | 과장 | 3000000 |
| 김대리 | 대리 | 2500000 |

이번엔 500만원 → 300만원 → 250만원으로 **점점 작아지는 내림차순** 정렬입니다.

:::warning 내림차순 정렬 시 DESC 생략 금지
`ORDER BY salary`처럼 아무것도 안 쓰면 **항상 오름차순**이 적용됩니다. 내림차순을 원할 때는 반드시 `DESC`를 명시해야 합니다. 이 실수는 생각보다 자주 발생하므로 주의하세요.
:::

## 🔤 문자 타입의 정렬 기준

### 문자 정렬은 어떻게 이루어지는가?

문자 데이터(`CHAR`, `VARCHAR` 등)의 정렬은 숫자처럼 직관적이지 않아서 처음에는 헷갈릴 수 있습니다. 컴퓨터는 문자를 직접 비교하지 않고, 문자를 **아스키 코드(ASCII Code)** 라는 숫자 값으로 변환한 다음 그 숫자를 비교해서 정렬합니다.

:::info 아스키 코드(ASCII Code)란?
아스키 코드는 각 문자에 고유한 숫자를 부여한 국제 표준 규칙입니다. 예를 들어 영문 대문자 'A'는 65, 'B'는 66, 'Z'는 90이고, 소문자 'a'는 97, 'b'는 98 같은 식입니다. 컴퓨터는 모든 문자를 이 숫자로 바꿔서 처리하기 때문에 문자 정렬도 결국 숫자 정렬과 같은 원리로 동작합니다.
:::

### 문자 정렬의 실제 규칙

문자 오름차순 정렬의 일반적인 순서는 다음과 같습니다.

| 순서 | 문자 종류 | 예시 |
|---|---|---|
| 1순위 | 숫자 문자 | '0', '1', '2', ... '9' |
| 2순위 | 영문 대문자 | 'A', 'B', 'C', ... 'Z' |
| 3순위 | 영문 소문자 | 'a', 'b', 'c', ... 'z' |
| 4순위 | 한글 | '가', '나', '다', ... |

```sql
-- ✅ 올바른 예시: 사원 이름을 알파벳 오름차순으로 정렬
SELECT emp_name,   -- 사원 이름
       title       -- 직급
FROM employee      -- employee 테이블에서
ORDER BY emp_name; -- 이름을 기준으로 오름차순 정렬 (아스키 코드 기준)
```

예를 들어 이름이 'Alice', 'Bob', 'Charlie', 'David'인 사원이 있다면, 오름차순 정렬 시 'Alice' → 'Bob' → 'Charlie' → 'David' 순서로 출력됩니다. 'A'의 아스키 코드가 65, 'B'가 66, 'C'가 67, 'D'가 68이기 때문입니다.

:::note 한글 정렬은 유니코드 기준
현대 데이터베이스에서 한글은 유니코드(Unicode) 값을 기준으로 정렬됩니다. 한글의 경우 가나다 순서대로 유니코드 값이 커지기 때문에, 한글 데이터를 오름차순 정렬하면 '가나다라...' 순서로 출력됩니다. 이는 우리가 사전에서 단어를 찾을 때와 동일한 순서입니다.
:::

## 📅 날짜 타입의 정렬 기준

### 날짜 정렬은 왜 중요한가?

날짜 데이터(`DATE`, `DATETIME`, `TIMESTAMP` 등)의 정렬은 **시간순서**를 다룰 때 필수입니다. 예를 들어 "가장 최근에 가입한 회원부터 보여줘", "가장 오래된 주문 내역부터 정렬해줘" 같은 요구사항이 여기에 해당합니다. 마치 일기를 날짜 순서대로 정리하는 것처럼, 날짜 타입은 시간의 흐름에 따라 정렬됩니다.

### 날짜 정렬의 동작 방식

날짜 타입을 오름차순 정렬하면 **날짜가 가장 빠른(오래된) 것**이 먼저 출력됩니다.

- **오름차순(ASC)**: 과거 → 현재 순서 (예: 2020-01-01 → 2021-06-15 → 2024-12-31)
- **내림차순(DESC)**: 현재 → 과거 순서 (예: 2024-12-31 → 2021-06-15 → 2020-01-01)

```sql
-- ✅ 올바른 예시: 직원을 입사일 기준 오름차순 정렬 (오래된 직원부터)
SELECT emp_name,    -- 사원 이름
       hire_date,   -- 입사일
       salary       -- 급여
FROM employee       -- employee 테이블에서
ORDER BY hire_date; -- 입사일 오름차순 (가장 오래 근무한 사람 먼저)
```

```sql
-- ✅ 올바른 예시: 가장 최근에 입사한 직원부터 정렬
SELECT emp_name,
       hire_date,
       salary
FROM employee
ORDER BY hire_date DESC; -- 입사일 내림차순 (최근 입사자 먼저)
```

:::warning DBMS마다 날짜 처리 방식이 다를 수 있음
날짜 타입의 정렬 기준과 내부 처리 방식은 **Oracle, MySQL, MS-SQL** 등 사용하는 데이터베이스 소프트웨어에 따라 세부적인 동작이 다를 수 있습니다. 실무에서 날짜 정렬이 예상과 다르게 동작한다면, 반드시 사용 중인 DBMS의 공식 매뉴얼을 확인하여 정확한 동작 방식을 파악하는 것이 중요합니다.
:::

## 🚫 NULL 값의 정렬 기준

### NULL이란 무엇인가?

**NULL**은 데이터베이스에서 "값이 없음" 또는 "알 수 없음"을 나타내는 특별한 상태입니다. 숫자 0이나 빈 문자열('')과는 완전히 다른 개념입니다. 마치 시험을 아예 응시하지 않은 학생의 점수칸이 비어있는 것처럼, NULL은 데이터 자체가 존재하지 않는 상태입니다.

그런데 정렬을 할 때 NULL 값은 어디에 위치해야 할까요? NULL은 숫자도 아니고, 문자도 아니고, 날짜도 아니기 때문에 일반적인 비교 방법으로는 위치를 결정할 수 없습니다. 그래서 데이터베이스는 **NULL에 대한 별도의 정렬 규칙**을 갖고 있습니다.

### NULL 정렬 규칙

일반적으로 SQL에서 NULL의 정렬 위치는 다음과 같습니다.

| 정렬 방향 | NULL의 위치 | 설명 |
|---|---|---|
| 오름차순 (ASC) | 맨 마지막 | 일반 값이 먼저 출력되고, NULL이 맨 뒤에 옴 |
| 내림차순 (DESC) | 맨 처음 | NULL이 먼저 출력되고, 이후 일반 값이 내림차순으로 옴 |

예를 들어 직원 테이블에서 상사(manager) 정보가 없는 직원(최고 직책자)의 `manager` 컬럼이 NULL이라고 가정해봅시다.

```sql
-- ✅ 오름차순 정렬 시 NULL은 맨 마지막에 위치
SELECT emp_name,    -- 사원 이름
       manager,     -- 상사 번호 (없으면 NULL)
       salary       -- 급여
FROM employee
ORDER BY manager ASC; -- manager 오름차순: 일반 값 먼저, NULL은 마지막
```

결과 예시:
| emp_name | manager | salary |
|---|---|---|
| 김대리 | 1001 | 2500000 |
| 이과장 | 1001 | 3000000 |
| 최부장 | 1002 | 4000000 |
| 박사장 | NULL | 5000000 |

NULL이 맨 마지막에 위치하는 것을 확인할 수 있습니다.

```sql
-- ✅ 내림차순 정렬 시 NULL은 맨 처음에 위치
SELECT emp_name,
       manager,
       salary
FROM employee
ORDER BY manager DESC; -- manager 내림차순: NULL이 먼저, 이후 큰 값부터
```

결과 예시:
| emp_name | manager | salary |
|---|---|---|
| 박사장 | NULL | 5000000 |
| 최부장 | 1002 | 4000000 |
| 김대리 | 1001 | 2500000 |
| 이과장 | 1001 | 2500000 |

NULL이 맨 처음에 오고, 이후 숫자 값이 큰 것부터 내림차순으로 출력됩니다.

:::tip NULL 위치를 직관적으로 기억하는 방법
- **오름차순(ASC)**: 1, 2, 3, ... 증가하다가 → NULL이 마지막에 (값이 없으니 제일 뒤로)
- **내림차순(DESC)**: NULL이 맨 앞에 → 9, 8, 7, ... 감소 순서

"오름차순에서 NULL은 꼴등" 이라고 외워두면 편합니다!
:::

:::warning DBMS별 NULL 정렬 차이
Oracle은 기본적으로 오름차순에서 NULL을 **마지막**에 배치하고, 내림차순에서 NULL을 **처음**에 배치합니다. MySQL도 비슷하게 동작하지만 세부 동작이 다를 수 있습니다. 실무에서 NULL 정렬이 중요한 경우, 사용 중인 DBMS의 공식 문서를 반드시 확인하세요.
:::

## 🔀 다중 정렬 기준 (여러 컬럼으로 정렬하기)

### 다중 정렬이 왜 필요한가?

단일 컬럼으로 정렬할 때 **기준 값이 동일한 경우**가 생길 수 있습니다. 예를 들어 나이를 기준으로 정렬했는데 나이가 같은 사람이 여러 명이라면, 그 사람들 사이의 순서는 어떻게 결정해야 할까요? 이런 상황에서 **두 번째, 세 번째 정렬 기준**을 추가적으로 지정할 수 있습니다.

마치 도서관에서 책을 정리할 때 "먼저 분야별로 나누고, 같은 분야 안에서는 제목 가나다순으로 정리하는 것"처럼, 여러 단계의 정렬 기준을 계층적으로 적용하는 것이 다중 정렬입니다.

### 다중 정렬 문법

```sql
ORDER BY 첫번째_기준 [ASC|DESC], 두번째_기준 [ASC|DESC], ...
```

첫 번째 기준으로 먼저 정렬하고, **첫 번째 기준 값이 같은 경우에만** 두 번째 기준을 적용합니다. 두 번째 기준도 같으면 세 번째 기준을 적용하는 방식으로 작동합니다.

### 다중 정렬 실제 예시

**"급여가 300만원 이상인 직원을 나이 오름차순으로 정렬하고, 나이가 같은 경우에는 급여 내림차순으로 정렬하여 이름을 출력하라"**

```sql
-- ✅ 올바른 예시: 나이 오름차순 + 동일 나이 내에서 급여 내림차순
SELECT emp_name      -- 직원 이름을 출력
FROM employee        -- employee 테이블에서
WHERE salary >= 3000000   -- 급여 300만원 이상인 직원만 필터링
ORDER BY age ASC,         -- 1순위: 나이 오름차순 (어린 사람 먼저)
         salary DESC;     -- 2순위: 나이가 같으면 급여 많은 사람 먼저
```

결과 예시:

| emp_name | age | salary |
|---|---|---|
| 김신입 | 28 | 3500000 |
| 이주임 | 31 | 5000000 |
| 박대리 | 31 | 4500000 |
| 최주임 | 31 | 3000000 |
| 정과장 | 35 | 4000000 |

- 나이가 28 → 31 → 31 → 31 → 35로 오름차순 정렬됨
- 나이가 31로 같은 사람들(이주임, 박대리, 최주임) 사이에서는 급여가 500만원 → 450만원 → 300만원으로 내림차순 정렬됨

:::note 다중 정렬 시 각 기준마다 오름차순/내림차순 독립 지정 가능
`ORDER BY age ASC, salary DESC`처럼 첫 번째 기준은 오름차순, 두 번째 기준은 내림차순으로 **각각 다르게** 지정할 수 있습니다. 쉼표(`,`)로 구분하여 원하는 만큼 기준을 추가할 수 있습니다.
:::

### 잘못된 예시와 올바른 예시 비교

```sql
-- ❌ 잘못된 예시: 정렬 기준 순서 오류
-- 급여가 같은 사람 중에서 나이로 정렬하고 싶었지만
-- 실제로는 급여 우선 정렬이 되어버림
SELECT emp_name
FROM employee
WHERE salary >= 3000000
ORDER BY salary DESC, age ASC;  -- 급여가 1순위가 되어버림

-- ✅ 올바른 예시: 의도에 맞게 나이를 1순위로 지정
SELECT emp_name          -- 직원 이름 출력
FROM employee            -- employee 테이블에서
WHERE salary >= 3000000  -- 급여 300만원 이상 필터
ORDER BY age ASC,        -- 1순위: 나이 오름차순
         salary DESC;    -- 2순위: 급여 내림차순 (나이 같을 때만 적용)
```

> **핵심 포인트**: `ORDER BY`에서 나열된 컬럼 순서 자체가 정렬 우선순위를 결정합니다. **왼쪽에 쓴 것이 1순위**, 그다음이 2순위, 그다음이 3순위입니다.

## 📋 실전 종합 예제 — employee 테이블로 연습하기

이제까지 배운 내용을 실제 테이블을 활용한 종합 예제로 정리해봅시다.

아래와 같은 `employee`(사원) 테이블이 있다고 가정합니다.

| emp_no | emp_name | title | salary | dno | hire_date |
|---|---|---|---|---|---|
| 1001 | 홍길동 | 사장 | 5000000 | 1 | 2010-03-15 |
| 1002 | 김영희 | 부장 | 4000000 | 2 | 2012-07-20 |
| 1003 | 이철수 | 과장 | 3000000 | 2 | 2015-01-05 |
| 1004 | 박민수 | 대리 | 2500000 | 3 | 2019-09-10 |
| 1005 | 최지은 | 사원 | 2000000 | 1 | 2022-04-25 |
| 1006 | 정호준 | 대리 | NULL | 3 | 2023-11-30 |

### 예제 1: 급여 오름차순 정렬

```sql
-- ✅ 전체 사원을 급여 오름차순으로 정렬
SELECT emp_name,    -- 사원 이름
       title,       -- 직급
       salary       -- 급여
FROM employee       -- employee 테이블에서
ORDER BY salary;    -- 급여 오름차순 (낮은 급여부터)
-- 결과: NULL 값은 맨 마지막에 출력됨
```

### 예제 2: 입사일 내림차순 (최근 입사자부터)

```sql
-- ✅ 가장 최근에 입사한 사원부터 정렬
SELECT emp_name,         -- 사원 이름
       hire_date,        -- 입사일
       title             -- 직급
FROM employee            -- employee 테이블에서
ORDER BY hire_date DESC; -- 입사일 내림차순 (최근 날짜가 먼저)
```

### 예제 3: 부서별 정렬 후 같은 부서 내에서 급여 내림차순

```sql
-- ✅ 부서 번호 오름차순, 같은 부서 내에서는 급여 내림차순
SELECT dno,           -- 부서 번호
       emp_name,      -- 사원 이름
       salary         -- 급여
FROM employee         -- employee 테이블에서
ORDER BY dno ASC,     -- 1순위: 부서 번호 오름차순 (1번 부서 → 2번 → 3번)

```

## ⚠️ 주의사항 및 실수 방지 팁

### 자주 하는 실수 정리

:::danger DESC를 빠트리는 실수
내림차순이 필요한 상황에서 `DESC`를 빠트리면 조용히 오름차순으로 정렬됩니다. 에러 메시지도 없기 때문에 결과를 보고 나서야 실수를 알아채는 경우가 많습니다. **내림차순이 필요하면 반드시 `DESC`를 명시**하는 습관을 들이세요.

```sql
-- ❌ 잘못된 예시: DESC 누락으로 오름차순이 되어버림
ORDER BY salary  -- 급여 내림차순을 원했지만 오름차순으로 실행됨

-- ✅ 올바른 예시
ORDER BY salary DESC  -- 급여 내림차순 명확히 지정
```
:::

:::warning WHERE와 ORDER BY의 순서 혼동
SQL 문법에서 `WHERE`는 항상 `ORDER BY`보다 먼저 작성해야 합니다. 처리 순서가 헷갈리면 쿼리 자체가 에러가 납니다.

```sql
-- ❌ 잘못된 예시: ORDER BY가 WHERE보다 앞에 작성됨
SELECT emp_name
FROM employee
ORDER BY salary DESC  -- 순서 오류!
WHERE dno = 2;

-- ✅ 올바른 예시: WHERE → ORDER BY 순서 준수
SELECT emp_name         -- 출력할 컬럼 지정
FROM employee           -- 테이블 지정
WHERE dno = 2           -- 조건 필터 (ORDER BY보다 먼저!)
ORDER BY salary DESC;   -- 정렬 기준 (맨 마지막에!)
```
:::

:::tip SELECT 절의 별칭(Alias)으로도 정렬 가능
`SELECT` 절에서 컬럼에 별칭을 붙였다면, `ORDER BY`에서 그 별칭을 사용하여 정렬할 수 있습니다. 쿼리가 길어질 때 가독성을 높이는 데 유용합니다.

```sql
-- ✅ 별칭을 사용한 정렬 예시
SELECT emp_name,
       salary * 1.1 AS adjusted_salary  -- 급여에 1.1 곱한 값에 별칭 부여
FROM employee
ORDER BY adjusted_salary DESC;          -- 별칭으로 정렬 가능
```
:::

### 데이터 타입별 정렬 기준 한눈에 비교

| 데이터 타입 | 오름차순(ASC) | 내림차순(DESC) | NULL 위치 |
|---|---|---|---|
| 숫자 (INT, FLOAT) | 작은 값 → 큰 값 | 큰 값 → 작은 값 | ASC: 마지막 / DESC: 처음 |
| 문자 (CHAR, VARCHAR) | 아스키/유니코드 작은 값 → 큰 값 | 아스키/유니코드 큰 값 → 작은 값 | ASC: 마지막 / DESC: 처음 |
| 날짜 (DATE, DATETIME) | 오래된 날짜 → 최근 날짜 | 최근 날짜 → 오래된 날짜 | ASC: 마지막 / DESC: 처음 |
| NULL | — | — | ASC: 맨 마지막 / DESC: 맨 처음 |

## 💡 핵심 정리

- **`ORDER BY`** 는 SQL 조회 결과를 특정 컬럼 기준으로 정렬해서 출력하는 명령어이며, `SELECT` 문의 맨 마지막에 작성한다.
- **오름차순(ASC)** 은 작은 값 → 큰 값 순서이며, 정렬 방향을 생략하면 기본적으로 오름차순이 적용된다.
- **내림차순(DESC)** 은 큰 값 → 작은 값 순서이며, 반드시 `DESC` 키워드를 명시해야 한다.
- **숫자 타입**은 수의 크기를 기준으로 정렬되며, 작은 수가 오름차순에서 먼저 출력된다.
- **문자 타입**은 아스키 코드(ASCII) 또는 유니코드 값을 기준으로 정렬되며, 한글은 가나다 순서로 정렬된다.
- **날짜 타입**은 시간의 흐름 기준으로 정렬되며, 오름차순 시 오래된 날짜가 먼저 출력된다. DBMS에 따라 세부 동작이 다를 수 있으므로 매뉴얼 확인이 필요하다.
- **NULL 값**은 오름차순 정렬 시 맨 마지막에, 내림차순 정렬 시 맨 처음에 위치한다.
- **다중 정렬**은 `ORDER BY 컬럼1, 컬럼2` 형태로 쉼표로 구분하여 여러 기준을 지정하며, 왼쪽 컬럼일수록 우선순위가 높다.
- 첫 번째 정렬 기준 값이 **동일한 경우에만** 두 번째 기준이 적용되는 방식으로 다중 정렬이 동작한다.
- `WHERE` 절은 항상 `ORDER BY`보다 먼저 작성해야 하며, 순서를 바꾸면 SQL 문법 오류가 발생한다.
