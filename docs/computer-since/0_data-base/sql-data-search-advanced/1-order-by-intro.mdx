# 📊 ORDER BY 개념과 정렬 방식 — SQL 데이터 검색 응용

지금까지 우리는 `SELECT`, `FROM`, `WHERE`라는 SQL의 기본 3총사를 배웠습니다. 이제부터는 **검색 결과를 더욱 의미 있게 가공하고 출력하는 방법**을 배울 차례입니다. 이번 문서에서는 그 첫 번째 주제인 `ORDER BY`(정렬)에 대해 아주 자세하게 다뤄보겠습니다.

> 💡 실무에서는 단순히 데이터를 가져오는 것보다, **어떤 순서로, 어떤 기준으로** 데이터를 보여주느냐가 훨씬 더 중요합니다. `ORDER BY`는 바로 그 역할을 담당합니다.

---

## 🧭 ORDER BY란 무엇인가?

### ORDER BY의 기본 개념

`ORDER BY`는 영어 그대로 해석하면 **"~순서로 정렬하라"** 는 뜻입니다. SQL에서 `SELECT` 명령으로 조회된 데이터를 **특정 컬럼(열, 속성)을 기준으로 정렬**하여 출력할 때 사용하는 구문입니다.

마치 도서관에서 책을 찾을 때 "제목 가나다순", "출판연도 오래된 순", "저자 이름순"으로 정렬된 목록을 보는 것과 같습니다. 데이터베이스도 마찬가지로, 우리가 원하는 기준에 따라 결과를 가지런히 정렬해서 보여주는 기능이 바로 `ORDER BY`입니다.

### 왜 ORDER BY가 필요한가? 🤔

데이터베이스에 저장된 데이터는 기본적으로 **입력된 순서대로 저장**됩니다. 하지만 실무에서는 단순히 입력 순서대로 데이터를 보는 경우가 거의 없습니다. 예를 들어볼까요?

- 🛒 **쇼핑몰 운영자**: "우리 사이트에서 구매 금액이 가장 많은 고객부터 보고 싶다"
- 📚 **학교 선생님**: "학생들의 성적을 높은 점수부터 낮은 점수 순으로 보고 싶다"
- 💼 **인사 담당자**: "급여가 가장 적은 직원부터 가장 많은 직원 순으로 보고 싶다"
- 🏪 **상품 관리자**: "판매 가격이 가장 낮은 상품부터 정렬해서 보고 싶다"

이처럼 **목적에 맞게 데이터를 정렬**해서 볼 수 있어야 실제 업무에서 유용하게 활용할 수 있습니다. `ORDER BY`가 없다면, 수천 수만 건의 데이터 중에서 원하는 정보를 찾아내기가 매우 어려울 것입니다.

:::info 핵심 포인트
`ORDER BY`는 **검색 결과의 출력 순서**를 사용자가 원하는 기준으로 설정할 수 있게 해주는 SQL 구문입니다. 데이터 자체를 바꾸는 것이 아니라, **보여주는 순서만** 바꾸는 것입니다.
:::

## 📐 오름차순과 내림차순 — 정렬의 두 가지 방향

### 오름차순(ASC)이란?

**오름차순(Ascending, 줄여서 ASC)**은 값이 **작은 것에서 큰 것으로** 정렬하는 방식입니다. 마치 계단을 올라가듯이 점점 값이 커지는 방향으로 정렬된다고 생각하시면 됩니다.

- 숫자: 1 → 2 → 3 → 4 → 5
- 문자: A → B → C → ... → Z
- 날짜: 오래된 날짜 → 최근 날짜

> 🔑 중요: `ORDER BY`에서 **정렬 방식을 생략하면 기본적으로 오름차순(ASC)**이 적용됩니다. 즉, 아무것도 쓰지 않으면 자동으로 작은 값부터 큰 값 순으로 정렬됩니다.

### 내림차순(DESC)이란?

**내림차순(Descending, 줄여서 DESC)**은 값이 **큰 것에서 작은 것으로** 정렬하는 방식입니다. 계단을 내려가듯이 점점 값이 작아지는 방향으로 정렬됩니다.

- 숫자: 5 → 4 → 3 → 2 → 1
- 문자: Z → Y → X → ... → A
- 날짜: 최근 날짜 → 오래된 날짜

내림차순을 사용하려면 반드시 `DESC` 키워드를 명시해야 합니다. 생략하면 오름차순이 적용되기 때문에, **"가장 큰 값부터 보고 싶다"면 반드시 `DESC`를 써야** 합니다.

| 구분 | 키워드 | 방향 | 숫자 예시 | 기본값 여부 |
|------|--------|------|-----------|-------------|
| 오름차순 | `ASC` | 작은 값 → 큰 값 | 1, 2, 3, 4, 5 | ✅ 기본값 (생략 가능) |
| 내림차순 | `DESC` | 큰 값 → 작은 값 | 5, 4, 3, 2, 1 | ❌ 반드시 명시 필요 |

:::tip 오름차순 생략 팁
`ORDER BY salary ASC`와 `ORDER BY salary`는 **완전히 동일한 결과**를 냅니다. 오름차순은 기본값이므로 `ASC`를 생략해도 됩니다. 하지만 가독성을 위해 명시적으로 써주는 것도 좋은 습관입니다.
:::

## 🔢 데이터 타입별 정렬 기준 이해하기

### 숫자 데이터 정렬

숫자 데이터는 가장 직관적입니다. 오름차순 정렬 시 **가장 작은 숫자가 먼저** 출력되고, 내림차순 정렬 시 **가장 큰 숫자가 먼저** 출력됩니다.

예를 들어, 급여 데이터가 `[300, 500, 250, 400, 150]`이라면:
- 오름차순: 150 → 250 → 300 → 400 → 500
- 내림차순: 500 → 400 → 300 → 250 → 150

### 문자 데이터 정렬

문자 데이터는 **아스키 코드(ASCII Code) 값**을 기준으로 정렬됩니다. 아스키 코드란 컴퓨터가 문자를 숫자로 표현하는 규칙표입니다. 예를 들어, 알파벳 'A'는 65, 'B'는 66, 'Z'는 90으로 변환되어 비교됩니다.

따라서 문자 오름차순 정렬은 알파벳 순서대로(A→Z), 내림차순은 역알파벳 순서(Z→A)로 정렬된다고 이해하면 됩니다. 한글도 마찬가지로 내부 코드값을 기준으로 정렬됩니다.

### 날짜 데이터 정렬

날짜 데이터는 오름차순 정렬 시 **가장 오래된(빠른) 날짜가 먼저** 출력됩니다. 마치 역사책이 오래된 사건부터 순서대로 나열되는 것과 같습니다.

:::warning DBMS별 날짜 정렬 차이 주의
날짜 타입의 정렬 방식은 **사용하는 DBMS(데이터베이스 관리 시스템)에 따라 다를 수 있습니다.** Oracle, MySQL, MS SQL Server 각각 날짜를 처리하는 방식이 조금씩 다르므로, 실무에서는 사용하는 DBMS의 공식 매뉴얼을 반드시 확인하세요.
:::

### NULL 값의 정렬 순서

**NULL(널)**은 "값이 없음"을 의미합니다. NULL 값이 있을 때 정렬 순서는 다음과 같습니다:

| 정렬 방식 | NULL의 위치 |
|-----------|-------------|
| 오름차순(ASC) | NULL이 **마지막**에 출력됨 |
| 내림차순(DESC) | NULL이 **처음**에 출력됨 |

예를 들어, 급여 데이터에 NULL이 포함되어 있다면:
- 오름차순: 150 → 250 → 300 → 400 → **NULL**
- 내림차순: **NULL** → 400 → 300 → 250 → 150

:::note NULL 값 주의
NULL은 0이나 빈 문자열(`''`)과는 다릅니다. NULL은 아예 **값 자체가 없는 상태**입니다. 정렬할 때 이 점을 반드시 기억하세요.
:::

## 💻 ORDER BY 실제 사용법과 코드 예시

### 기본 ORDER BY 문법

```sql
-- ORDER BY 기본 문법 구조
SELECT 컬럼명1, 컬럼명2, ...
FROM 테이블명
WHERE 조건
ORDER BY 정렬기준컬럼 [ASC | DESC];
```

이제 실제 예시를 통해 어떻게 사용하는지 살펴봅시다. 아래와 같은 `employee`(사원) 테이블이 있다고 가정합니다.

| emp_no | emp_name | title | salary | dno |
|--------|----------|-------|--------|-----|
| 1001 | 김철수 | 사장 | 500 | 1 |
| 1002 | 이영희 | 과장 | 300 | 2 |
| 1003 | 박민준 | 대리 | 250 | 2 |
| 1004 | 최지수 | 부장 | 400 | 3 |
| 1005 | 정수연 | 사원 | 200 | 2 |

### 예시 1: 급여 오름차순 정렬 ✅

```sql
-- 2번 부서 직원의 이름, 직급, 급여를 급여 오름차순으로 조회
SELECT emp_name,   -- 사원 이름을 출력
       title,      -- 직급을 출력
       salary      -- 급여를 출력
FROM employee      -- employee 테이블에서 조회
WHERE dno = 2      -- 부서 번호가 2번인 사원만 필터링
ORDER BY salary;   -- 급여를 기준으로 오름차순 정렬 (ASC 생략 가능)
```

**결과:**

| emp_name | title | salary |
|----------|-------|--------|
| 정수연 | 사원 | 200 |
| 박민준 | 대리 | 250 |
| 이영희 | 과장 | 300 |

💡 **코드 설명:**
- `SELECT emp_name, title, salary`: 화면에 출력할 컬럼 3개를 지정합니다.
- `FROM employee`: 데이터를 가져올 테이블을 지정합니다.
- `WHERE dno = 2`: 부서 번호(`dno`)가 2인 사원만 걸러냅니다. 냉장고에서 특정 재료만 골라내는 것과 같습니다.
- `ORDER BY salary`: 급여(`salary`) 기준으로 정렬하며, 뒤에 `ASC`/`DESC`가 없으므로 기본값인 **오름차순** 적용. 급여가 200 → 250 → 300 순으로 정렬됩니다.

### 예시 2: 급여 내림차순 정렬 ✅

```sql
-- 2번 부서 직원의 이름, 직급, 급여를 급여 내림차순으로 조회 (급여 높은 사람 순)
SELECT emp_name,    -- 사원 이름을 출력
       title,       -- 직급을 출력
       salary       -- 급여를 출력
FROM employee       -- employee 테이블에서 조회
WHERE dno = 2       -- 부서 번호가 2번인 사원만 필터링
ORDER BY salary DESC; -- 급여를 기준으로 내림차순 정렬 (DESC 반드시 명시)
```

**결과:**

| emp_name | title | salary |
|----------|-------|--------|
| 이영희 | 과장 | 300 |
| 박민준 | 대리 | 250 |
| 정수연 | 사원 | 200 |

### ❌ 잘못된 예시 vs ✅ 올바른 예시

```sql
-- ❌ 잘못된 예시: 내림차순을 원하는데 DESC를 쓰지 않음
SELECT emp_name, salary
FROM employee
WHERE dno = 2
ORDER BY salary; -- ASC(오름차순)가 기본값으로 적용되어 의도와 반대 결과가 나옴
```

```sql
-- ✅ 올바른 예시: 급여 높은 사람부터 보고 싶을 때는 반드시 DESC 명시
SELECT emp_name, salary
FROM employee
WHERE dno = 2
ORDER BY salary DESC; -- DESC를 명확히 써야 내림차순 정렬이 됨
```

## 🔀 다중 정렬 기준 — ORDER BY 여러 컬럼 사용하기

### 왜 다중 정렬이 필요한가?

실무에서는 단일 기준으로만 정렬해서는 해결되지 않는 경우가 자주 발생합니다. 예를 들어, "나이 순으로 정렬했더니 나이가 같은 사람이 여러 명 있다"는 상황입니다. 이때 **나이가 같은 사람들끼리는 어떤 순서로 출력할 것인지** 두 번째 정렬 기준이 필요합니다.

마치 마라톤 대회에서 결승선을 같은 시간에 통과한 선수들을 배번 번호 순으로 정렬하는 것처럼, 첫 번째 기준이 같을 때 두 번째 기준으로 다시 정렬하는 것입니다.

### 다중 정렬 문법

```sql
-- 다중 정렬 기준 문법: 콤마(,)로 여러 기준을 연결
ORDER BY 첫번째기준컬럼 [ASC|DESC], 두번째기준컬럼 [ASC|DESC], ...
```

### 다중 정렬 예시 ✅

```sql
-- 급여가 300만원 이상인 직원을 나이 오름차순으로 정렬하고,
-- 나이가 같은 경우 급여 내림차순으로 정렬하여 이름 조회
SELECT emp_name    -- 직원 이름을 출력
FROM employee      -- employee 테이블에서 조회
WHERE salary >= 300  -- 급여가 300만원 이상인 직원만 필터링
ORDER BY age ASC,    -- 첫 번째 기준: 나이 오름차순 (어린 사람부터)
         salary DESC; -- 두 번째 기준: 나이가 같으면 급여 내림차순 (급여 많은 사람 먼저)
```

**실행 순서 이해하기:**

위 쿼리가 실행되면 다음과 같은 순서로 처리됩니다:

1. `FROM employee`: employee 테이블 전체 데이터를 가져옵니다.
2. `WHERE salary >= 300`: 급여가 300 이상인 행(row)만 필터링합니다.
3. `ORDER BY age ASC, salary DESC`: 먼저 나이 기준 오름차순 정렬 → 나이가 같으면 급여 기준 내림차순 정렬합니다.
4. `SELECT emp_name`: 최종 결과에서 이름 컬럼만 출력합니다.

**결과 예시 (나이와 급여 데이터가 있다고 가정):**

| emp_name | age | salary |
|----------|-----|--------|
| 정수연 | 28 | 350 |
| 박민준 | 30 | 500 |
| 이영희 | 30 | 450 |
| 김철수 | 30 | 300 |
| 최지수 | 35 | 400 |

- 나이 28세인 정수연이 가장 먼저 등장합니다.
- 나이가 30으로 같은 박민준, 이영희, 김철수는 **급여 내림차순**(500 → 450 → 300)으로 정렬됩니다.
- 35세 최지수는 마지막에 나옵니다.

:::tip 다중 정렬 실무 팁
`ORDER BY`에서 정렬 기준은 이론상 제한 없이 여러 개를 추가할 수 있습니다. 다만 **각 컬럼마다 `ASC` 또는 `DESC`를 개별적으로 지정**해야 합니다. 한 번만 써도 모두 적용되지 않습니다.

```sql
-- ❌ 잘못된 예시: DESC가 두 번째 컬럼에만 적용될 수도 있음 (의도 불명확)
ORDER BY age, salary DESC

-- ✅ 올바른 예시: 각 컬럼에 명확히 정렬 방식 지정
ORDER BY age ASC, salary DESC
```
:::

## 📋 ORDER BY 주의사항과 실전 팁

### 주의사항 1: ORDER BY는 SELECT 문의 마지막에 온다

SQL 문장의 처리 순서와 작성 순서를 혼동하지 마세요. `ORDER BY`는 반드시 `SELECT`문의 **가장 마지막**에 작성해야 합니다.

```sql
-- ✅ 올바른 순서
SELECT emp_name, salary
FROM employee
WHERE dno = 2
ORDER BY salary DESC;
```

```sql
-- ❌ 잘못된 순서: ORDER BY가 WHERE 앞에 오면 에러 발생
SELECT emp_name, salary
ORDER BY salary DESC  -- 여기에 오면 안 됨!
FROM employee
WHERE dno = 2;
```

### 주의사항 2: 컬럼 별칭(Alias)으로도 정렬 가능

`SELECT` 절에서 컬럼에 별칭을 붙인 경우, `ORDER BY`에서 그 별칭을 사용할 수 있습니다.

```sql
-- 급여를 '연봉'이라는 별칭으로 지정하고, 별칭으로 정렬
SELECT emp_name,
       salary AS 연봉  -- salary 컬럼에 '연봉'이라는 별칭 부여
FROM employee
ORDER BY 연봉 DESC;   -- 별칭 '연봉'으로 정렬 가능
```

### 주의사항 3: 컬럼 번호로도 정렬 가능 (권장하지 않음)

`SELECT` 절에 나열된 컬럼의 **순서 번호**로 정렬할 수 있습니다. 하지만 코드 가독성이 떨어져 권장하지 않습니다.

```sql
-- ✅ 컬럼 이름으로 정렬 (권장)
SELECT emp_name, title, salary
FROM employee
ORDER BY salary DESC;

-- ⚠️ 컬럼 번호로 정렬 (비권장 - 가독성이 나쁨)
SELECT emp_name, title, salary
FROM employee
ORDER BY 3 DESC; -- 3번째 컬럼인 salary를 기준으로 정렬
```

:::danger 컬럼 번호 사용 주의
컬럼 번호로 `ORDER BY`를 사용하면, 나중에 `SELECT` 절의 컬럼 순서가 바뀌었을 때 **의도하지 않은 컬럼으로 정렬**될 수 있습니다. 유지보수 시 버그의 원인이 되므로, 반드시 컬럼 이름을 직접 써주세요.
:::

### 주의사항 4: WHERE절과 ORDER BY의 관계

`WHERE`절로 먼저 조건에 맞는 데이터를 필터링한 **후에** `ORDER BY`로 정렬이 이루어집니다. 즉, 전체 데이터를 정렬하는 게 아니라, **필터링된 결과만 정렬**됩니다.

```sql
-- 처리 순서:
-- 1단계: FROM employee (테이블 전체 로드)
-- 2단계: WHERE dno = 2 (2번 부서 사원만 필터링)
-- 3단계: ORDER BY salary ASC (필터링된 결과만 정렬)
-- 4단계: SELECT (최종 출력)
SELECT emp_name, salary
FROM employee
WHERE dno = 2
ORDER BY salary ASC;
```

## 🗂️ SQL SELECT 문 전체 구조 정리

지금까지 배운 내용을 포함하여 `SELECT`문의 전체 구조를 정리해봅시다.

```sql
-- SELECT 문의 전체 구조 (작성 순서)
SELECT 컬럼명1, 컬럼명2, ...   -- 4단계: 출력할 컬럼 지정
FROM 테이블명                   -- 1단계: 데이터를 가져올 테이블 지정
WHERE 조건                      -- 2단계: 행(row) 필터링 조건
ORDER BY 정렬기준 [ASC|DESC];  -- 3단계: 정렬 방식 지정
```

> 📌 **작성 순서**와 **실제 처리 순서**는 다릅니다!  
> - **작성 순서**: SELECT → FROM → WHERE → ORDER BY  
> - **처리 순서**: FROM → WHERE → SELECT → ORDER BY

## 📝 완전한 실전 예제 — 쇼핑몰 데이터 정렬

실제 업무에서 자주 사용되는 패턴을 통해 `ORDER BY`를 종합적으로 연습해봅시다.

아래와 같은 `products`(상품) 테이블이 있다고 가정합니다:

| product_id | product_name | category | price | stock |
|------------|--------------|----------|-------|-------|
| 1 | 노트북 | 전자제품 | 1500000 | 10 |
| 2 | 마우스 | 전자제품 | 25000 | 50 |
| 3 | 청바지 | 의류 | 59000 | 30 |
| 4 | 티셔츠 | 의류 | 19000 | 100 |
| 5 | 헤드폰 | 전자제품 | 120000 | 20 |

```sql
-- 예제 1: 전자제품 카테고리 상품을 가격 오름차순(저렴한 순)으로 조회
SELECT product_name,   -- 상품명 출력
       price,          -- 가격 출력
       stock           -- 재고 수량 출력
FROM products          -- products 테이블에서 데이터 조회
WHERE category = '전자제품'  -- 전자제품 카테고리만 필터링
ORDER BY price ASC;    -- 가격 기준 오름차순 정렬 (저렴한 것부터)
```

```sql
-- 예제 2: 모든 상품을 카테고리 오름차순으로 정렬하고,
-- 같은 카테고리 내에서는 가격 내림차순(비싼 순)으로 정렬
SELECT product_name,   -- 상품명 출력
       category,       -- 카테고리 출력
       price           -- 가격 출력
FROM products          -- products 테이블에서 조회
ORDER BY category ASC,  -- 첫 번째 기준: 카테고리 오름차순 (가나다순)
         price DESC;    -- 두 번째 기준: 같은 카테고리면 가격 내림차순 (비싼 것부터)
```

**예제 2 결과:**

| product_name | category | price |
|--------------|----------|-------|
| 노트북 | 전자제품 | 1500000 |
| 헤드폰 | 전자제품 | 120000 |
| 마우스 | 전자제품 | 25000 |
| 청바지 | 의류 | 59000 |
| 티셔츠 | 의류 | 19000 |

- 전자제품과 의류가 가나다순으로 분류됩니다.
- 전자제품 내에서는 노트북(1,500,000) → 헤드폰(120,000) → 마우스(25,000) 순으로 내림차순 정렬됩니다.
- 의류 내에서는 청바지(59,000) → 티셔츠(19,000) 순으로 내림차순 정렬됩니다.

## ✅ 핵심 정리

이번 문서에서 배운 `ORDER BY`의 핵심 내용을 정리합니다.

- **`ORDER BY`**: SQL `SELECT`문의 결과를 특정 컬럼 기준으로 정렬하여 출력하는 구문
- **오름차순(ASC)**: 작은 값 → 큰 값 순으로 정렬, 기본값이므로 생략 가능
- **내림차순(DESC)**: 큰 값 → 작은 값 순으로 정렬, 반드시 `DESC` 키워드를 명시해야 함
- **정렬 방식을 생략하면 기본적으로 오름차순(ASC)** 이 자동 적용됨
- **문자 데이터**는 아스키 코드(ASCII Code) 값을 기준으로 정렬됨
- **날짜 데이터** 정렬 방식은 DBMS(Oracle, MySQL, MS SQL 등)마다 다를 수 있으므로 공식 매뉴얼 확인 필요
- **NULL 값**은 오름차순 시 마지막, 내림차순 시 첫 번째에 출력됨
- **다중 정렬**: `ORDER BY 컬럼1 ASC, 컬럼2 DESC` 처럼 콤마(`,`)로 여러 기준을 연결하여 사용 가능
- **다중 정렬 처리 순서**: 첫 번째 기준으로 정렬 후, 같은 값이 있을 때 두 번째 기준 적용
- **`ORDER BY`는 `SELECT`문의 가장 마지막**에 작성해야 함
- **실제 처리 순서**: `FROM` → `WHERE` → `SELECT` → `ORDER BY`
- **`WHERE`절 먼저 필터링 후 `ORDER BY` 정렬** 순서로 처리됨을 반드시 기억
