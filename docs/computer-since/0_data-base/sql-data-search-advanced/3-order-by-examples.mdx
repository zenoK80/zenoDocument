# 📊 ORDER BY 활용 예제 (사원 테이블 기준)

SQL을 처음 배우면 데이터를 *가져오는* 것 자체에 집중하게 됩니다. 그런데 실무에서는 단순히 데이터를 가져오는 것을 넘어서, **"어떤 순서로 보여줄 것인가"** 가 훨씬 더 중요한 경우가 많습니다. 쇼핑몰에서 상품을 가격순으로 정렬하거나, 인사 담당자가 급여가 높은 직원부터 순서대로 확인하고 싶을 때처럼요. 이번 문서에서는 `ORDER BY` 구문을 중심으로 SQL 데이터 검색을 응용하는 방법을 아주 자세히 살펴보겠습니다.

---

## 🔤 ORDER BY란 무엇인가?

### 정렬이 왜 필요한가?

데이터베이스 테이블에 저장된 데이터는 기본적으로 **입력된 순서**대로 저장됩니다. 마치 도서관에 책을 들어온 순서대로 그냥 쌓아두는 것과 같습니다. 그 상태에서 "수학 관련 책 중에 가장 최근에 출판된 책이 뭐야?"라고 찾으려면 모든 책을 하나하나 뒤져야 하죠. 하지만 책이 출판 연도 순으로 정렬되어 있다면 훨씬 빠르고 쉽게 찾을 수 있습니다.

SQL에서도 마찬가지입니다. `SELECT ... FROM ...`만 사용하면 데이터가 들어온 순서대로 나오기 때문에, 사용자가 원하는 기준으로 **정렬된 결과**를 보려면 별도의 명령어가 필요합니다. 바로 그 역할을 하는 것이 `ORDER BY`입니다.

> **ORDER BY는 SQL 조회 결과를 특정 컬럼(열)을 기준으로 오름차순 또는 내림차순으로 정렬하여 출력하는 명령어입니다.**

### 실생활 예시로 이해하기

쇼핑몰을 운영한다고 생각해봅시다. 고객 데이터가 수천 건 쌓여 있을 때, 다음과 같은 질문들이 생길 수 있습니다.

- 우리 사이트에서 **총 구매금액이 가장 많은 고객**부터 순서대로 보고 싶다.
- **가격이 가장 싼 상품**부터 가장 비싼 상품 순으로 목록을 보고 싶다.
- **가장 최근에 가입한 회원**부터 오래된 회원 순으로 정렬하고 싶다.

이 모든 상황에서 활용되는 것이 바로 `ORDER BY`입니다. 특정 컬럼을 기준으로 내가 원하는 순서대로 결과를 정렬하여 출력할 수 있습니다.

## 📐 ORDER BY의 핵심 원리: 오름차순과 내림차순

### 오름차순(ASC)과 내림차순(DESC)

`ORDER BY`를 사용할 때 반드시 알아야 할 개념이 바로 **오름차순(ASC, Ascending)** 과 **내림차순(DESC, Descending)** 입니다.

| 구분 | 영어 | 방향 | 숫자 예시 | 문자 예시 |
|------|------|------|-----------|-----------|
| **오름차순** | ASC (Ascending) | 작은 값 → 큰 값 | 1, 2, 3, 4, 5 | A, B, C, D |
| **내림차순** | DESC (Descending) | 큰 값 → 작은 값 | 5, 4, 3, 2, 1 | D, C, B, A |

오름차순은 **엘리베이터가 1층부터 올라가는 것**처럼 값이 점점 커지는 방향으로 정렬되는 것입니다. 내림차순은 반대로 **높은 층에서 아래로 내려오는 것**처럼 값이 점점 작아지는 방향으로 정렬됩니다.

:::info 기본값은 오름차순!
`ORDER BY` 뒤에 정렬 방식을 아무것도 적지 않으면, SQL은 자동으로 **오름차순(ASC)** 으로 정렬합니다. 내림차순으로 정렬하고 싶을 때만 반드시 `DESC`를 명시해야 합니다.
:::

### 데이터 타입별 정렬 기준

정렬 대상이 되는 데이터의 종류에 따라 정렬 기준이 조금씩 달라집니다.

**숫자 데이터의 경우:**
오름차순 정렬 시 가장 작은 숫자(예: 1)가 먼저 출력됩니다. 내림차순 정렬 시 가장 큰 숫자(예: 100)가 먼저 출력됩니다. 이건 우리가 일상적으로 생각하는 정렬 방식과 동일합니다.

**문자 데이터의 경우:**
문자는 내부적으로 **아스키 코드(ASCII Code)** 값으로 변환되어 비교됩니다. 아스키 코드란 컴퓨터가 문자를 숫자로 표현하는 규칙인데, 예를 들어 알파벳 'A'는 65, 'B'는 66, 'Z'는 90처럼 각 문자마다 고유한 숫자가 지정되어 있습니다. 따라서 오름차순 정렬 시 알파벳 순서(A → Z)로, 내림차순 정렬 시 역순(Z → A)으로 정렬됩니다.

**날짜 데이터의 경우:**
오름차순 정렬 시 **가장 오래된 날짜**(과거)가 먼저 출력됩니다. 단, 날짜 정렬의 세부적인 동작 방식은 사용하는 DBMS(데이터베이스 관리 시스템)에 따라 Oracle, MySQL, MSSQL 등에서 조금씩 차이가 있을 수 있으므로, 실무에서는 해당 DBMS의 공식 매뉴얼을 확인하는 것이 좋습니다.

**NULL 값의 경우:**
NULL(널)은 "값이 없음"을 의미하는 특수한 상태입니다. NULL 값이 포함된 데이터를 정렬할 때는 다음과 같이 동작합니다.

- **오름차순(ASC):** NULL 값이 **맨 마지막**에 출력됩니다. (실제 값들이 먼저 나오고, NULL이 뒤에)
- **내림차순(DESC):** NULL 값이 **맨 처음**에 출력됩니다. (NULL이 앞에 나오고, 실제 값들이 뒤에)

:::warning NULL 정렬 주의
NULL 값의 정렬 순서도 DBMS에 따라 다를 수 있습니다. 예를 들어 Oracle에서는 오름차순 정렬 시 NULL이 맨 마지막에 오지만, 다른 DBMS에서는 다를 수 있습니다. 사용 환경을 항상 확인하세요.
:::

## 💻 ORDER BY 기본 문법과 실제 예시

### 사원 테이블(employee) 구조

이번 예제에서는 아래와 같은 구조의 `employee`(사원) 테이블을 기준으로 설명합니다.

| 컬럼명 | 설명 |
|--------|------|
| `emp_number` | 사원 번호 |
| `emp_name` | 사원 이름 |
| `title` | 직급 (예: 사장, 과장, 대리 등) |
| `manager` | 상사 번호 |
| `salary` | 급여 |
| `dno` | 부서 번호 |

아래는 예시 데이터입니다.

| emp_number | emp_name | title | salary | dno |
|------------|----------|-------|--------|-----|
| 1001 | 김철수 | 사장 | 500 | 2 |
| 1002 | 이영희 | 과장 | 300 | 2 |
| 1003 | 박민준 | 대리 | 250 | 2 |
| 1004 | 최수진 | 부장 | 400 | 1 |
| 1005 | 정호준 | 대리 | 200 | 3 |

### 기본 ORDER BY 문법

```sql
-- 기본 ORDER BY 문법 형태
SELECT 컬럼명1, 컬럼명2, ...
FROM 테이블명
WHERE 조건
ORDER BY 정렬기준컬럼 [ASC | DESC];
```

- `SELECT` : 출력할 컬럼들을 지정합니다.
- `FROM` : 데이터를 가져올 테이블을 지정합니다.
- `WHERE` : 조건에 맞는 행(row)만 필터링합니다. (생략 가능)
- `ORDER BY` : 정렬 기준 컬럼을 지정합니다.
- `[ASC | DESC]` : 오름차순(ASC) 또는 내림차순(DESC)을 선택합니다. 생략하면 ASC가 기본값입니다.

### 예제 1 — 2번 부서 사원을 급여 오름차순으로 정렬

**문제:** 2번 부서에 근무하는 사원의 이름, 직급, 급여를 급여 오름차순으로 출력하라.

```sql
-- 2번 부서 사원의 이름, 직급, 급여를 급여 기준 오름차순으로 조회
SELECT emp_name,   -- 사원 이름 출력
       title,      -- 직급 출력
       salary      -- 급여 출력
FROM employee      -- employee(사원) 테이블에서
WHERE dno = 2      -- 부서 번호(dno)가 2번인 사원만 필터링
ORDER BY salary;   -- salary(급여) 기준으로 오름차순 정렬 (ASC 생략)
```

**쿼리 한 줄씩 설명:**
- `SELECT emp_name, title, salary` : 화면에 출력할 컬럼 세 개를 지정합니다. 사원 이름, 직급, 급여만 보겠다는 뜻입니다.
- `FROM employee` : 데이터를 가져올 테이블이 `employee`임을 지정합니다.
- `WHERE dno = 2` : 부서 번호(`dno`)가 2번인 사원만 필터링합니다. 2번 부서가 아닌 사원은 결과에서 제외됩니다.
- `ORDER BY salary` : `salary`(급여) 컬럼을 기준으로 정렬합니다. 뒤에 `ASC`나 `DESC`가 없으므로 기본값인 오름차순(낮은 급여 → 높은 급여)으로 정렬됩니다.

**예상 결과 (오름차순 — 급여 낮은 순):**

| emp_name | title | salary |
|----------|-------|--------|
| 박민준 | 대리 | 250 |
| 이영희 | 과장 | 300 |
| 김철수 | 사장 | 500 |

급여가 250 → 300 → 500 순으로 점점 커지는 것을 확인할 수 있습니다. 이것이 오름차순입니다.

### 예제 2 — 2번 부서 사원을 급여 내림차순으로 정렬

**문제:** 2번 부서에 근무하는 사원의 이름, 직급, 급여를 **급여가 많은 사람 순**으로 출력하라.

```sql
-- 2번 부서 사원의 급여를 내림차순(많은 순)으로 조회
SELECT emp_name,        -- 사원 이름 출력
       title,           -- 직급 출력
       salary           -- 급여 출력
FROM employee           -- employee 테이블에서
WHERE dno = 2           -- 부서 번호가 2번인 사원만
ORDER BY salary DESC;   -- salary 기준으로 내림차순(DESC) 정렬
```

**예상 결과 (내림차순 — 급여 높은 순):**

| emp_name | title | salary |
|----------|-------|--------|
| 김철수 | 사장 | 500 |
| 이영희 | 과장 | 300 |
| 박민준 | 대리 | 250 |

급여가 500 → 300 → 250 순으로 점점 작아지는 것을 확인할 수 있습니다. 이것이 내림차순이고, 반드시 `DESC` 키워드를 명시해야 합니다.

:::tip ASC와 DESC 구분 팁
- **ASC (Ascending)** : 알파벳 A로 시작, "A부터 Z로 올라간다"고 기억하세요. 작은 값이 먼저!
- **DESC (Descending)** : 알파벳 D로 시작, "Down(아래로)"처럼 큰 값이 먼저 내려온다고 기억하세요.
:::

## 🔀 다중 정렬 기준 — ORDER BY에 여러 컬럼 사용하기

### 왜 다중 정렬이 필요한가?

실무 데이터는 항상 완벽하게 고유한 값을 갖지 않습니다. 예를 들어 직원들의 나이를 오름차순으로 정렬할 때, **나이가 같은 직원이 여러 명**이라면 어떻게 해야 할까요? 컴퓨터는 "나이가 같은 경우 누구를 먼저 보여줄지" 스스로 결정하지 못합니다. 이때 **두 번째, 세 번째 정렬 기준**을 추가로 지정하는 것이 다중 정렬입니다.

마치 학교에서 학생을 정렬할 때 "1학년이면 반 번호 순, 같은 반이면 이름 순"처럼 여러 기준을 순서대로 적용하는 것과 같습니다.

### 다중 정렬 문법

```sql
-- 다중 정렬 기준 문법 — 콤마(,)로 여러 컬럼 연결
SELECT 컬럼명1, 컬럼명2
FROM 테이블명
WHERE 조건
ORDER BY 첫번째기준컬럼 ASC, 두번째기준컬럼 DESC;
```

콤마(`,`)로 구분하여 여러 개의 정렬 기준을 지정할 수 있습니다. **첫 번째 기준**으로 먼저 정렬하고, **첫 번째 기준이 같은 경우**에만 두 번째 기준을 적용합니다.

### 예제 3 — 나이 오름차순, 같은 나이면 급여 내림차순

**문제:** 급여가 300만 원 이상인 직원을 나이 오름차순으로 정렬하되, 나이가 같은 경우 급여가 많은 사람 순으로 직원 이름을 출력하라.

```sql
-- 급여 300 이상 직원을 나이 오름차순, 동일 나이는 급여 내림차순으로 조회
SELECT emp_name            -- 직원 이름 출력
FROM employee              -- employee 테이블에서
WHERE salary >= 300        -- 급여가 300 이상인 직원만 필터링
ORDER BY age ASC,          -- 1순위: 나이(age) 기준 오름차순 정렬
         salary DESC;      -- 2순위: 나이가 같을 경우 급여(salary) 내림차순 정렬
```

**쿼리 한 줄씩 설명:**
- `SELECT emp_name` : 결과로 직원의 이름만 출력합니다.
- `FROM employee` : `employee` 테이블을 대상으로 합니다.
- `WHERE salary >= 300` : 급여가 300 이상인 직원만 대상으로 합니다. 300 미만인 직원은 제외됩니다.
- `ORDER BY age ASC` : 첫 번째 정렬 기준은 나이(`age`) 오름차순입니다. 나이가 어린 사람이 먼저 나옵니다.
- `salary DESC` : 두 번째 정렬 기준입니다. 나이가 같은 사람들끼리는 급여가 많은 사람이 먼저 나옵니다.

**결과 예시:**

나이가 31살인 직원이 두 명이고, 각각 급여가 500만 원, 450만 원이라면 아래처럼 정렬됩니다.

| emp_name | age | salary |
|----------|-----|--------|
| 직원A | 31 | 500 |
| 직원B | 31 | 450 |
| 직원C | 32 | 300 |
| 직원D | 33 | 400 |

나이가 같은 직원A와 직원B는 급여가 높은 직원A가 먼저 나옵니다. 나이가 다른 직원C와 직원D는 나이 순서대로 정렬됩니다.

:::note 다중 정렬 기준은 무한히 추가 가능
이론적으로 `ORDER BY` 뒤에 콤마로 구분하여 세 개, 네 개 이상의 정렬 기준을 추가할 수 있습니다. 다만 실무에서는 두 개 정도가 가장 흔하게 사용됩니다. 중요한 것은 **앞에 있는 기준이 같을 때만 다음 기준이 적용된다**는 점입니다.
:::

## ❌ 잘못된 예시 vs ✅ 올바른 예시

초보자가 `ORDER BY`를 사용할 때 자주 실수하는 패턴들을 정리합니다.

### 실수 1 — 내림차순에 DESC를 빠뜨리는 경우

```sql
-- ❌ 잘못된 예시: 급여가 많은 순으로 정렬하고 싶은데 DESC를 빠뜨림
SELECT emp_name, salary
FROM employee
ORDER BY salary;  -- DESC 없으면 오름차순(낮은 급여부터)이 됨!

-- ✅ 올바른 예시: 내림차순 정렬 시 반드시 DESC 명시
SELECT emp_name, salary
FROM employee
ORDER BY salary DESC;  -- DESC 명시하여 급여 높은 순으로 정렬
```

`DESC`를 빠뜨리면 정반대 결과가 나옵니다. 급여가 가장 낮은 사람부터 출력되므로 반드시 확인해야 합니다.

### 실수 2 — WHERE와 ORDER BY 순서를 뒤바꾸는 경우

```sql
-- ❌ 잘못된 예시: ORDER BY를 WHERE보다 앞에 쓰면 문법 오류 발생
SELECT emp_name, salary
FROM employee
ORDER BY salary DESC   -- ORDER BY가 WHERE보다 앞에 위치 → 오류!
WHERE dno = 2;

-- ✅ 올바른 예시: SQL 절의 순서는 항상 SELECT → FROM → WHERE → ORDER BY
SELECT emp_name, salary
FROM employee
WHERE dno = 2          -- WHERE 조건이 먼저
ORDER BY salary DESC;  -- ORDER BY는 맨 마지막
```

SQL 절(clause)의 작성 순서는 규칙이 있습니다. 반드시 `SELECT → FROM → WHERE → ORDER BY` 순서를 지켜야 합니다.

### 실수 3 — 다중 정렬 시 콤마 없이 작성하는 경우

```sql
-- ❌ 잘못된 예시: 두 번째 정렬 기준 앞에 콤마를 빠뜨림
SELECT emp_name, age, salary
FROM employee
ORDER BY age ASC salary DESC;  -- 콤마 없음 → 문법 오류!

-- ✅ 올바른 예시: 정렬 기준 사이에 반드시 콤마(,) 사용
SELECT emp_name, age, salary
FROM employee
ORDER BY age ASC,      -- 첫 번째 기준: 나이 오름차순
         salary DESC;  -- 두 번째 기준: 급여 내림차순 (콤마로 구분!)
```

:::danger 절대 하면 안 되는 것
`ORDER BY` 뒤의 여러 정렬 기준 사이에 콤마(`,`)를 반드시 넣어야 합니다. 콤마 없이 작성하면 SQL 문법 오류(Syntax Error)가 발생하여 쿼리가 실행되지 않습니다.
:::

## 📋 ORDER BY 작성 시 주의사항과 꿀팁

### 주의사항 1 — SELECT에 없는 컬럼으로도 정렬 가능

`ORDER BY` 뒤에 오는 컬럼은 반드시 `SELECT`에 포함되지 않아도 됩니다. 예를 들어 이름만 출력하더라도 급여를 기준으로 정렬할 수 있습니다.

```sql
-- SELECT에 salary가 없어도 ORDER BY에서 salary 기준으로 정렬 가능
SELECT emp_name           -- 이름만 출력 (salary는 출력 안 함)
FROM employee
WHERE dno = 2
ORDER BY salary DESC;     -- 하지만 salary 기준으로 내림차순 정렬 적용
```

화면에는 이름만 보이지만, 내부적으로는 급여 순서대로 정렬된 결과가 출력됩니다.

### 주의사항 2 — 컬럼 번호로도 정렬 가능 (권장하지 않음)

`ORDER BY` 뒤에 컬럼 이름 대신 `SELECT` 목록에서의 순서 번호를 사용할 수도 있습니다.

```sql
-- 컬럼 번호로 정렬하는 방법 (권장하지 않음)
SELECT emp_name,   -- 1번째 컬럼
       title,      -- 2번째 컬럼
       salary      -- 3번째 컬럼
FROM employee
ORDER BY 3 DESC;   -- 3번째 컬럼(salary)을 기준으로 내림차순 정렬
```

```sql
-- ✅ 권장하는 방법: 컬럼 이름을 직접 명시
SELECT emp_name, title, salary
FROM employee
ORDER BY salary DESC;   -- 컬럼 이름을 직접 사용하면 가독성이 훨씬 좋음
```

컬럼 번호 방식은 `SELECT` 목록이 바뀌면 의도와 다른 컬럼으로 정렬될 수 있어서 유지보수할 때 문제가 생길 수 있습니다. 가능하면 **컬럼 이름을 직접 명시하는 것**이 좋습니다.

### 주의사항 3 — ORDER BY는 SELECT 문의 맨 마지막에 위치

SQL 문장의 실행 순서와 작성 순서를 헷갈리지 않도록 주의해야 합니다. `ORDER BY`는 항상 SQL 문장의 **맨 마지막**에 위치합니다.

```
SQL 절 작성 순서:
1. SELECT (어떤 컬럼을 출력할 것인가?)
2. FROM (어떤 테이블에서 가져올 것인가?)
3. WHERE (어떤 조건으로 필터링할 것인가?)
4. ORDER BY (어떤 기준으로 정렬할 것인가?)
```

:::tip SQL 절 순서 암기법
**"선택(SELECT) → 출처(FROM) → 조건(WHERE) → 정렬(ORDER BY)"**
"선출조정"이라고 줄여서 기억해도 좋습니다!
:::

## 🧩 ORDER BY 종합 실습 예제

지금까지 배운 내용을 종합해서 다양한 상황의 쿼리를 작성해봅시다.

### 종합 예제 1 — 전체 사원 급여 오름차순 정렬

```sql
-- 전체 사원의 이름과 급여를 급여 오름차순으로 조회
SELECT emp_name,   -- 사원 이름
       salary      -- 급여
FROM employee      -- employee 테이블에서
ORDER BY salary;   -- 급여 기준 오름차순 정렬 (ASC 생략)
```

### 종합 예제 2 — 부서별, 급여 내림차순 다중 정렬

```sql
-- 사원의 이름, 부서번호, 급여를 부서번호 오름차순 + 급여 내림차순으로 조회
SELECT emp_name,       -- 사원 이름
       dno,            -- 부서 번호
       salary          -- 급여
FROM employee          -- employee 테이블에서
ORDER BY dno ASC,      -- 1순위: 부서번호(dno) 오름차순 (1번 부서 → 2번 → 3번)
         salary DESC;  -- 2순위: 같은 부서 내에서는 급여 내림차순 (많은 급여 먼저)
```

**예상 결과:**

| emp_name | dno | salary |
|----------|-----|--------|
| 최수진 | 1 | 400 |
| 김철수 | 2 | 500 |
| 이영희 | 2 | 300 |
| 박민준 | 2 | 250 |
| 정호준 | 3 | 200 |

1번 부서 → 2번 부서 → 3번 부서 순으로 정렬되고, 같은 2번 부서 내에서는 급여가 높은 순(500 → 300 → 250)으로 정렬됩니다.

### 종합 예제 3 — 조건 필터링 + 다중 정렬 조합

```sql
-- 급여가 200 이상이고 3번 부서가 아닌 사원을 이름 오름차순, 급여 내림차순으로 조회
SELECT emp_name,        -- 사원 이름
       title,           -- 직급
       salary,          -- 급여
       dno              -- 부서 번호
FROM employee           -- employee 테이블에서
WHERE salary >= 200     -- 급여가 200 이상인 사원만
  AND dno <> 3          -- 부서번호가 3번이 아닌 사원만 (<>는 '같지 않다' 의미)
ORDER BY emp_name ASC,  -- 1순위: 이름 오름차순 (가나다 순)
         salary DESC;   -- 2순위: 이름이 같으면 급여 내림차순
```

:::note AND 조건과 ORDER BY 함께 사용
`WHERE`절에서 `AND`로 여러 조건을 연결할 수 있습니다. `WHERE` 조건이 먼저 데이터를 필터링한 후, `ORDER BY`가 그 결과를 정렬합니다. 순서를 이해하면 복잡한 쿼리도 단계적으로 이해할 수 있습니다.
:::

## ✅ 핵심 정리

- **`ORDER BY`** 는 SQL 조회 결과를 특정 컬럼 기준으로 정렬하는 명령어이며, `SELECT` 문의 맨 마지막에 위치한다.
- **오름차순(ASC)** 은 작은 값부터 큰 값 순으로 정렬되며, `ORDER BY` 뒤에 아무것도 쓰지 않으면 기본값으로 적용된다.
- **내림차순(DESC)** 은 큰 값부터 작은 값 순으로 정렬되며, 반드시 `DESC` 키워드를 명시해야 한다.
- 숫자는 크기 순, 문자는 아스키 코드 값(알파벳 순) 기준으로 정렬된다.
- **NULL 값**은 오름차순 정렬 시 맨 마지막에, 내림차순 정렬 시 맨 처음에 나타난다.
- **다중 정렬**은 콤마(`,`)로 여러 정렬 기준을 연결하며, 앞 기준이 같을 때만 뒤 기준이 적용된다.
- SQL 절 작성 순서는 `SELECT → FROM → WHERE → ORDER BY` 이며, 이 순서를 반드시 지켜야 한다.
- `ORDER BY`에 사용하는 컬럼은 `SELECT`에 포함되지 않아도 되지만, 컬럼 이름을 직접 명시하는 것이 가독성 측면에서 권장된다.
- `WHERE`절 조건과 `ORDER BY` 정렬을 함께 사용할 때는 WHERE가 먼저 필터링한 결과에 ORDER BY가 적용된다.
