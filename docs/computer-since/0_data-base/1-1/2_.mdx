# 데이터베이스의 정의와 특성

데이터베이스가 무엇인지 정확하게 이해하고, 데이터베이스가 가져야 할 핵심 특성들을 살펴보겠습니다.

---

## 데이터베이스의 정의

### 한 문장 정의
**"조직에서 업무 수행과 의사결정을 위해 여러 사람이 공유하여 사용할 목적으로, 흩어져 있는 데이터를 통합하여 저장 장치에 저장한 운영 데이터의 집합"**

### 정의를 구성하는 핵심 요소

```
데이터베이스 = 공용 데이터 + 통합된 데이터 + 저장된 데이터 + 운영 데이터
```

---

## 1. 통합된 데이터 (Integrated Data)

### 개념
여러 곳에 흩어져 있는 데이터를 한 곳으로 모아 중복을 최소화한 데이터입니다.

### 통합의 의미

**통합 전:**
```
부서A: 직원정보.xlsx
부서B: 직원정보.xlsx (복사본)
부서C: 직원정보.xlsx (복사본)
```

**통합 후:**
```
중앙DB: 직원정보 (단일 저장소)
       ↓
부서A, B, C 모두 접근
```

### 중복 제거의 중요성

**중복으로 인한 문제:**
- 데이터 불일치 발생
- 저장 공간 낭비
- 갱신 비용 증가
- 일관성 유지 어려움

**최소한의 중복은 허용:**
- 100% 중복 제거는 비효율적
- **통제된 중복(Controlled Redundancy)** 허용
- 성능 향상을 위한 의도적 중복 가능

### 예시
```sql
-- 잘못된 설계 (중복)
학생테이블: 학번, 이름, 학과명, 학과전화
교수테이블: 교수번호, 이름, 학과명, 학과전화

-- 올바른 설계 (통합)
학생테이블: 학번, 이름, 학과코드
교수테이블: 교수번호, 이름, 학과코드
학과테이블: 학과코드, 학과명, 학과전화
```

## 2. 저장된 데이터 (Stored Data)

### 개념
컴퓨터가 접근 가능한 저장 매체에 디지털 형태로 저장된 데이터입니다.

### 저장 매체
- **하드디스크 (HDD)**
- **SSD (Solid State Drive)**
- **클라우드 스토리지**
- **네트워크 스토리지 (NAS, SAN)**

### 저장 형태

**물리적 저장:**
```
디스크 → 파일 시스템 → 데이터베이스 파일 (.mdf, .ibd 등)
```

**논리적 저장:**
```
데이터베이스 → 테이블 → 레코드 → 필드
```

### 특징
- 영구적 보존 (Persistent)
- 빠른 검색 가능
- 대용량 저장 가능
- 백업 및 복구 지원

---

## 3. 공용 데이터 (Shared Data)

### 개념
조직 내 여러 사용자와 응용 시스템이 함께 사용하는 데이터입니다.

### 공유의 특징

**동시 접근 (Concurrent Access):**
```
사용자A: 데이터 조회 중
사용자B: 데이터 수정 중  → 동시에 가능
사용자C: 데이터 입력 중
```

**다중 사용자 지원:**
- 수십 명 ~ 수천 명 동시 접속
- 실시간 데이터 공유
- 권한 관리 필요

### 공유를 위한 메커니즘

**동시성 제어 (Concurrency Control):**
- 여러 사용자의 동시 작업 조율
- 데이터 일관성 유지
- 충돌 방지

**권한 관리:**
```sql
-- 읽기 권한만 부여
GRANT SELECT ON 학생테이블 TO 일반사용자;

-- 모든 권한 부여
GRANT ALL ON 학생테이블 TO 관리자;
```

### 예시
```
온라인 쇼핑몰 데이터베이스
├─ 고객: 상품 조회, 주문
├─ 관리자: 재고 관리, 주문 처리
└─ 분석팀: 판매 데이터 분석
```

## 4. 운영 데이터 (Operational Data)

### 개념
조직의 고유 기능을 수행하기 위해 필요한 핵심 업무 데이터입니다.

### 특징
- **업무 필수 데이터**: 조직 운영에 꼭 필요한 정보
- **지속적 갱신**: 실시간으로 변경되고 업데이트됨
- **높은 활용도**: 자주 조회되고 사용됨

### 조직별 운영 데이터 예시

**대학교:**
```
학생 정보, 수강 신청, 성적 관리, 학사 일정
```

**은행:**
```
계좌 정보, 거래 내역, 대출 정보, 고객 정보
```

**병원:**
```
환자 정보, 진료 기록, 처방전, 예약 관리
```

**쇼핑몰:**
```
상품 정보, 주문 내역, 재고 관리, 배송 정보
```

### 운영 데이터 vs 임시 데이터

| 구분 | 운영 데이터 | 임시 데이터 |
|------|-------------|-------------|
| 중요도 | 높음 | 낮음 |
| 보관 기간 | 장기 | 단기 |
| 백업 | 필수 | 선택 |
| 예시 | 고객 주문 내역 | 임시 계산 결과 |

## 데이터베이스의 4대 특성 종합

### 특성 비교표

| 특성 | 핵심 개념 | 목적 | 효과 |
|------|-----------|------|------|
| **통합된 데이터** | 중복 최소화 | 일관성 유지 | 저장 공간 절약, 관리 용이 |
| **저장된 데이터** | 디지털 저장 | 영구 보존 | 빠른 검색, 대용량 처리 |
| **공용 데이터** | 다중 사용자 | 정보 공유 | 협업 가능, 효율성 증대 |
| **운영 데이터** | 업무 필수 | 조직 운영 | 의사결정 지원, 업무 수행 |

---

## 데이터베이스가 아닌 것들

### 운영 데이터가 아닌 경우
```
❌ 임시 파일 (temp files)
❌ 백업 파일 (backup files)
❌ 로그 파일 (log files)
❌ 입출력용 데이터
```

### 저장되지 않은 데이터
```
❌ 메모리에만 존재하는 데이터
❌ 휘발성 캐시 데이터
❌ 프로그램 실행 중 임시 변수
```

### 공유되지 않는 데이터
```
❌ 개인 로컬 파일
❌ 암호화된 개인 문서
❌ 특정 프로그램 전용 데이터
```

## 실제 사례로 이해하기

### 사례 1: 대학교 학사 관리 시스템

**통합된 데이터:**
- 학생, 교수, 수강, 성적 정보를 하나의 DB에 통합
- 중복된 학과 정보를 별도 테이블로 분리

**저장된 데이터:**
- 대학 서버의 하드디스크에 영구 저장
- 매일 자동 백업 수행

**공용 데이터:**
- 학생: 수강 신청, 성적 조회
- 교수: 성적 입력, 강의 관리
- 직원: 학사 행정 처리

**운영 데이터:**
- 학생 등록, 수강 관리, 성적 처리 등 필수 업무 데이터

### 사례 2: 온라인 쇼핑몰

**통합된 데이터:**
- 상품, 주문, 고객, 배송 정보 통합 관리
- 중복된 카테고리 정보는 별도 관리

**저장된 데이터:**
- 클라우드 데이터베이스에 저장
- 실시간 동기화

**공용 데이터:**
- 고객: 상품 검색, 주문
- 판매자: 재고 관리
- CS팀: 문의 응대

**운영 데이터:**
- 주문 처리, 재고 관리, 결제 정보 등

## 핵심 정리

### 데이터베이스의 완전한 정의

```
데이터베이스는 조직의 업무 수행과 의사결정을 위해
여러 사람이 공유하여 사용할 목적으로
흩어진 데이터를 통합하여
컴퓨터 저장 장치에 저장한
운영 데이터의 집합입니다.
```

### 4대 특성 암기법

```
통저공운 (통합-저장-공용-운영)
```

### 체크리스트

데이터베이스인지 확인하려면:
- ✅ 중복이 최소화되어 있는가?
- ✅ 디지털 형태로 저장되어 있는가?
- ✅ 여러 사람이 공유하여 사용하는가?
- ✅ 조직의 업무 수행에 필수적인가?

4가지 모두 만족하면 → **데이터베이스입니다!**
