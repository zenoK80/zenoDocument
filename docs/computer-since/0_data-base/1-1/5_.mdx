# DBMS (데이터베이스 관리 시스템)

데이터베이스를 효율적으로 관리하는 시스템인 DBMS에 대해 알아보겠습니다.

---

## 1. DBMS의 정의

### DBMS란?

**Database Management System (데이터베이스 관리 시스템)**

```
DBMS = 데이터베이스를 관리하는 소프트웨어
```

### 한 문장 정의

**"응용 프로그램과 데이터베이스 사이에서 데이터에 대한 모든 접근을 처리하는 중재자 역할을 하는 소프트웨어"**

---

## 2. DBMS의 구조와 역할

### 기본 구조

```
[사용자/응용 프로그램]
         ↓
    [DBMS] ← 중재자
         ↓
   [데이터베이스]
```

### 상세 구조

```
사용자 ──→ 응용 프로그램
              ↓
          ┌──────┐
          │      │
          │ DBMS │ ← 인터페이스 & 중재자
          │      │
          └──────┘
              ↓
        데이터베이스
        (하드디스크)
```

### DBMS의 역할

#### 1. 중재자 (Intermediary)
```
응용 프로그램 ←─ DBMS ─→ 데이터베이스

- 직접 접근을 막음
- 모든 요청을 중개
- 데이터 보호
```

#### 2. 인터페이스 (Interface)
```
사용자 요청 → DBMS 변환 → 데이터베이스 처리

- 사용자 친화적 언어 제공 (SQL)
- 복잡한 처리 과정 숨김
- 표준화된 접근 방법
```

#### 3. 관리자 (Manager)
```
DBMS가 관리하는 것:
- 데이터 저장
- 데이터 검색
- 데이터 수정
- 보안 및 권한
- 백업 및 복구
```

## 3. DBMS의 탄생 배경

### 파일 시스템의 문제점

```
파일 시스템:
❌ 종속성 문제
❌ 중복성 문제
❌ 데이터 불일치
❌ 무결성 유지 어려움
```

### DBMS의 해결

```
DBMS 도입:
✅ 데이터 독립성 제공
✅ 중복 최소화
✅ 일관성 보장
✅ 무결성 유지
```

### 발전 과정

```
파일 시스템 (1960년 이전)
      ↓
문제점 발견
      ↓
DBMS 개발 필요성 대두
      ↓
계층형 DBMS (1960년대)
      ↓
네트워크형 DBMS (1970년대)
      ↓
관계형 DBMS (1980년대~현재)
```

## 4. DBMS의 필수 3대 기능

### 핵심 요약

```
DBMS 필수 기능 = 정의 + 조작 + 제어
```

---

### 4.1 정의 기능 (Definition)

#### 개념
데이터베이스의 구조를 생성하고 정의하는 기능입니다.

#### 주요 역할
- **테이블 생성** (CREATE)
- **테이블 수정** (ALTER)
- **테이블 삭제** (DROP)
- **인덱스 정의**
- **제약조건 설정**

#### 사용 언어
**DDL (Data Definition Language) - 데이터 정의어**

#### 예시

```sql
-- 테이블 생성
CREATE TABLE 학생 (
    학번 VARCHAR(10) PRIMARY KEY,
    이름 VARCHAR(50) NOT NULL,
    학과 VARCHAR(50),
    학년 INT
);

-- 테이블 수정
ALTER TABLE 학생 ADD COLUMN 이메일 VARCHAR(100);

-- 테이블 삭제
DROP TABLE 학생;
```

#### 특징
```
정의 기능 → 구조 생성 → 메타데이터 저장

메타데이터 저장 위치:
- 데이터 사전 (Data Dictionary)
- 카탈로그 (Catalog)
```

---

### 4.2 조작 기능 (Manipulation)

#### 개념
데이터를 검색, 삽입, 수정, 삭제하는 기능입니다.

#### 주요 역할
- **검색** (SELECT)
- **삽입** (INSERT)
- **수정** (UPDATE)
- **삭제** (DELETE)

#### 사용 언어
**DML (Data Manipulation Language) - 데이터 조작어**

#### 예시

```sql
-- 검색 (SELECT)
SELECT 학번, 이름, 학과 
FROM 학생 
WHERE 학년 = 3;

-- 삽입 (INSERT)
INSERT INTO 학생 (학번, 이름, 학과, 학년)
VALUES ('2024001', '홍길동', '컴퓨터공학', 3);

-- 수정 (UPDATE)
UPDATE 학생 
SET 학과 = '전자공학' 
WHERE 학번 = '2024001';

-- 삭제 (DELETE)
DELETE FROM 학생 
WHERE 학번 = '2024001';
```

#### 특징
```
조작 기능 → 데이터 처리 → 실제 값 변경

구조는 그대로, 값만 변경
```

---

### 4.3 제어 기능 (Control)

#### 개념
데이터의 정확성, 안전성, 무결성을 유지하고 관리하는 기능입니다.

#### 주요 역할

**1. 무결성 제약조건 (Integrity Constraints)**
```
데이터의 정확성 보장
- 기본키 제약
- 외래키 제약
- NOT NULL 제약
- CHECK 제약
```

**2. 보안 및 권한 관리 (Security & Authorization)**
```
데이터 접근 제어
- 사용자 권한 부여
- 접근 제한
- 암호화
```

**3. 병행 제어 (Concurrency Control)**
```
동시 접근 관리
- 여러 사용자 동시 사용
- 충돌 방지
- 데드락 해결
```

**4. 회복 (Recovery)**
```
장애 발생 시 복구
- 백업
- 트랜잭션 롤백
- 데이터 복원
```

#### 사용 언어
**DCL (Data Control Language) - 데이터 제어어**

#### 예시

```sql
-- 권한 부여
GRANT SELECT, INSERT ON 학생 TO 사용자A;

-- 권한 회수
REVOKE DELETE ON 학생 FROM 사용자A;

-- 트랜잭션 제어
BEGIN TRANSACTION;
UPDATE 계좌 SET 잔액 = 잔액 - 10000 WHERE 계좌번호 = '1234';
UPDATE 계좌 SET 잔액 = 잔액 + 10000 WHERE 계좌번호 = '5678';
COMMIT;  -- 성공 시 저장
-- ROLLBACK;  -- 실패 시 취소
```

---

### 3대 기능 비교표

| 기능 | 언어 | 대상 | 역할 | 예시 |
|------|------|------|------|------|
| **정의** | DDL | 구조 | 생성/수정/삭제 | CREATE, ALTER, DROP |
| **조작** | DML | 데이터 | 검색/삽입/수정/삭제 | SELECT, INSERT, UPDATE, DELETE |
| **제어** | DCL | 관리 | 보안/무결성/회복 | GRANT, REVOKE, COMMIT |

### 3대 기능의 관계

```
1. 정의 기능으로 테이블 생성
         ↓
2. 조작 기능으로 데이터 입력/수정
         ↓
3. 제어 기능으로 안전성 보장
```

## 5. DBMS의 장점

### 주요 장점 목록

```
✅ 중복 최소화
✅ 동시 공유
✅ 일관성 유지
✅ 무결성 유지
✅ 보안 보장
✅ 표준화
✅ 효율적 백업 및 회복
```

### 상세 설명

#### 1. 중복 최소화 (Minimizing Redundancy)
```
파일 시스템: 3개 파일에 동일 데이터 저장
DBMS: 1개 테이블에 한 번만 저장

→ 저장 공간 절약
→ 관리 용이
```

#### 2. 동시 공유 (Concurrent Sharing)
```
여러 사용자가 동시에 데이터 접근 가능

사용자A: 데이터 조회
사용자B: 데이터 수정  ← 동시 실행
사용자C: 데이터 입력

→ 병행 제어 기능으로 충돌 방지
```

#### 3. 일관성 유지 (Maintaining Consistency)
```
한 곳에서 수정 → 모든 곳에 반영

학생 이름 변경:
DBMS: 1번 수정 → 자동 반영
파일: 여러 파일 각각 수정 필요

→ 일관된 정보 보장
```

#### 4. 무결성 유지 (Maintaining Integrity)
```
데이터 정확성 보장

제약조건 설정:
- 학년은 1~4만 가능
- 학번은 중복 불가
- 성적은 0~100 범위

→ 잘못된 데이터 입력 방지
```

#### 5. 보안 보장 (Security)
```
사용자별 권한 관리

일반 사용자: 조회만 가능
관리자: 모든 작업 가능
특정 부서: 해당 데이터만 접근

→ 통합된 보안 정책
```

#### 6. 표준화 (Standardization)
```
SQL이라는 표준 언어 사용

Oracle, MySQL, MSSQL 모두:
SELECT * FROM 학생;

→ 배우기 쉬움
→ 이식성 좋음
```

#### 7. 효율적 백업 및 회복 (Backup & Recovery)
```
자동 백업 기능
장애 발생 시 자동 복구

파일 시스템: 수동 백업
DBMS: 자동 백업 + 트랜잭션 로그

→ 데이터 손실 최소화
```

## 6. DBMS의 단점

### 주요 단점 목록

```
❌ 높은 비용
❌ 복잡성
❌ 시스템 위험성
```

### 상세 설명

#### 1. 높은 비용 (High Cost)

**초기 도입 비용:**
```
- 소프트웨어 라이선스 구매
  (Oracle: 수천만 원~수억 원)
- 하드웨어 구입
  (서버, 스토리지)
- 통신망 구축
```

**운영 비용:**
```
- 전문 인력 고용 (DBA)
- 유지보수 비용
- 업그레이드 비용
- 교육 비용
```

**예시:**
```
중소기업 DBMS 도입 비용:
- Oracle Enterprise: 5천만 원~
- 서버: 2천만 원
- DBA 인건비: 연 5천만 원
- 유지보수: 연 1천만 원

→ 초기 비용만 억 단위
```

#### 2. 복잡성 (Complexity)

**시스템 복잡도:**
```
파일 시스템: 단순 구조
DBMS: 복잡한 계층 구조

[사용자]
   ↓
[응용 프로그램]
   ↓
[DBMS 엔진]
   ↓
[운영체제]
   ↓
[하드웨어]

→ 문제 발생 시 원인 파악 어려움
```

**문제 진단의 어려움:**
```
성능 저하 발생 시:
- 네트워크 문제?
- DBMS 설정 문제?
- 쿼리 최적화 문제?
- 하드웨어 문제?

→ 전문가 필요
```

#### 3. 시스템 위험성 (System Vulnerability)

**전체 시스템 마비 위험:**
```
일부 오류 → 전체 시스템 정지

파일 시스템:
파일A 손상 → 파일A만 사용 불가

DBMS:
DB 서버 다운 → 전체 업무 중단

→ 단일 장애점 (Single Point of Failure)
```

**보안 위협:**
```
중앙 집중식 구조:
- 해킹 시 모든 데이터 유출 가능
- 랜섬웨어 공격 시 전체 데이터 암호화
- 권한 탈취 시 전체 데이터 접근 가능

→ 보안 사고 시 피해 범위 큼
```

**예시:**
```
2023년 A기업 DB 해킹 사건:
- DB 서버 1대 해킹
- 전체 고객 정보 유출 (500만 건)
- 피해액: 수백억 원

파일 시스템이었다면:
- 일부 파일만 유출
- 피해 범위 제한적
```

## 7. 장단점 비교

### 종합 비교표

| 측면 | 장점 | 단점 | 결론 |
|------|------|------|------|
| **데이터 관리** | 중복 최소화, 일관성 | - | ✅ 효율적 |
| **동시 사용** | 병행 제어 지원 | - | ✅ 우수 |
| **데이터 품질** | 무결성 보장 | - | ✅ 높음 |
| **보안** | 통합 관리 | 해킹 시 전체 피해 | ⚠️ 양면성 |
| **비용** | - | 초기/운영 비용 높음 | ❌ 부담 |
| **복잡도** | - | 관리 복잡 | ❌ 어려움 |
| **안정성** | 백업/복구 우수 | 장애 시 전체 중단 | ⚠️ 양면성 |

### 언제 DBMS를 사용해야 하나?

#### DBMS 사용이 적합한 경우

```
✅ 다수의 사용자가 동시 접근
✅ 데이터 양이 많음 (GB 이상)
✅ 데이터 정확성이 중요
✅ 복잡한 질의 필요
✅ 보안이 중요
✅ 데이터 공유 필요

예: 은행, 병원, 학교, 대기업
```

#### 파일 시스템이 적합한 경우

```
✅ 단순한 데이터
✅ 소량의 데이터 (MB 단위)
✅ 개인 사용
✅ 비용 최소화 필요
✅ 빠른 개발 필요

예: 개인 문서, 소규모 애플리케이션
```

## 8. 대표적인 DBMS 제품

### 상용 DBMS

#### Oracle
```
특징:
- 가장 강력한 기능
- 대규모 시스템에 적합
- 높은 안정성
- 비용: 매우 높음

주요 사용처: 대기업, 금융권, 정부기관
```

#### Microsoft SQL Server (MSSQL)
```
특징:
- Windows 환경 최적화
- .NET 통합 우수
- 관리 도구 편리
- 비용: 중간~높음

주요 사용처: Windows 기반 기업
```

### 오픈소스 DBMS

#### MySQL
```
특징:
- 무료 (커뮤니티 버전)
- 빠른 성능
- 웹 서비스에 적합
- 비용: 무료

주요 사용처: 웹 서비스, 스타트업
```

#### PostgreSQL
```
특징:
- 무료 + 오픈소스
- 표준 SQL 준수
- 확장성 우수
- 비용: 무료

주요 사용처: 개발자, 연구기관
```

#### MariaDB
```
특징:
- MySQL 호환
- 완전 오픈소스
- 빠른 발전
- 비용: 무료

주요 사용처: MySQL 대체
```

## 핵심 정리

### DBMS의 정의

```
DBMS = 응용 프로그램과 데이터베이스 사이의 중재자
      = 데이터를 관리하는 소프트웨어
```

### DBMS의 필수 3대 기능

```
정의 (DDL) + 조작 (DML) + 제어 (DCL) = DBMS
```

### DBMS의 핵심 장점

```
중복 최소화
일관성 유지
무결성 보장
동시 공유
```

### DBMS의 주요 단점

```
높은 비용
복잡성
시스템 위험성
```

### 암기 팁

```
정조제 (정의-조작-제어) → DBMS 3대 기능

중동일무보표백 → DBMS 7가지 장점
(중복최소화, 동시공유, 일관성, 무결성, 보안, 표준화, 백업)

비복위 → DBMS 3가지 단점
(비용, 복잡성, 위험성)
```

### 핵심 메시지

```
"DBMS는 파일 시스템의 문제를 해결하기 위해 만들어졌다.
 비용과 복잡성이라는 단점이 있지만,
 다수가 사용하는 환경에서는 필수적이다."
```
