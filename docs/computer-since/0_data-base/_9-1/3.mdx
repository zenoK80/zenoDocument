# 함수적 종속성 (Functional Dependency)

## 개요

함수적 종속성은 속성 간의 관계를 나타내는 제약조건으로, 정규화 과정의 핵심 개념입니다. 릴레이션 내의 속성들이 어떻게 서로를 결정하는지를 정의합니다.

## 함수 종속의 기본 개념

### 정의

**X → Y** (X가 Y를 함수적으로 결정한다)

- **X**: 결정자 (Determinant)
- **Y**: 종속자 (Dependent)
- X 값이 정해지면 Y 값이 유일하게 결정됨

### 표기법

```
X → Y
또는
Y는 X에 함수 종속이다
```

### 실생활 예시

```
학번 → 이름, 학년, 전공
- 학번이 정해지면 이름, 학년, 전공이 유일하게 결정됨

주민등록번호 → 이름, 생년월일, 성별
- 주민등록번호로 개인정보가 결정됨

(학번, 과목번호) → 성적
- 학번과 과목번호를 알면 성적이 결정됨
```

## 함수 종속성 판단 방법

### 테이블로 확인하기

```
릴레이션 R
A | B | C | D
--|---|---|---
1 | 2 | 3 | 4
1 | 2 | 3 | 4
2 | 3 | 4 | 5
2 | 3 | 4 | 1
```

**B가 C를 결정하는가? (B → C)**
- B = 2 일 때, C = 3 (첫 번째, 두 번째 행)
- B = 3 일 때, C = 4 (세 번째, 네 번째 행)
- B 값이 같으면 C 값도 항상 같음
- **✅ B → C 성립**

**D가 B를 결정하는가? (D → B)**
- D = 4 일 때, B = 2, 3 (여러 값 존재)
- D 값이 같아도 B 값이 다름
- **❌ D → B 성립하지 않음**

### 판단 규칙

```
X → Y가 성립하려면:
X 값이 같은 모든 튜플에서 Y 값도 반드시 같아야 함
```

## 함수 종속의 유형

### 1. 완전 함수 종속 (Full Functional Dependency)

**정의**: 종속자가 기본키 전체에 종속되는 경우

```
수강(학번, 과목번호, 성적, 담당교수)
기본키: (학번, 과목번호)

(학번, 과목번호) → 성적  ✅ 완전 함수 종속
- 학번과 과목번호 둘 다 필요해야 성적 결정
```

**특징**:
- 기본키의 모든 속성이 필요
- 기본키의 일부만으로는 종속자 결정 불가

### 2. 부분 함수 종속 (Partial Functional Dependency)

**정의**: 종속자가 기본키의 일부분에만 종속되는 경우

```
수강(학번, 과목번호, 성적, 학년)
기본키: (학번, 과목번호)

학번 → 학년  ✅ 부분 함수 종속
- 기본키의 일부(학번)만으로 학년 결정 가능
- 과목번호는 필요 없음
```

**문제점**:
- 이상현상의 주요 원인
- 데이터 중복 발생
- 2정규형(2NF)에서 제거 대상

### 3. 이행적 함수 종속 (Transitive Functional Dependency)

**정의**: X → Y, Y → Z 관계가 있을 때, X → Z가 성립하는 간접 종속

```
학생(학번, 학과코드, 학과사무실)

학번 → 학과코드
학과코드 → 학과사무실
∴ 학번 → 학과사무실  ✅ 이행적 함수 종속
```

**특징**:
- 직접 종속이 아닌 간접 종속
- A를 통해 B를 거쳐서 C가 결정됨

**문제점**:
- 갱신 이상 발생 가능
- 3정규형(3NF)에서 제거 대상

## 함수 종속 예시

### 예시 1: 학생 정보

```
학생(학번, 이름, 학과, 지도교수, 교수연락처)

함수 종속성:
학번 → 이름
학번 → 학과
학번 → 지도교수
지도교수 → 교수연락처  ← 이행적 종속
∴ 학번 → 교수연락처  (이행적)
```

### 예시 2: 주문 정보

```
주문(주문번호, 상품번호, 고객ID, 수량, 상품명, 고객명)
기본키: (주문번호, 상품번호)

함수 종속성:
(주문번호, 상품번호) → 수량      ← 완전 함수 종속
주문번호 → 고객ID                ← 부분 함수 종속
주문번호 → 고객명                ← 부분 함수 종속
상품번호 → 상품명                ← 부분 함수 종속
고객ID → 고객명                  ← 이행적 종속
```

### 예시 3: 부서-직원

```
직원(사번, 이름, 부서코드, 부서명, 부서위치)

함수 종속성:
사번 → 이름
사번 → 부서코드
부서코드 → 부서명
부서코드 → 부서위치
∴ 사번 → 부서명    (이행적)
∴ 사번 → 부서위치  (이행적)
```

## 함수 종속성과 정규화

### 정규화 단계별 제거 대상

| 정규형 | 제거 대상 | 결과 |
|--------|-----------|------|
| **1NF** | 반복 그룹, 다치 속성 | 원자값만 존재 |
| **2NF** | 부분 함수 종속 | 완전 함수 종속만 존재 |
| **3NF** | 이행적 함수 종속 | 직접 종속만 존재 |
| **BCNF** | 결정자가 후보키가 아닌 경우 | 모든 결정자가 후보키 |

### 함수 종속성 분해 예시

```
원본 테이블:
학생(학번, 이름, 학과코드, 학과명, 지도교수)

함수 종속성:
학번 → 이름, 학과코드, 지도교수
학과코드 → 학과명  ← 이행적 종속 발견

분해:
학생(학번, 이름, 학과코드, 지도교수)
학과(학과코드, 학과명)
```

## 함수 종속성 다이어그램

```
완전 함수 종속:
(A, B) ────────→ C
  전체 필요

부분 함수 종속:
(A, B)
  │
  A ─────→ C
  일부만으로 결정

이행적 함수 종속:
A ──→ B ──→ C
따라서 A ──→ C
```

## 요약

함수적 종속성의 핵심:

1. **기본 개념**: X → Y (X가 Y를 결정)
2. **완전 함수 종속**: 기본키 전체에 종속
3. **부분 함수 종속**: 기본키 일부에 종속 (2NF 위반)
4. **이행적 함수 종속**: 간접 종속 (3NF 위반)
5. **정규화와의 관계**: 각 정규형은 특정 종속성을 제거

함수 종속성을 정확히 파악하는 것이 **올바른 정규화의 출발점**입니다.
