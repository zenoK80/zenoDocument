# 최소 커버 (Minimal Cover)

## 개요

최소 커버(Minimal Cover 또는 Canonical Cover)는 함수 종속성 집합에서 **중복되는 종속성을 제거**하여 최소한의 종속성만 남긴 집합입니다. 정규화 과정에서 효율적인 스키마 설계를 위해 필수적입니다.

## 최소 커버의 개념

### 정의

함수 종속성 집합 F의 최소 커버 Fc는 다음 조건을 만족합니다:

1. **F와 논리적으로 동등** (F와 Fc로부터 같은 종속성 유도 가능)
2. **모든 종속성의 오른쪽은 단일 속성**
3. **중복 종속성 없음**
4. **불필요한 속성 없음**

### 중복이란?

```
예시:
X → Y
Y → Z
X → Z  ← 이것은 중복! (이행 규칙으로 유도 가능)

최소 커버:
X → Y
Y → Z
(X → Z는 제거)
```

## 최소 커버 찾기 규칙

### 규칙 1: 오른쪽 속성을 단일화

**모든 함수 종속성의 오른쪽이 하나의 속성만 가지도록 분해**

```
변환 전:
A → BC

변환 후:
A → B
A → C
```

### 규칙 2: 왼쪽 속성 축소

**왼쪽(결정자)에서 불필요한 속성 제거**

```
AB → C에서
A만으로 C를 결정할 수 있다면
A → C로 축소
```

### 규칙 3: 중복 종속성 제거

**다른 종속성들로부터 유도 가능한 종속성 제거**

```
이행 종속 제거:
A → B, B → C, A → C
→ A → C 제거 (이행 규칙으로 유도 가능)
```

## 최소 커버 찾는 알고리즘

### 단계별 절차

```
1단계: 오른쪽 단일화
   모든 종속성의 오른쪽을 단일 속성으로 분해

2단계: 왼쪽 축소
   각 종속성의 왼쪽에서 불필요한 속성 제거

3단계: 중복 제거
   이행 종속 등 중복 종속성 제거
```

## 예시 1: 기본 예제

### 문제

```
F = {
  B → A,
  D → A,
  AB → D
}
```

### 풀이 과정

**1단계: 오른쪽 단일화**
```
B → A   ✓ (이미 단일)
D → A   ✓ (이미 단일)
AB → D  ✓ (이미 단일)
```

**2단계: 왼쪽 축소**
```
AB → D를 축소할 수 있는가?

B만으로 D를 결정할 수 있는가?
- B → A (주어짐)
- 양쪽에 B를 증가: BB → AB, 즉 B → AB
- AB → D (주어짐)
- 따라서 B → D (가이행 규칙) ✓

AB → D를 B → D로 축소 가능!
```

**축소 후:**
```
B → A
D → A
B → D
```

**3단계: 중복 제거**
```
이행 종속 확인:
- B → D, D → A이므로 B → A (이행 규칙)
- B → A는 중복! 제거 가능
```

**최소 커버:**
```
Fc = {
  D → A,
  B → D
}
```

## 예시 2: 복잡한 예제

### 문제

```
F = {
  A → BC,
  B → C,
  A → B,
  AB → C
}
```

### 풀이 과정

**1단계: 오른쪽 단일화**
```
A → BC  →  A → B, A → C
B → C      B → C
A → B      A → B
AB → C     AB → C

중복 제거 (A → B가 두 번):
A → B
A → C
B → C
AB → C
```

**2단계: 왼쪽 축소**
```
AB → C를 축소 가능한가?

A만으로 C 결정 가능?
- A → C가 이미 있음
- 중복! AB → C 제거 가능
```

**축소 후:**
```
A → B
A → C
B → C
```

**3단계: 중복 제거**
```
이행 종속 확인:
- A → B, B → C이므로 A → C (이행 규칙)
- A → C는 중복! 제거 가능
```

**최소 커버:**
```
Fc = {
  A → B,
  B → C
}
```

## 최소 커버의 특성

### 유일성 (Non-uniqueness)

**최소 커버는 여러 개 존재할 수 있습니다!**

```
F = {A → B, B → A, B → C}

최소 커버 1: {A → B, B → C}
최소 커버 2: {B → A, B → C}

둘 다 올바른 최소 커버!
```

### 동등성 (Equivalence)

**모든 최소 커버는 원래 F와 논리적으로 동등**

```
F와 Fc로부터 같은 종속성 집합을 유도할 수 있음
```

## 최소 커버의 활용

### 1. 정규화에서의 활용

```
3NF 분해 알고리즘:
1. 최소 커버 Fc 구하기
2. Fc의 각 종속성마다 릴레이션 생성
3. 후보키를 포함하는 릴레이션 추가
```

### 2. 스키마 최적화

```
불필요한 종속성 제거
→ 테이블 수 최소화
→ 효율적인 데이터베이스 설계
```

## 요약

최소 커버를 찾는 핵심:

1. **오른쪽 단일화**: 모든 종속성의 오른쪽을 하나의 속성으로
2. **왼쪽 축소**: 결정자에서 불필요한 속성 제거
3. **중복 제거**: 이행 규칙 등으로 유도 가능한 종속성 제거
4. **동등성 유지**: 원래 종속성 집합과 논리적으로 동등

최소 커버는 **정규화의 효율성**을 높이고 **스키마를 단순화**하는 데 필수적인 개념입니다.
