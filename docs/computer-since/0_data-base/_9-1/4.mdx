# 암스트롱 공리 (Armstrong's Axioms)

## 암스트롱 공리란?

**암스트롱 공리**(Armstrong's Axioms)는 기존의 함수 종속성으로부터  
새로운 함수 종속성을 논리적으로 유추할 수 있는 추론 규칙입니다.

### 목적

- 명시된 함수 종속성으로부터 암묵적인 함수 종속성 도출
- 정규화 과정에서 모든 함수 종속성 파악
- 데이터베이스 설계의 이론적 근거 제공

---

## 기본 규칙 (3가지)

암스트롱 공리의 **핵심 규칙 3가지**를 살펴보겠습니다.

### 1. 재귀 규칙 (Reflexivity)

**정의:**
```
X → Y이면, X → X
```

**의미:**
- 모든 속성 집합은 자기 자신을 결정함
- Y가 X의 부분집합이면 X → Y

**예시:**
```
(학번, 이름) → 학번  ✅
학번 → 학번          ✅
```

**특징:**
- 자명한(Trivial) 함수 종속
- 정규화에서 일반적으로 고려하지 않음

---

### 2. 증가 규칙 (Augmentation)

**정의:**
```
X → Y이면, WX → WY
(양쪽에 같은 속성 W를 추가해도 성립)
```

**수학적 비유:**
```
a = b이면, a + c = b + c
```

**예시:**
```
학번 → 이름

증가 규칙 적용 (W = 학과코드):
(학번, 학과코드) → (이름, 학과코드)
```

**증명 논리:**
```
X → Y이면
WX → W      (재귀 규칙)
WX → Y      (X → Y이므로)
────────────
∴ WX → WY   (결합하면)
```

---

### 3. 이행 규칙 (Transitivity)

**정의:**
```
X → Y이고 Y → Z이면, X → Z
```

**일상적 예:**
```
나 → 사람
사람 → 죽는 존재
──────────────
∴ 나 → 죽는 존재
```

**데이터베이스 예시:**
```
학번 → 학과코드
학과코드 → 학과장
──────────────────
∴ 학번 → 학과장
```

**활용:**
- 이행적 함수 종속 발견
- 3NF 정규화의 이론적 근거

---

## 추가 규칙 (파생 규칙)

기본 규칙 3가지로부터 유도되는 **유용한 추가 규칙들**입니다.

### 4. 유니온 규칙 (Union Rule)

**정의:**
```
X → Y이고 X → Z이면, X → YZ
```

**예시:**
```
학번 → 이름
학번 → 학년
─────────────────
∴ 학번 → (이름, 학년)
```

**활용:**
- 여러 함수 종속을 하나로 통합
- 릴레이션 설계 시 속성 그룹화

**증명:**
```
X → Y      (주어짐)
X → Z      (주어짐)
XZ → YZ    (증가 규칙: X → Y에 Z 추가)
X → YZ     (X → Z이고 XZ → YZ이므로, 이행 규칙)
```

---

### 5. 분해 규칙 (Decomposition Rule)

**정의:**
```
X → YZ이면, X → Y이고 X → Z
```

**예시:**
```
학번 → (이름, 학년)
──────────────────
∴ 학번 → 이름
∴ 학번 → 학년
```

**특징:**
- 유니온 규칙의 역
- 속성 분리 시 사용

---

### 6. 가이행 규칙 (Pseudotransitivity)

**정의:**
```
W → X이고 XY → Z이면, WY → Z
```

**증명 과정:**

**Step 1:** W → X에 Y를 증가
```
W → X
─────────── (증가 규칙: 양쪽에 Y 추가)
WY → XY
```

**Step 2:** 이행 규칙 적용
```
WY → XY    (Step 1 결과)
XY → Z     (주어진 조건)
──────────────────
∴ WY → Z   (이행 규칙)
```

**예시:**
```
학번 → 학과코드
(학과코드, 학년) → 졸업학점
────────────────────────────
∴ (학번, 학년) → 졸업학점
```

## 암스트롱 공리 적용 예제

### 주어진 함수 종속성

```
F = {
  A → B,
  B → H,
  A → C,
  CG → H,
  CG → I
}
```

### 유추 과정

#### 예제 1: A → H 성립 확인

```
A → B    (주어짐)
B → H    (주어짐)
─────────────
∴ A → H  (이행 규칙)
```

**결과:** ✅ 성립

---

#### 예제 2: CG → HI 성립 확인

```
CG → H   (주어짐)
CG → I   (주어짐)
──────────────
∴ CG → HI (유니온 규칙)
```

**결과:** ✅ 성립

---

#### 예제 3: AG → I 성립 확인

**Step 1:** A → C 양쪽에 G 추가
```
A → C
──────────── (증가 규칙)
AG → CG
```

**Step 2:** 이행 규칙 적용
```
AG → CG   (Step 1)
CG → I    (주어짐)
─────────────────
∴ AG → I  (이행 규칙)
```

**결과:** ✅ 성립 (가이행 규칙)

---

### 예제 4: C → F 성립 확인

**주어진 추가 종속성:**
```
C → BD
B → F
```

**Step 1:** C → BD 분해
```
C → BD
────────── (분해 규칙)
C → B
C → D
```

**Step 2:** 이행 규칙 적용
```
C → B    (Step 1)
B → F    (주어짐)
─────────────
∴ C → F  (이행 규칙)
```

**결과:** ✅ 성립

---

### 예제 5: AB → E 성립 확인

**주어진 추가 종속성:**
```
B → F
AF → E
```

**Step 1:** B → F 양쪽에 A 추가
```
B → F
──────────── (증가 규칙)
AB → AF
```

**Step 2:** 이행 규칙 적용
```
AB → AF   (Step 1)
AF → E    (주어짐)
─────────────────
∴ AB → E  (이행 규칙)
```

**결과:** ✅ 성립 (가이행 규칙)

## 암스트롱 공리의 특징

### 건전성 (Soundness)

**암스트롱 공리로 유도된 모든 함수 종속은 타당합니다.**

```
F에서 유도된 모든 함수 종속은 F를 만족하는 모든 릴레이션에서 성립
```

---

### 완전성 (Completeness)

**함수 종속 집합 F에서 논리적으로 유도 가능한 모든 함수 종속을 암스트롱 공리로 유도할 수 있습니다.**

```
F가 논리적으로 내포하는 모든 함수 종속을 암스트롱 공리로 도출 가능
```

---

## 클로저 (Closure)

### 속성 집합의 클로저

**정의:**
```
X+ = X로부터 함수적으로 결정되는 모든 속성의 집합
```

**계산 방법:**

1. X+ = X로 초기화
2. X+에서 다른 속성을 결정할 수 있으면 추가
3. 더 이상 추가할 속성이 없을 때까지 반복

**예시:**
```
F = {A → B, B → C, C → D}
A+ = ?

초기: A+ = {A}
Step 1: A → B이므로 A+ = {A, B}
Step 2: B → C이므로 A+ = {A, B, C}
Step 3: C → D이므로 A+ = {A, B, C, D}

∴ A+ = {A, B, C, D}
```

---

## 실무 활용

### 1. 후보키 찾기

```
속성 집합 X가 후보키 ⟺ X+ = 모든 속성
```

### 2. 슈퍼키 판단

```
X가 슈퍼키 ⟺ X+ = 모든 속성
```

### 3. 함수 종속 확인

```
X → Y 성립 여부 ⟺ Y ⊆ X+
```

## 핵심 요약

### 암스트롱 공리 기본 규칙

| 규칙 | 표현 | 이름 |
|------|------|------|
| **재귀** | X → Y ⇒ X → X | Reflexivity |
| **증가** | X → Y ⇒ WX → WY | Augmentation |
| **이행** | X → Y, Y → Z ⇒ X → Z | Transitivity |

### 추가 규칙 (파생)

| 규칙 | 표현 | 이름 |
|------|------|------|
| **유니온** | X → Y, X → Z ⇒ X → YZ | Union |
| **분해** | X → YZ ⇒ X → Y, X → Z | Decomposition |
| **가이행** | W → X, XY → Z ⇒ WY → Z | Pseudotransitivity |

### 활용 분야

- ✅ 함수 종속성 추론
- ✅ 후보키 찾기
- ✅ 정규화 이론적 근거
- ✅ 릴레이션 분해 정당성 검증

암스트롱 공리는 **정규화의 이론적 토대**이며, 데이터베이스 설계의 정확성을 보장합니다.
