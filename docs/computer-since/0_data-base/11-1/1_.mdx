# 프로그래밍의 기본 개념

안녕하세요! 이번 강의에서는 **프로그래밍**이 무엇인지에 대한 기초 개념을 함께 살펴보겠습니다.

## 컴퓨터 언어 vs 프로그래밍 언어

### 컴퓨터 언어란?

**컴퓨터 언어**는 컴퓨터 시스템에서 소프트웨어를 만들 때 필요한 언어입니다. 더 정확히 말하면, **컴퓨터와 인간이 의사소통하기 위해 서로 약속된 규칙**으로 만들어진 언어입니다.

### 프로그래밍 언어란?

**프로그래밍 언어**는 프로그램을 직접 작성할 때 필요한 언어입니다. 소프트웨어를 만들기 위해 우리가 실제로 코드를 작성하는 데 사용합니다.

### 쉽게 이해하기 📖

큰 틀에서 보면:
- **컴퓨터 언어** = 컴퓨터의 소프트웨어를 만들기 위한 언어
- **프로그래밍 언어** = 프로그래밍을 하기 위한 언어 (실제 코딩)

둘 다 **정해진 문법적 규칙** 내에서만 사용할 수 있습니다. 이 규칙을 지키지 않으면 컴퓨터가 이해하지 못합니다.

:::info
비유하면, 영어 문법 규칙을 따라야 영어가 통하는 것처럼, 각 프로그래밍 언어의 문법 규칙을 따라야 정상적으로 작동합니다.
:::

## C 언어: 프로그래밍의 기초

### C 언어가 중요한 이유

C 언어는 **모든 프로그래밍의 가장 기본적인 기초**입니다.

- 개발자를 꿈꾸는 분이라면 **반드시 배워야 할 언어**
- 프로그래밍을 처음 배운다면 **C 언어부터 시작**해야 합니다
- 다른 언어를 배울 때도 C 언어의 기초가 매우 도움이 됩니다

### C 언어 vs C 프로그램

이 두 용어는 자주 헷갈리니까 구분해서 알아봅시다:

- **C 언어** = C로 프로그래밍할 수 있게 해주는 컴파일러 소프트웨어
- **C 프로그램** = C 언어를 이용해서 개발된 소프트웨어 (결과물)

### 실행 과정 이해하기

```cpp
#include <stdio.h>

int main() {
    int A = 10;  // A라는 변수에 10을 저장
    int B = 20;  // B라는 변수에 20을 저장
    int SUM = A + B;  // A와 B를 더해서 SUM에 저장
    printf("합계: %d", SUM);  // 결과 출력
    return 0;
}
```

**이 코드가 실행되는 과정:**

1. **코딩** 👨‍💻 - 개발자가 프로그래밍 언어로 소스 코드 작성
2. **컴파일** 🔍 - 문법적 오류가 없는지 확인
3. **링크** 🔗 - 실행 가능한 형태로 변환
4. **실행** ▶️ - 프로그램 실행해서 결과 출력 (이 경우 "합계: 30")

:::tip
컴파일러가 자동으로 문법을 검사해주므로, 우리는 **소스 코드만 문법적으로 정확하게 작성**하면 됩니다!
:::

## 코더 vs 프로그래머의 차이

### 코더(Coder)란?

단순히 프로그래밍 언어를 이용해 **기술적으로만 코딩하는 사람**입니다.

- 누군가가 주는 알고리즘을 그대로 구현하기만 함
- 이미 만들어진 프로그램을 유지보수하기만 함
- 단순 반복 작업 수행

### 프로그래머(Programmer)란?

**논리적 사고**와 **체계적인 계획**으로 프로그래밍하는 사람입니다.

- 사용자의 요구조건을 분석
- 비용과 일정을 고려해서 전략 수립
- 체계적인 개발 방법 적용
- 다양한 상황을 미리 예측

:::warning
개발자를 꿈꾼다면 **코더가 아니라 프로그래머**가 되어야 합니다!
:::

## 프로그래밍의 목적

### 건물 짓기와 같다

건물을 지을 때를 생각해봅시다:
1. **설계도** 📐 - 건축 설계도가 필요
2. **기초공사부터 차근차근** - 설계도에 따라 단계별 시공
3. **완성** - 설계도 완성

### 프로그래밍도 동일합니다

**프로그래밍의 흐름:**

1. **요구사항 분석** 📋 - 사용자가 원하는 것이 뭔지 파악
2. **설계** 📐 - 입력(Input) → 처리(Process) → 출력(Output) 결정
3. **코딩** 💻 - 설계에 맞게 코드 작성
4. **테스트** ✅ - 제대로 작동하는지 확인

### 설계 시 결정할 사항

```
입력(Input)은 어떻게 받을까?
├─ 키보드 입력?
├─ 음성 입력?
└─ 파일에서 읽기?

처리(Process)는 어떻게 할까?
├─ 어떤 계산?
└─ 어떤 로직?

출력(Output)은 어떤 형태일까?
├─ 텍스트 화면 출력?
├─ 파일로 저장?
└─ 그래프/이미지로 표시?
```

## 프로그래밍 언어 선택의 중요성

### 언어 선택 기준

개발하려는 프로그램의 **스타일과 목적**에 맞게 선택해야 합니다:

- **절차형 프로그래밍** (C 언어)
  - 간단한 순차적 처리
  - 선형적으로 처리하는 프로그램

- **객체지향 프로그래밍** (C++, Java, Python)
  - 재사용 가능한 객체 설계
  - 상속, 다형성 활용
  - 복잡한 대규모 프로젝트

- **웹 기반 프로그래밍** (JavaScript, PHP, Python)
  - 웹사이트 개발
  - 웹 애플리케이션 개발

### 올바른 선택이 곧 효율성입니다

**같은 결과도 언어에 따라 효율이 달라집니다!**

예를 들어, 1부터 100까지의 합을 구하는 프로그램을 100명의 학생이 각각 다르게 작성합니다:
- 어떤 코드는 간단하고 명확
- 어떤 코드는 복잡하고 이해하기 어려움

**좋은 프로그래밍 = 간단하고 명확한 코딩** ✨
