# 질의어 처리 단계 개요

## 🎯 이 시간에 배울 내용

우리가 이번 시간에 배울 주제는 **SQL 질의어(Query)가 데이터베이스에서 어떻게 처리되는지**에 관한 것입니다.  
학교 숙제를 제출할 때도 여러 단계를 거치죠? 마찬가지로 SQL도 처리 과정을 거쳐서 최종 결과가 나옵니다.

:::info 왜 배워야 할까요?
**성능**과 **최적화**를 이해하기 위함입니다.
더 빠르고 효율적인 데이터베이스를 만드는 방법을 배우게 됩니다!
:::

---

## 📚 고급 언어 vs 저급 언어

### SQL은 "고급 언어"입니다

**고급 언어란?**
- 사람이 이해하기 쉬운 형태로 명령을 내린다
- 예: `SELECT`, `FROM`, `WHERE` 같은 키워드를 직접 사용

**사용하는 방법 2가지:**
1. **응용 프로그램에 삽입** - 파이썬, 자바 등의 프로그램에 SQL 코드를 넣어서 사용
2. **직접 서버 접속** - 데이터베이스 관리 도구를 열어서 직접 SQL 명령을 입력

---

## 🔄 SQL 질의어 처리의 전체 흐름

질의어가 들어오면 **5가지 단계**를 거쳐서 처리됩니다:

```
고급 질의어 (SQL)
    ⬇️
┌─────────────────────────────────────────┐
│  1️⃣ 검사기 (Scanner)                     │ ← 어떤 명령어인지 확인
└─────────────────────────────────────────┘
    ⬇️
    📄 내부 형태 질의문 (중간 결과물)
    ⬇️
┌─────────────────────────────────────────┐
│  2️⃣ 파서 (Parser)                       │ ← 문법이 맞는지 확인
└─────────────────────────────────────────┘
    ⬇️
    📄 파서 결과물
    ⬇️
┌─────────────────────────────────────────┐
│  3️⃣ 질의어 최적화기 (Query Optimizer)  │ ← 가장 빠른 방법을 찾기
└─────────────────────────────────────────┘
    ⬇️
    📄 질의문 계획 (실행 방법)
    ⬇️
┌─────────────────────────────────────────┐
│  4️⃣ 코드 생성기 (Code Generator)       │ ← 실행 가능한 코드 만들기
└─────────────────────────────────────────┘
    ⬇️
    📄 실행코드
    ⬇️
┌─────────────────────────────────────────┐
│  5️⃣ 런타임 처리기 (Runtime Processor)  │ ← 실제로 실행
└─────────────────────────────────────────┘
    ⬇️
✅ 최적화된 결과
```

:::tip 정리해보면
네모 모양 = **시스템 구성 요소** (하는 일)
흐린 박스 = **각 단계의 결과물**
:::

## 📖 간단한 예시로 이해하기

### 예제: 모든 학생 정보 조회

```sql
SELECT * FROM 학생;
```

이 간단한 명령어가 어떻게 처리되는지 봅시다:

| 단계 | 역할 | 결과 |
|------|------|------|
| **검사기** | "아, SELECT 명령어구나. FROM 절도 있고, 테이블 이름은 '학생'이구나" | 각 요소를 식별 |
| **파서** | "SELECT FROM 형식이 맞네. 문법 오류 없음!" | 문법 검증 완료 |
| **최적화기** | "가장 효율적인 방법을 생각해보자..." | 실행 계획 수립 |
| **코드 생성기** | "이렇게 해야겠다!" | 실행 코드 작성 |
| **런타임** | "자, 실행해볼까!" | 결과 출력 |

## 🎓 이게 왜 "고급 과정"인가?

### 데이터베이스 학습 단계

```
📚 기본 과정 (이전에 배운 것)
├─ SQL 명령어 배우기 (SELECT, INSERT, UPDATE, DELETE)
├─ 권한 부여하기 (GRANT)
└─ 기본 개념 이해하기

🚀 고급 과정 (이번부터 배울 것) ← 우리는 여기!
├─ 질의어 최적화 ← 이번 시간
├─ 병행 제어 (여러 사용자가 동시에 접근할 때)
└─ 데이터 회복 (문제 발생 시 복구)
```

:::success 고급 과정의 의미
고급 과정을 배우면 **데이터베이스 관리자(DBA)**처럼 생각할 수 있습니다!
- 어떻게 더 빠르게 만들 것인가?
- 어떻게 효율적으로 관리할 것인가?
- 성능을 어떻게 높일 것인가?
:::

## 💡 핵심 요점 정리

:::note 꼭 기억하세요!
- SQL은 **고급 언어**이다 → 사람이 이해하기 쉽다
- 질의어는 **5단계**를 거쳐 처리된다 → 검사 → 파싱 → 최적화 → 코드생성 → 실행
- 최적화는 **데이터베이스의 성능**을 높인다
- 하나의 SQL도 **여러 방법**으로 실행 가능하다 → 가장 빠른 방법을 선택!
:::
