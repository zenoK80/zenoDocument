# 🗄️ SQL이란 무엇인가 – 개념과 특징

데이터베이스를 공부하다 보면 가장 먼저, 그리고 가장 많이 마주치는 단어가 바로 **SQL**입니다. 오라클(Oracle), MSSQL, MySQL 등 어떤 데이터베이스를 사용하든 SQL은 공통적으로 사용되는 핵심 언어입니다. 이 문서에서는 SQL이 무엇인지, 왜 필요한지, 그리고 실제로 어떻게 사용하는지를 초보자도 쉽게 이해할 수 있도록 아주 자세하게 설명합니다.

---

## 1️⃣ SQL이란 무엇인가? – 개념 소개

### SQL의 정의

**SQL**은 **Structured Query Language(구조화된 질의 언어)** 의 약자입니다. 실무에서는 흔히 **"시퀄(sequel)"** 이라고 읽습니다. 쉽게 말하면, **데이터베이스에게 말을 거는 언어**입니다.

마치 우리가 도서관 사서에게 "역사 관련 책을 찾아주세요"라고 부탁하는 것처럼, SQL은 데이터베이스에게 "이 조건에 맞는 데이터를 찾아줘", "새로운 데이터를 저장해줘", "이 데이터를 수정해줘"라고 명령하는 언어입니다.

> **SQL은 관계형 데이터베이스(테이블 형태로 데이터를 저장하는 데이터베이스)를 대상으로, 데이터를 만들고, 조회하고, 수정하고, 삭제하는 모든 작업을 수행하기 위한 표준 언어입니다.**

### SQL의 주요 특징

SQL에는 몇 가지 중요한 특징이 있습니다. 하나씩 살펴보겠습니다.

**① 고급 언어(High-level Language)**

SQL은 **고급 언어**입니다. 고급 언어란 사람이 이해하기 쉬운 형태로 작성된 언어를 말합니다. 마치 영어 문장처럼 `SELECT name FROM student WHERE age > 20` 이라고 쓰면, "학생 테이블에서 나이가 20보다 큰 학생의 이름을 가져와라"라는 의미를 직관적으로 이해할 수 있습니다.

**② 비절차적 언어(Non-procedural Language)**

SQL은 **비절차적 언어**입니다. 절차적 언어는 "어떻게(How) 처리할 것인가"를 단계별로 명시해야 하지만, 비절차적 언어는 **"무엇을(What) 원하는가"만 명시**하면 됩니다. 예를 들어 C언어라면 "배열을 하나하나 순회하면서 조건을 확인하는" 과정을 직접 코딩해야 하지만, SQL은 그냥 `WHERE age > 20`이라고 조건만 적으면 됩니다. 나머지 처리는 데이터베이스가 알아서 해줍니다.

**③ 결과가 레코드 집합(테이블)으로 반환**

SQL의 결과는 **하나씩 나오는 것이 아니라 테이블 형태로** 나옵니다. 테이블에 대해 질의하고, 테이블을 처리하고, 결과도 테이블로 반환됩니다. 마치 엑셀 표에서 원하는 행들을 필터링하면 결과가 또 하나의 표로 나타나는 것과 같습니다.

**④ 독립적으로도, 내장해서도 사용 가능**

SQL은 **두 가지 방식으로 사용**할 수 있습니다.

| 사용 방식 | 설명 | 예시 |
|---|---|---|
| **대화식(Interactive) 방식** | DB 서버에 직접 접속해서 SQL을 입력하고 결과를 바로 확인 | Oracle SQL Developer, MySQL Workbench 등 |
| **내장(Embedded) 방식** | Java, Python 등의 프로그래밍 언어 코드 안에 SQL을 포함시켜 사용 | Java 코드 안에 `SELECT * FROM ...` 삽입 |

:::info SQL이 두 가지 방식 모두 가능한 이유
SQL이 고급 질의어이기 때문입니다. 고급 언어는 사람이 이해하기 쉬운 형태로 작성되어 있어 독립적으로도 사용할 수 있고, 다른 언어에 포함시켜도 자연스럽게 동작합니다.
:::

## 2️⃣ SQL은 왜 필요한가? – 배경과 역사

### 데이터베이스에서 SQL이 필요한 이유

데이터베이스는 수많은 데이터를 저장하고 관리하는 시스템입니다. 그런데 이 데이터를 **효율적으로 활용**하려면 어떤 방법이 필요할까요?

마치 거대한 창고에 수백만 개의 물건이 쌓여 있다고 생각해 보세요. 창고에서 특정 물건을 꺼내거나, 새 물건을 추가하거나, 낡은 물건을 버리거나, 물건의 위치를 바꾸려면 **창고 관리 규칙**이 필요합니다. SQL이 바로 그 역할을 합니다. 데이터베이스라는 거대한 창고를 관리하기 위한 표준 규칙이자 명령어 체계가 SQL입니다.

### SQL의 역사 – 어떻게 발전해왔나?

SQL은 수십 년에 걸쳐 발전해왔습니다. 그 역사를 간단히 살펴보면 다음과 같습니다.

| 연도 | 버전 | 주요 내용 |
|---|---|---|
| **1986년** | 최초 표준화 | SQL의 첫 번째 국제 표준이 만들어짐 |
| **1989년** | SQL-1 | 외래키(참조무결성) 개념이 추가됨 |
| **1992년** | SQL-2 | 응용 프로그램에 내장해서 사용하는 삽입 SQL, 다양한 데이터 타입 추가 |
| **1999년** | SQL-3 | 객체 지향 데이터베이스 표준이 추가됨 |

:::note SQL 표준과 각 DBMS
SQL은 국제 표준이 있지만, Oracle, MSSQL, MySQL 등 각 데이터베이스마다 약간씩 문법이 다를 수 있습니다. 그러나 핵심 명령어(SELECT, INSERT, UPDATE, DELETE, CREATE 등)는 대부분 공통적으로 동일하게 사용됩니다.
:::

## 3️⃣ SQL의 핵심 원리 – 명령어 분류와 SQL vs NoSQL

### SQL 명령어의 3가지 분류

SQL 명령어는 크게 **3가지 종류**로 나뉩니다. 이 분류를 이해하면 SQL 전체 구조가 한눈에 보입니다.

#### 📌 DDL – 데이터 정의어 (Data Definition Language)

**DDL**은 테이블의 구조(뼈대)를 만들고, 수정하고, 삭제하는 명령어입니다. 마치 집을 짓기 전에 설계도를 그리고, 방을 추가하거나 없애는 것과 같습니다.

| 명령어 | 기능 | 비유 |
|---|---|---|
| `CREATE` | 테이블 생성 | 새 방 만들기 |
| `ALTER` | 테이블 수정 | 방 구조 변경하기 |
| `DROP` | 테이블 삭제 | 방 없애기 |

#### 📌 DML – 데이터 조작어 (Data Manipulation Language)

**DML**은 이미 만들어진 테이블 안의 데이터를 다루는 명령어입니다. 집이 완성된 후 그 안에서 가구를 넣고, 빼고, 바꾸고, 찾는 작업에 해당합니다.

| 명령어 | 기능 | 비유 |
|---|---|---|
| `INSERT` | 데이터 삽입 | 방에 가구 넣기 |
| `UPDATE` | 데이터 수정 | 가구 교체하기 |
| `DELETE` | 데이터 삭제 | 가구 버리기 |
| `SELECT` | 데이터 검색 | 원하는 가구 찾기 |

#### 📌 DCL – 데이터 제어어 (Data Control Language)

**DCL**은 데이터베이스의 권한을 관리하고, 데이터 무결성(데이터가 정확하고 일관되게 유지되는 것)을 지키기 위한 명령어입니다.

| 명령어 | 기능 |
|---|---|
| `GRANT` | 권한 부여 |
| `REVOKE` | 권한 회수 |
| `COMMIT` | 작업 확정(저장) |
| `ROLLBACK` | 작업 취소(되돌리기) |

:::tip 명령어를 배우는 순서
실제 학습할 때는 **DDL → DML → DCL** 순서로 배우는 것이 자연스럽습니다. 테이블을 먼저 만들어야(DDL) 데이터를 넣고 조작할 수 있고(DML), 그 과정에서 권한과 무결성 제어(DCL)가 필요하기 때문입니다.
:::

### SQL vs NoSQL – 어떻게 다른가?

최근 빅데이터 시대가 되면서 **NoSQL**이라는 개념이 등장했습니다. NoSQL은 **Not Only SQL**의 약자로, 기존 관계형 데이터베이스(SQL)와 다른 방식의 데이터베이스를 말합니다.

#### 🔵 SQL (관계형 데이터베이스)

SQL은 **관계형 데이터베이스**를 대상으로 동작합니다. 관계형 데이터베이스는 데이터를 **테이블(표)** 형태로 저장하며, 다음과 같은 **ACID 특성**을 반드시 만족해야 합니다.

| 특성 | 영어 | 설명 |
|---|---|---|
| **원자성** | Atomicity | 작업이 완전히 실행되거나, 전혀 실행되지 않아야 함 (중간에 끊기면 안 됨) |
| **일관성** | Consistency | 처리 전후로 데이터가 항상 정확하고 일관된 상태여야 함 |
| **고립성** | Isolation | 한 작업이 처리되는 동안 다른 사용자가 그 데이터에 간섭하면 안 됨 |
| **지속성** | Durability | 한 번 저장된 데이터는 하드디스크에 영구적으로 남아야 함 |

:::info ACID는 나중에 더 자세히!
ACID 특성은 **트랜잭션(작업 묶음)과 병행 제어** 단원에서 더 자세하게 다룹니다. 지금은 "관계형 데이터베이스는 원자성, 일관성, 고립성, 지속성이라는 4가지 특성을 가져야 한다"는 정도만 기억하세요.
:::

#### 🟠 NoSQL (비관계형 데이터베이스)

NoSQL은 **비관계형 데이터베이스**를 대상으로 동작합니다. 최근의 빅데이터(대용량 데이터)처럼 테이블 형태로 정리하기 어려운 데이터(텍스트, 이미지, 위치 정보 등)를 다룰 때 사용합니다.

예를 들어, 쇼핑몰 상품평을 생각해보세요. "이 제품 정말 좋아요! 배송도 빠르고 품질도 훌륭합니다"와 같은 자유로운 텍스트 데이터는 정해진 칸이 있는 테이블에 깔끔하게 넣기 어렵습니다. 이런 비정형 데이터를 분석하는 데 NoSQL이 적합합니다.

NoSQL은 **BASE 특성**을 따릅니다.

| 특성 | 설명 |
|---|---|
| **가용성 (Availability)** | 데이터를 언제든지 이용할 수 있어야 함 |
| **소프트 스테이트 (Soft State)** | 사용자가 관리하지 않으면 데이터가 영속성을 가지지 못할 수 있음 (만료 가능) |
| **최종 일관성 (Eventually Consistent)** | 지금 당장 완전한 일관성보다는, 언젠가는 일관성을 갖추는 방향으로 동작 |

#### SQL과 NoSQL 비교 정리

| 구분 | SQL (관계형) | NoSQL (비관계형) |
|---|---|---|
| **데이터 형태** | 테이블 (행과 열) | 텍스트, 이미지, JSON 등 다양한 형태 |
| **핵심 가치** | 정확성, 일관성 | 가용성, 확장성, 처리 속도 |
| **정규화** | 필수 (무결성 유지) | 불가능하거나 어려움 |
| **적합한 상황** | 금융, 행정, 일반 업무 | 빅데이터, 실시간 분석, SNS |
| **대표 제품** | Oracle, MySQL, MSSQL | MongoDB, Cassandra, Redis |

:::warning NoSQL과 빅데이터의 관계
NoSQL은 빅데이터에 자주 사용되지만, **빅데이터 처리를 위해 반드시 NoSQL이 필요한 것은 아닙니다.** 빅데이터를 처리하는 방법에는 NoSQL 외에도 다양한 방법이 존재합니다. 둘을 혼동하지 않도록 주의하세요.
:::

## 4️⃣ 실제 SQL 명령어 예시 – DDL (테이블 생성, 수정, 삭제)

### CREATE TABLE – 테이블 생성

테이블을 만드는 명령어는 `CREATE TABLE`입니다. 마치 엑셀에서 새 시트를 만들고 열 제목을 정하는 것과 같습니다.

#### 기본 문법

```sql
-- CREATE TABLE 기본 형식
CREATE TABLE 테이블이름 (
    속성명1 데이터타입 [NOT NULL],          -- 필수 입력 속성
    속성명2 데이터타입 [DEFAULT 기본값],    -- 기본값 설정
    속성명3 데이터타입,
    PRIMARY KEY (기본키속성명),             -- 기본키 설정
    UNIQUE (대체키속성명),                  -- 대체키(유일값) 설정
    FOREIGN KEY (외래키속성명)              -- 외래키 설정
        REFERENCES 참조테이블명(참조속성명),
    CHECK (조건식)                          -- 값 범위 제약 조건
);
```

각 구성 요소를 하나씩 설명하겠습니다.

- **`NOT NULL`**: 이 속성은 반드시 값을 입력해야 합니다. 비어있으면 안 된다는 의미입니다. 예를 들어 이름이 없는 학생은 존재할 수 없으니 이름 속성에는 `NOT NULL`을 붙입니다.
- **`DEFAULT 값`**: 값을 입력하지 않을 경우 자동으로 들어갈 기본값을 지정합니다. 예를 들어 적립금을 입력하지 않으면 자동으로 0이 들어가게 할 때 사용합니다.
- **`PRIMARY KEY`**: 기본키(테이블에서 각 행을 유일하게 구별하는 값)를 지정합니다. 학번, 주민등록번호, 사원번호, 상품코드 등이 대표적입니다. 기본키는 중복될 수 없고 반드시 값이 있어야 합니다.
- **`UNIQUE`**: 대체키(후보키 중 기본키가 아닌 것)를 지정합니다. 유일한 값이어야 하지만, `NULL` 값은 허용합니다.
- **`FOREIGN KEY ~ REFERENCES`**: 외래키(다른 테이블의 기본키를 참조하는 속성)를 설정합니다. 참조 무결성(두 테이블 간의 데이터 일관성)을 지키기 위해 사용합니다.
- **`CHECK`**: 특정 속성에 들어올 수 있는 값의 범위를 제한합니다. 예를 들어 재고량이 0 이상 10,000 이하여야 한다는 조건을 걸 수 있습니다.

:::note SQL 명령어 대소문자 구분
SQL 명령어(`SELECT`, `CREATE`, `FROM` 등)는 **대소문자를 구분하지 않습니다.** `select`와 `SELECT`는 동일합니다. 그러나 **문자열 데이터 값**은 대소문자를 구분합니다. `'Korea'`와 `'korea'`는 서로 다른 값입니다.
:::

#### ✅ 고객 테이블 생성 예시

아래 조건으로 고객 테이블을 만들어봅니다.
- 속성: 고객아이디, 고객이름, 나이, 등급, 직업, 적립금
- 기본키: 고객아이디
- 고객이름과 등급은 반드시 입력
- 적립금은 입력하지 않으면 기본값 0

```sql
-- 고객 테이블 생성
CREATE TABLE 고객 (
    고객아이디  VARCHAR(20)  NOT NULL,   -- 고객 아이디, 최대 20자, 필수 입력
    고객이름    VARCHAR(10)  NOT NULL,   -- 고객 이름, 최대 10자, 필수 입력
    나이        INTEGER,                 -- 나이, 정수형, 생략 가능
    등급        VARCHAR(10)  NOT NULL,   -- 등급, 최대 10자, 필수 입력
    직업        VARCHAR(20),             -- 직업, 최대 20자, 생략 가능
    적립금      INTEGER      DEFAULT 0,  -- 적립금, 정수형, 기본값 0
    PRIMARY KEY (고객아이디)             -- 고객아이디를 기본키로 설정
);
```

#### ✅ 제품 테이블 생성 예시 (CHECK 제약 조건 포함)

```sql
-- 제품 테이블 생성 (재고량 범위 제약 포함)
CREATE TABLE 제품 (
    제품번호    CHAR(3)      NOT NULL,   -- 제품번호, 3자리 고정 문자, 필수 입력
    제품명      VARCHAR(20),             -- 제품명, 최대 20자 (한글 10자)
    재고량      INTEGER,                 -- 재고량, 정수형
    단가        INTEGER,                 -- 단가, 정수형
    제조업체    VARCHAR(20),             -- 제조업체명, 최대 20자
    PRIMARY KEY (제품번호),             -- 제품번호를 기본키로 설정
    CHECK (재고량 >= 0 AND 재고량 <= 10000)  -- 재고량은 0 이상 10,000 이하만 허용
);
```

:::tip 한글 데이터 타입 크기 계산
`VARCHAR(20)`은 최대 20바이트를 저장할 수 있습니다. 영문은 1글자 = 1바이트이지만, **한글은 1글자 = 2바이트**입니다. 따라서 `VARCHAR(20)`이면 한글은 최대 10글자까지 저장할 수 있습니다.
:::

#### ✅ 외래키가 포함된 주문 테이블 생성 예시

외래키가 2개 포함된 주문 테이블을 만들어봅니다.

```sql
-- 주문 테이블 생성 (외래키 2개 포함)
CREATE TABLE 주문 (
    주문번호    INTEGER      NOT NULL,   -- 주문번호, 정수형, 필수 입력
    주문고객    VARCHAR(20),             -- 주문한 고객 아이디 (외래키)
    주문제품    CHAR(3),                 -- 주문한 제품 번호 (외래키)
    수량        INTEGER,                 -- 주문 수량
    배송지      VARCHAR(30),             -- 배송 주소
    주문일자    DATE,                    -- 주문 날짜
    PRIMARY KEY (주문번호),             -- 주문번호를 기본키로 설정
    FOREIGN KEY (주문고객)              -- 주문고객은 외래키
        REFERENCES 고객(고객아이디),    -- 고객 테이블의 고객아이디를 참조
    FOREIGN KEY (주문제품)              -- 주문제품도 외래키
        REFERENCES 제품(제품번호)       -- 제품 테이블의 제품번호를 참조
);
```

이 예시에서 `주문고객`은 `고객` 테이블의 `고객아이디`를 참조하고, `주문제품`은 `제품` 테이블의 `제품번호`를 참조합니다. 이렇게 외래키를 설정하면 존재하지 않는 고객 아이디나 제품 번호로 주문을 넣으려 할 때 오류가 발생하여 데이터의 정확성이 보장됩니다.

#### 외래키 옵션 정리

외래키에는 참조되는 데이터가 변경되거나 삭제될 때 어떻게 처리할지 옵션을 지정할 수 있습니다.

| 옵션 | 설명 | 비유 |
|---|---|---|
| `NO ACTION` / `RESTRICT` | 참조하는 데이터가 있으면 삭제/수정을 거부 | "아직 참조 중이라 못 지워요" |
| `CASCADE` | 참조되는 데이터가 삭제/수정되면 참조하는 데이터도 함께 삭제/수정 | "연결된 것도 같이 삭제" |
| `SET NULL` | 참조되는 데이터가 삭제/수정되면 외래키 값을 NULL로 변경 | "참조 대상이 사라지면 빈칸으로" |
| `SET DEFAULT` | 참조되는 데이터가 삭제/수정되면 외래키 값을 미리 정해둔 기본값으로 변경 | "기본값으로 대체" |

---

### ALTER TABLE – 테이블 수정

이미 만들어진 테이블에 새 속성을 추가하거나, 기존 속성을 삭제하거나, 제약 조건을 추가/삭제할 때 `ALTER TABLE`을 사용합니다. 마치 이미 지어진 집에 방을 추가하거나, 필요 없는 방을 없애는 것과 같습니다.

#### ✅ 속성 추가 – ADD

```sql
-- 고객 테이블에 '가입날짜' 속성 추가
ALTER TABLE 고객                         -- 수정할 테이블: 고객
ADD 가입날짜 DATETIME;                   -- 가입날짜 속성을 날짜형으로 추가
```

#### ✅ 속성 삭제 – DROP

```sql
-- 고객 테이블에서 '등급' 속성 삭제 (연쇄 삭제 옵션 포함)
ALTER TABLE 고객                         -- 수정할 테이블: 고객
DROP 등급 CASCADE;                       -- 등급 속성 삭제, 다른 테이블에서 참조 중이면 함께 삭제
```

`CASCADE` 대신 `RESTRICT`를 쓰면, 다른 테이블에서 이 속성을 참조하고 있을 경우 삭제가 거부됩니다.

#### ✅ 제약 조건 추가 – ADD CONSTRAINT

```sql
-- 고객 테이블에 '나이 20세 이상' 제약 조건 추가
ALTER TABLE 고객                          -- 수정할 테이블: 고객
ADD CONSTRAINT check_age                  -- 제약 조건 이름: check_age
CHECK (나이 >= 20);                       -- 나이는 20 이상이어야 한다는 조건
```

#### ✅ 제약 조건 삭제 – DROP CONSTRAINT

```sql
-- 아까 추가한 'check_age' 제약 조건 삭제
ALTER TABLE 고객                          -- 수정할 테이블: 고객
DROP CONSTRAINT check_age;               -- check_age라는 이름의 제약 조건 삭제
```

:::tip 제약 조건을 추가할 때와 삭제할 때의 차이
제약 조건을 **추가**할 때는 조건의 내용을 자세하게 써줘야 하고, **삭제**할 때는 제약 조건 이름만 써주면 됩니다. 그래서 제약 조건을 만들 때 이름을 잘 지어두는 것이 중요합니다.
:::

---

### DROP TABLE – 테이블 삭제

테이블 자체를 완전히 없앨 때 `DROP TABLE`을 사용합니다. 이 명령어는 테이블 안의 데이터뿐 아니라 테이블 구조(설계 자체)까지 모두 삭제합니다.

```sql
-- 고객 테이블 삭제 (다른 테이블이 참조 중이면 삭제 거부)
DROP TABLE 고객 RESTRICT;

-- 고객 테이블 삭제 (다른 테이블도 함께 삭제)
DROP TABLE 고객 CASCADE;
```

:::danger DROP TABLE은 되돌릴 수 없습니다
`DROP TABLE`은 테이블과 그 안의 모든 데이터를 영구적으로 삭제합니다. 실수로 중요한 테이블을 삭제하면 복구가 매우 어렵습니다. 실무에서는 반드시 백업 후에 사용하세요.

또한, 옵션을 생략하면 **기본값은 `RESTRICT`**입니다. 데이터 무결성을 지키기 위해 다른 테이블에서 참조 중인 테이블은 삭제를 거부합니다.
:::

## 5️⃣ 실제 SQL 명령어 예시 – SELECT (데이터 조회)

### SELECT 기본 구조

`SELECT`는 가장 자주 사용하는 명령어입니다. 데이터베이스에서 원하는 데이터를 조회(검색)할 때 사용합니다.

```sql
-- SELECT 기본 형식
SELECT [DISTINCT] 속성명1, 속성명2, ...   -- 출력할 속성 목록 (DISTINCT: 중복 제거)
FROM 테이블명                              -- 조회할 테이블
WHERE 조건식                               -- 검색 조건
GROUP BY 속성명                            -- 그룹화 기준 속성
HAVING 조건식                              -- 그룹에 대한 조건
ORDER BY 속성명 [ASC | DESC];             -- 정렬 기준 (ASC: 오름차순, DESC: 내림차순)
```

각 절(clause)의 역할을 정리하면 다음과 같습니다.

| 절 | 역할 | 비유 |
|---|---|---|
| `SELECT` | 출력할 속성(열) 지정 | "어떤 정보를 볼 건지" |
| `DISTINCT` | 중복된 결과 제거 | "같은 것은 한 번만 보여줘" |
| `FROM` | 데이터를 가져올 테이블 지정 | "어느 창고에서 가져올지" |
| `WHERE` | 검색 조건 지정 | "어떤 조건에 맞는 것만" |
| `GROUP BY` | 특정 속성을 기준으로 그룹화 | "부서별로, 학년별로 묶어서" |
| `HAVING` | 그룹에 대한 조건 | "그룹 중 이 조건을 만족하는 것만" |
| `ORDER BY` | 결과 정렬 | "이 순서대로 나열해줘" |

### SELECT 사용 예시

아래 예시들은 `employee`(사원) 테이블과 `department`(부서) 테이블을 기준으로 합니다.

#### ✅ 모든 데이터 조회

```sql
-- department 테이블의 모든 데이터 조회
SELECT *            -- * 는 모든 속성을 의미
FROM department;    -- department 테이블에서 가져옴
```

`*`는 "모든 열을 출력해라"는 의미입니다. 마치 엑셀에서 모든 열을 표시하는 것과 같습니다.

#### ✅ 특정 속성만 조회

```sql
-- department 테이블에서 부서번호와 부서명만 조회 (층수 제외)
SELECT deptno, deptname   -- 출력할 속성만 명시
FROM department;          -- department 테이블에서
```

#### ✅ 중복 제거하여 조회 (DISTINCT)

```sql
-- employee 테이블에서 직급 목록 조회 (중복 제거)
SELECT DISTINCT title    -- DISTINCT로 중복된 직급은 한 번만 표시
FROM employee;           -- employee 테이블에서
```

`DISTINCT`가 없으면 같은 직급이 여러 번 반복해서 나타납니다. `DISTINCT`를 붙이면 중복이 제거되어 고유한 직급 목록만 나타납니다.

#### ✅ 조건을 사용한 조회 (WHERE)

```sql
-- 부서번호가 2번인 사원의 모든 정보 조회
SELECT *                   -- 모든 속성 출력
FROM employee              -- employee 테이블에서
WHERE dno = 2;             -- 조건: 부서번호(dno)가 2인 것만
```

```sql
-- 직급이 과장이면서 1번 부서에 속한 사원의 이름과 급여 조회
SELECT empname, salary     -- 이름과 급여만 출력
FROM employee              -- employee 테이블에서
WHERE title = '과장'       -- 조건1: 직급이 과장
AND dno = 1;               -- 조건2: 부서번호가 1 (AND: 두 조건 모두 만족)
```

#### ✅ 부정 조건 조회

```sql
-- 직급이 과장이지만 1번 부서가 아닌 사원의 이름과 급여 조회
SELECT empname, salary     -- 이름과 급여 출력
FROM employee              -- employee 테이블에서
WHERE title = '과장'       -- 조건1: 직급이 과장
AND dno <> 1;              -- 조건2: 부서번호가 1이 아닌 것 (<>는 같지 않다는 의미)
```

:::warning SQL에서 같지 않다는 표현
SQL에서 "같지 않다"는 `!=` 또는 `<>` 로 표현합니다. 일반적인 프로그래밍에서 쓰는 `≠` 기호는 SQL에서 사용할 수 없습니다.
:::

#### ✅ BETWEEN – 범위 조건

```sql
-- 급여가 300만원 이상 450만원 이하인 사원의 이름, 직급, 급여 조회

-- 방법 1: BETWEEN 사용
SELECT empname, title, salary    -- 이름, 직급, 급여 출력
FROM employee                    -- employee 테이블에서
WHERE salary BETWEEN 300 AND 450; -- salary가 300 이상 450 이하 (300, 450 포함)

-- 방법 2: 비교 연산자 사용 (위와 동일한 결과)
SELECT empname, title, salary    -- 이름, 직급, 급여 출력
FROM employee                    -- employee 테이블에서
WHERE salary >= 300              -- salary가 300 이상
AND salary <= 450;               -- AND salary가 450 이하
```

`BETWEEN A AND B`는 A와 B 값을 포함한 범위입니다. `BETWEEN 60 AND 90`이면 60, 61, 62... 89, 90이 모두 포함됩니다.

#### ✅ IN – 여러 값 중 하나

```sql
-- 1번 부서 또는 3번 부서에 속한 사원의 모든 정보 조회
SELECT *                          -- 모든 속성 출력
FROM employee                     -- employee 테이블에서
WHERE dno IN (1, 3);              -- dno가 1이거나 3인 것 (OR과 같은 의미)
```

`IN`은 OR의 축약 표현입니다. `dno IN (1, 3)`은 `dno = 1 OR dno = 3`과 동일합니다. 여러 값 중 하나라도 일치하면 조건을 만족합니다.

#### ✅ LIKE – 문자열 패턴 검색

```sql
-- 이름이 '이'로 시작하는 사원의 이름, 직급, 부서번호 조회
SELECT empname, title, dno        -- 이름, 직급, 부서번호 출력
FROM employee                     -- employee 테이블에서
WHERE empname LIKE '이%';         -- 이름이 '이'로 시작하는 것 (% = 어떤 문자든 0개 이상)
```

`LIKE`에서 사용하는 특수 기호:
- **`%`** : 어떤 문자든 0개 이상의 문자를 의미합니다. `'이%'`는 "이로 시작하는 모든 것", `'%김'`은 "김으로 끝나는 모든 것", `'%서울%'`은 "서울이 포함된 모든 것"을 의미합니다.
- **`_`(언더바)** : 정확히 1개의 문자를 의미합니다. `'KOR__'`는 "KOR으로 시작하는 5글자" 단어를 의미합니다.

#### ✅ SELECT 열에 수식 사용하기

```sql
-- 직급이 과장인 사원의 이름, 현재 급여, 10% 인상 후 급여 조회
SELECT empname,           -- 이름 출력
       salary,            -- 현재 급여 출력
       salary * 1.1       -- 현재 급여의 1.1배 = 10% 인상된 급여 출력
FROM employee             -- employee 테이블에서
WHERE title = '과장';     -- 조건: 직급이 과장
```

`SELECT` 절에 단순 속성명뿐 아니라 **수식**도 사용할 수 있습니다. `salary * 1.1`은 급여에 1.1을 곱한 값(10% 인상된 값)을 출력합니다. **이 명령은 실제 데이터를 변경하지 않고 화면에 계산된 값을 보여줄 뿐입니다.** 실제 데이터를 변경하려면 `UPDATE` 명령을 사용해야 합니다.

### WHERE 절의 다양한 연산자 정리

```sql
-- 비교 연산자 예시
WHERE score = 80      -- score가 80과 같다
WHERE score > 80      -- score가 80보다 크다
WHERE score >= 80     -- score가 80 이상
WHERE score < 80      -- score가 80보다 작다
WHERE score <= 80     -- score가 80 이하
WHERE score <> 80     -- score가 80이 아니다 (같지 않다)

-- SQL 전용 연산자 예시
WHERE score BETWEEN 60 AND 90    -- score가 60 이상 90 이하
WHERE dno IN (1, 2, 3)           -- dno가 1, 2, 3 중 하나
WHERE name LIKE '김%'            -- name이 '김'으로 시작
WHERE address IS NULL            -- address 값이 NULL(비어있음)
WHERE address IS NOT NULL        -- address 값이 NULL이 아님

-- 논리 연산자 예시
WHERE title = '과장' AND dno = 1    -- 두 조건 모두 만족
WHERE dno = 1 OR dno = 3           -- 둘 중 하나만 만족해도 됨
WHERE NOT title = '사원'            -- 직급이 사원이 아닌 것
WHERE score NOT BETWEEN 60 AND 90  -- score가 60~90 범위 밖
WHERE dno NOT IN (1, 3)            -- dno가 1도 아니고 3도 아닌 것
```

**연산자 우선순위**: 괄호`()` → `NOT` → 비교 연산자(`=`, `>`, `<` 등) → `AND` → `OR` 순으로 처리됩니다. 복잡한 조건은 괄호를 사용해서 명확하게 표현하는 것이 좋습니다.

## 6️⃣ 주의사항과 실수하기 쉬운 포인트

### ⚠️ 자주 하는 실수들

#### ❌ NOT NULL과 NULL의 혼동

```sql
-- ❌ 잘못된 예: NULL을 = 로 비교 (동작하지 않음)
SELECT * FROM 고객
WHERE 직업 = NULL;       -- NULL은 = 으로 비교할 수 없음!

-- ✅ 올바른 예: IS NULL 사용
SELECT * FROM 고객
WHERE 직업 IS NULL;      -- NULL 값 비교는 반드시 IS NULL 사용
```

NULL은 "값이 없음"을 의미하는 특수한 상태입니다. 일반적인 비교 연산자(`=`, `<>`)로는 NULL을 비교할 수 없습니다. 반드시 `IS NULL` 또는 `IS NOT NULL`을 사용해야 합니다.

#### ❌ PRIMARY KEY와 UNIQUE의 혼동

```sql
-- PRIMARY KEY: 반드시 유일 + NULL 허용 안 함
PRIMARY KEY (학번)      -- 학번은 유일하고 반드시 값이 있어야 함

-- UNIQUE: 유일하지만 NULL 허용 가능
UNIQUE (이메일)         -- 이메일은 유일하지만, 비워둘 수 있음
```

**PRIMARY KEY**는 NULL을 허용하지 않고, **UNIQUE**는 NULL을 허용합니다. 이 차이를 잘 구분해야 합니다.

#### ❌ CASCADE 옵션의 무분별한 사용

```sql
-- ❌ 위험한 예: 부주의한 CASCADE 사용
DROP TABLE 고객 CASCADE;   -- 고객 테이블을 참조하는 모든 테이블도 함께 삭제됨!

-- ✅ 안전한 예: RESTRICT 사용 (참조 중이면 삭제 거부)
DROP TABLE 고객 RESTRICT;  -- 다른 테이블이 참조 중이면 삭제되지 않음
```

`CASCADE` 옵션은 편리하지만, 연쇄적으로 여러 테이블의 데이터나 테이블 자체가 삭제될 수 있습니다. 실무에서는 신중하게 사용해야 합니다.

#### ❌ SELECT 결과가 실제 데이터를 변경한다는 오해

```sql
-- ❌ 오해: 이 명령이 실제 급여를 올려준다고 생각하는 것
SELECT salary * 1.1 FROM employee;  -- 이건 화면 출력만 할 뿐, 실제 데이터 변경 없음

-- ✅ 실제 데이터 변경은 UPDATE 사용
UPDATE employee
SET salary = salary * 1.1           -- 실제 salary 데이터를 10% 올림
WHERE title = '과장';
```

:::danger SELECT는 데이터를 변경하지 않습니다
`SELECT` 명령은 **조회 전용**입니다. 수식을 사용해 계산된 값을 보여줄 수 있지만, 실제 데이터베이스의 값은 변경되지 않습니다. 데이터를 실제로 변경하려면 반드시 `UPDATE` 명령을 사용해야 합니다.
:::

### 💡 실무에서 유용한 팁

**① SQL 문장은 세미콜론(`;`)으로 끝납니다.**
모든 SQL 문장의 끝에는 반드시 `;`를 붙여야 합니다. 빠뜨리면 오류가 발생할 수 있습니다.

**② 명령어는 보기 좋게 여러 줄로 작성하세요.**
SQL은 줄바꿈을 구분하지 않으므로 한 줄에 모두 써도 되고, 여러 줄로 나눠 써도 됩니다. 하지만 가독성을 위해 여러 줄로 나눠 쓰는 것이 좋습니다.

**③ 기본키는 테이블당 반드시 하나만 설정합니다.**
기본키(`PRIMARY KEY`)는 테이블당 1개만 존재할 수 있습니다. 단, 복합키(여러 속성을 조합한 기본키)는 가능합니다.

**④ 외래키는 반드시 참조 대상 테이블이 먼저 생성되어 있어야 합니다.**
`주문` 테이블이 `고객` 테이블을 외래키로 참조한다면, `주문` 테이블보다 `고객` 테이블이 먼저 생성되어 있어야 합니다.

## 📝 핵심 정리

이 문서에서 배운 핵심 개념을 정리합니다.

- **SQL(Structured Query Language)** 은 관계형 데이터베이스를 다루기 위한 표준 질의 언어로, 실무에서 "시퀄"이라고 읽는다.
- SQL은 **고급 언어, 비절차적 언어**이며 결과가 테이블(레코드 집합) 형태로 반환된다.
- SQL은 **대화식(직접 접속)** 방식과 **내장(프로그램 내 삽입)** 방식 모두 사용 가능하다.
- SQL 명령어는 **DDL(정의어)**, **DML(조작어)**, **DCL(제어어)** 3가지로 분류된다.
- **DDL 명령어**: `CREATE`(생성), `ALTER`(수정), `DROP`(삭제)
- **DML 명령어**: `INSERT`(삽입), `UPDATE`(수정), `DELETE`(삭제), `SELECT`(조회)
- **SQL(관계형)** 은 정확성·일관성(ACID)을 중시하고, **NoSQL(비관계형)** 은 가용성·확장성(BASE)을 중시한다.
- `CREATE TABLE`의 주요 옵션: `NOT NULL`(필수 입력), `DEFAULT`(기본값), `PRIMARY KEY`(기본키), `UNIQUE`(대체키), `FOREIGN KEY`(외래키), `CHECK`(범위 제약)
- 외래키 옵션: `RESTRICT`(거부), `CASCADE`(연쇄 삭제/수정), `SET NULL`(NULL로 변경), `SET DEFAULT`(기본값으로 변경)
- `SELECT`의 구조: `SELECT` → `FROM` → `WHERE` → `GROUP BY` → `HAVING` → `ORDER BY`
- `DISTINCT`는 중복된 결과를 제거하고, `*`는 모든 속성을 의미한다.
- `WHERE` 절에서 사용하는 연산자: `BETWEEN`, `IN`, `LIKE`, `IS NULL`, `AND`, `OR`, `NOT`
- `NULL` 비교는 반드시 `IS NULL` / `IS NOT NULL`을 사용해야 하며, `= NULL`은 동작하지 않는다.
- `SELECT` 절에 수식을 사용할 수 있지만, 이는 화면 출력만 할 뿐 실제 데이터를 변경하지 않는다.
- SQL 명령어는 **대소문자를 구분하지 않지만**, 문자열 데이터 값은 대소문자를 구분한다.
- SQL 표준은 1986년 최초 제정 이후 SQL-1(1989), SQL-2(1992), SQL-3(1999)으로 발전해왔다.
