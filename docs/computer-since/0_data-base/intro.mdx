# 데이터베이스의 개요

## 데이터베이스란 무엇인가?

데이터베이스(Database)는 **여러 사람이 공유하여 사용할 목적으로 체계적으로 관리되는 데이터의 집합**입니다.

데이터베이스가 등장한 배경은 **방대한 양의 데이터를 효율적으로 저장하고, 필요할 때 빠르게 찾아내기 위해서**였습니다.  
우리가 은행에서 계좌를 조회하거나, 온라인 쇼핑을 하거나, SNS를 사용하는 것들이 모두 데이터베이스와 밀접하게 연관되어 있습니다.

:::tip 핵심 개념
데이터베이스를 배우는 목적은 **대량의 데이터를 체계적으로 관리하고 효율적으로 활용**하는 것입니다.
:::

## 일상생활 속 데이터베이스

우리 주변에서 쉽게 찾아볼 수 있는 데이터베이스의 예시들입니다:

### 실생활 vs 데이터베이스 대응

| 일상생활 | 데이터베이스 활용 | 설명 |
|---------|-----------------|------|
| 은행 계좌 조회 | 고객 정보 DB | 계좌번호로 잔액과 거래내역을 빠르게 조회 |
| 온라인 쇼핑 | 상품 정보 DB | 수많은 상품 중 원하는 것을 검색하고 주문 |
| 병원 진료 | 환자 정보 DB | 과거 진료기록과 처방전을 관리 |
| 도서관 | 도서 정보 DB | 책의 위치, 대출 여부를 빠르게 확인 |
| SNS (인스타그램, 페이스북) | 사용자·게시물 DB | 친구 목록, 사진, 댓글 등을 저장하고 표시 |
| 포털 사이트 검색 | 웹 페이지 색인 DB | 검색어에 맞는 수억 개의 결과를 순식간에 제공 |
| 학교 성적 관리 | 학생·성적 DB | 학생별 성적, 출결 정보를 체계적으로 관리 |

## 데이터베이스의 주요 특징

### 1. 데이터의 독립성 (Data Independence)
- 데이터의 구조가 변경되어도 응용 프로그램은 영향을 받지 않습니다
- 예: 고객 정보에 이메일 항목이 추가되어도 기존 프로그램은 그대로 작동

### 2. 데이터의 무결성 (Data Integrity)
- 데이터의 정확성과 일관성을 보장합니다
- 예: 은행 계좌 잔액은 항상 정확해야 하고, 마이너스가 될 수 없음

### 3. 데이터의 보안성 (Data Security)
- 허가된 사용자만 데이터에 접근할 수 있습니다
- 예: 본인의 의료기록은 본인과 담당의사만 볼 수 있음

### 4. 데이터의 중복 최소화
- 같은 데이터를 여러 곳에 중복 저장하지 않습니다
- 예: 고객의 주소는 한 곳에만 저장하고 필요한 곳에서 참조

### 5. 데이터 공유 (Data Sharing)
- 여러 사용자가 동시에 데이터를 사용할 수 있습니다
- 예: 여러 직원이 동시에 재고 현황을 조회

:::info 데이터베이스 vs 파일 시스템
예전에는 엑셀 파일이나 텍스트 파일로 데이터를 관리했습니다.  
하지만 **데이터양이 많아지고, 여러 사람이 동시에 사용**해야 하면서 데이터베이스 시스템이 필수가 되었습니다.
:::

## 데이터베이스 관리 시스템 (DBMS)

### DBMS란?
**DBMS**(Database Management System)는 데이터베이스를 관리하고 운영하는 소프트웨어입니다.

### 대표적인 DBMS

| DBMS | 유형 | 주요 사용처 |
|------|------|-----------|
| MySQL | 관계형 | 웹 사이트, 중소규모 서비스 |
| Oracle | 관계형 | 대기업, 금융권, 대규모 시스템 |
| PostgreSQL | 관계형 | 공공기관, 연구 분야 |
| MongoDB | NoSQL | SNS, 빅데이터, 실시간 분석 |
| MariaDB | 관계형 | MySQL의 대안으로 많이 사용 |

### DBMS가 하는 일

**데이터 정의** (Definition)
- 데이터베이스의 구조를 정의합니다
- 예: 학생 테이블에는 학번, 이름, 학과 정보가 있다

**데이터 조작** (Manipulation)
- 데이터를 저장, 수정, 삭제, 조회합니다
- 예: 새로운 학생 정보를 입력하거나 성적을 수정

**데이터 제어** (Control)
- 데이터의 보안과 무결성을 관리합니다
- 예: 학생은 자신의 성적만 볼 수 있고, 교수는 담당 과목 학생들의 성적을 수정 가능

## 데이터베이스의 발전 과정

### 1960년대 - 파일 시스템
- 각 응용 프로그램마다 별도의 파일로 데이터 관리
- **문제점**: 데이터 중복, 불일치 발생

### 1970년대 - 관계형 데이터베이스 등장
- 테이블(표) 형태로 데이터를 관리
- **장점**: 데이터 중복 최소화, 체계적 관리

### 1990년대 - 객체지향 데이터베이스
- 복잡한 데이터(멀티미디어 등)도 저장 가능
- **특징**: 프로그래밍 언어와 통합

### 2000년대 이후 - 빅데이터 시대
- NoSQL, 분산 데이터베이스 등장
- **특징**: 대용량 데이터를 빠르게 처리

### 현재 - 클라우드 데이터베이스
- 인터넷을 통해 어디서나 접근 가능
- **예시**: AWS RDS, Google Cloud SQL, Azure SQL

## 이 과목에서 배울 내용

### 데이터베이스 설계
- 효율적인 데이터베이스 구조를 설계하는 방법
- ER 다이어그램, 정규화 등

### SQL (Structured Query Language)
- 데이터를 조회하고 관리하는 표준 언어
- SELECT, INSERT, UPDATE, DELETE 등

### 데이터 모델링
- 현실 세계의 데이터를 데이터베이스로 표현하는 방법
- 개체-관계 모델, 관계형 모델 등

### 데이터베이스 관리
- 트랜잭션, 동시성 제어, 회복, 보안 등
- 데이터의 무결성과 안정성 보장

### 고급 주제
- 분산 데이터베이스, 데이터 웨어하우스
- 빅데이터, NoSQL, 클라우드 DB 등

## 데이터베이스를 배워야 하는 이유

### 모든 IT 시스템의 핵심
- 웹/앱 개발, 인공지능, 빅데이터 분석 모두 데이터베이스가 기반

### 취업 경쟁력 향상
- 개발자 채용 시 SQL과 데이터베이스 지식은 필수

### 데이터 기반 의사결정
- 기업의 모든 의사결정이 데이터에 기반하는 시대

### 디지털 세상의 이해
- 우리가 사용하는 모든 디지털 서비스의 작동 원리 파악

:::info 데이터베이스는 어렵지 않아요!
처음에는 생소한 용어들이 많아 어렵게 느껴질 수 있습니다.  
하지만 **실생활의 사례와 연결**하여 이해하면 누구나 쉽게 배울 수 있습니다.  
천천히, 단계적으로 함께 공부해 나가면 됩니다!
:::
