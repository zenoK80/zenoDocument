# 버스 시스템

## 버스의 개념과 역할

버스(Bus)는 컴퓨터 시스템의 각 구성 요소를 연결하여 데이터를 전송하는 통로입니다.  
CPU, 메모리, 입출력 장치 등이 서로 데이터를 주고받을 수 있도록 하는 중요한 역할을 담당합니다.

버스라는 이름은 여러 명의 승객을 동시에 운반하는 대중교통 버스와 유사하게, 여러 개의 데이터를 한꺼번에 전송할 수 있다는 점에서 유래되었습니다. 컴퓨터를 열어보면 버스 형태로 되어 있는 케이블들을 확인할 수 있습니다.

### 버스의 주요 특징

- **병렬 전송**: 여러 개의 신호선을 통해 동시에 데이터를 전송합니다
- **공유 자원**: 여러 장치가 버스를 공유하여 사용합니다
- **양방향 통신**: 데이터를 송신하거나 수신할 수 있습니다

## 시스템 버스의 종류

시스템 버스는 전송하는 정보의 종류에 따라 세 가지로 분류됩니다.

### 데이터 버스 (Data Bus)

CPU와 메모리, 입출력 장치 간에 실제 데이터를 전송하는 버스입니다.

**주요 특징**:
- 양방향 전송이 가능합니다
- 버스의 폭(bit 수)이 CPU의 처리 능력을 결정합니다
- 일반적으로 8비트, 16비트, 32비트, 64비트 단위로 구성됩니다

**예시**:
- 32비트 데이터 버스 → 한 번에 4바이트 전송 가능
- 64비트 데이터 버스 → 한 번에 8바이트 전송 가능

### 주소 버스 (Address Bus)

CPU가 메모리나 입출력 장치의 특정 위치를 지정하기 위해 사용하는 버스입니다.

**주요 특징**:
- 단방향 전송만 가능합니다 (CPU → 메모리/장치)
- 주소 버스의 폭이 메모리 주소 지정 범위를 결정합니다
- n비트 주소 버스는 2^n개의 주소를 지정할 수 있습니다

**메모리 주소 지정 예시**:
```
20비트 주소 버스 → 2^20 = 1,048,576 (1MB) 주소 지정 가능
32비트 주소 버스 → 2^32 = 4,294,967,296 (4GB) 주소 지정 가능
```

### 제어 버스 (Control Bus)

CPU가 시스템의 각 장치를 제어하기 위한 신호를 전송하는 버스입니다.

**주요 제어 신호**:
- **읽기(Read)**: 메모리나 장치로부터 데이터를 읽어옴
- **쓰기(Write)**: 메모리나 장치에 데이터를 저장
- **인터럽트(Interrupt)**: 장치가 CPU의 처리를 요청
- **클럭(Clock)**: 동기화 신호를 제공

## Front Side Bus (FSB)

Front Side Bus는 CPU와 메인 메모리, 그래픽 카드 등 주요 시스템 구성 요소를 연결하는 버스입니다.

**특징**:
- CPU 외부의 시스템 버스를 의미합니다
- 노스 브릿지를 통해 메모리 및 그래픽 장치와 연결됩니다
- 시스템 성능에 중요한 영향을 미칩니다

**동작 예시**:
```
CPU → FSB → 노스 브릿지 → 메모리
CPU → FSB → 노스 브릿지 → 그래픽 카드
```

## Back Side Bus (BSB)

Back Side Bus는 CPU 내부에서 캐시 메모리와 연결하는 버스입니다.

**특징**:
- CPU 내부 버스로 매우 빠른 속도를 자랑합니다
- L2/L3 캐시와 CPU 코어 간의 데이터 전송을 담당합니다
- FSB보다 훨씬 빠른 속도로 동작합니다

### FSB와 BSB의 속도 차이

CPU 내부의 BSB는 매우 빠르게 동작하지만, 외부로 데이터를 전송하는 FSB는 상대적으로 느립니다. 이러한 속도 차이로 인해 **작업 지연**이 발생할 수 있으며, 이를 해결하기 위해 **캐시(Cache) 메모리**를 사용하여 완충지대 역할을 수행합니다.

## 노스 브릿지와 사우스 브릿지

메인보드의 칩셋은 노스 브릿지와 사우스 브릿지로 구성되어 각기 다른 장치들을 제어합니다.

### 노스 브릿지 (North Bridge)

CPU와 가까운 위치에 있으며, 고속 처리가 필요한 장치들을 제어합니다.

**담당 장치**:
- 메인 메모리 (RAM)
- 그래픽 카드 (AGP/PCI Express)
- FSB 제어

**특징**:
- CPU에 근접하게 배치됩니다
- 방열판이 부착되어 있습니다
- 고속 데이터 전송이 필요한 장치들을 담당합니다

### 사우스 브릿지 (South Bridge)

상대적으로 속도가 느린 입출력 장치들을 제어합니다.

**담당 장치**:
- PCI 슬롯
- USB 포트
- SATA/IDE 컨트롤러
- 사운드 카드
- 네트워크 카드

**연결 구조**:
```
CPU ↔ 노스 브릿지 ↔ 사우스 브릿지 ↔ 입출력 장치
         ↓
    메모리/그래픽
```

## 버스 대역폭 계산

버스 대역폭(Bus Bandwidth)은 버스를 통해 단위 시간당 전송할 수 있는 데이터의 최대 크기를 의미합니다.

### 대역폭 계산 공식

```
버스 대역폭 = 데이터 버스의 폭 × 버스 클럭 주파수
단위: 바이트/초 (Bytes per Second)
```

### 주파수와 주기의 관계

클럭 주파수(f)와 클럭 주기(T)는 반비례 관계에 있습니다.

```
주기(T) = 1 / 주파수(f)

예시:
100MHz = 100 × 10^6 Hz
주기 = 1 / (100 × 10^6) = 10 × 10^-9 초 = 10 나노초
```

### 대역폭 계산 예제

**조건**:
- 데이터 버스 폭: 32비트 (4바이트)
- 버스 클럭 주파수: 100MHz

**계산 과정**:

```javascript
// 1단계: 주파수를 Hz로 변환
주파수 = 100MHz = 100 × 10^6 Hz

// 2단계: 주기 계산
주기 = 1 / (100 × 10^6) = 10 × 10^-9 초 = 10 나노초

// 3단계: 대역폭 계산
버스 폭 = 32비트 = 4바이트
대역폭 = 4 바이트 × 100 × 10^6 Hz
       = 400 × 10^6 바이트/초
       = 400 MB/s
```

**결과**: 이 시스템의 버스 대역폭은 **400MB/s**입니다.

### 다양한 대역폭 계산 예시

| 데이터 버스 폭 | 클럭 주파수 | 대역폭 계산 | 결과 |
|---------------|-------------|-------------|------|
| 32비트 (4바이트) | 100MHz | 4 × 100M | 400 MB/s |
| 64비트 (8바이트) | 200MHz | 8 × 200M | 1,600 MB/s (1.6 GB/s) |
| 64비트 (8바이트) | 800MHz | 8 × 800M | 6,400 MB/s (6.4 GB/s) |
| 128비트 (16바이트) | 1GHz | 16 × 1,000M | 16,000 MB/s (16 GB/s) |

### 클럭(Clock) 개념

클럭은 일정한 시간 간격으로 발생하는 펄스 신호로, 디지털 시스템의 동기화에 사용됩니다.

**클럭 주파수 예시**:
```
1Hz = 1초에 1번 클럭 발생
1KHz = 1초에 1,000번 클럭 발생
1MHz = 1초에 1,000,000번 클럭 발생
1GHz = 1초에 1,000,000,000번 클럭 발생
3.2GHz = 1초에 3,200,000,000번 클럭 발생
```

**CPU 클럭 예시**:
- 3.2GHz CPU의 경우
- 1초에 약 32억 번의 클럭 틱이 발생합니다
- 각 클럭마다 32비트 또는 64비트 데이터를 처리할 수 있습니다

## 정리

버스 시스템은 컴퓨터의 각 구성 요소를 연결하여 데이터를 효율적으로 전송하는 핵심 요소입니다. 데이터 버스, 주소 버스, 제어 버스가 각각의 역할을 수행하며, FSB와 BSB는 서로 다른 속도로 동작하여 시스템의 성능을 결정합니다. 노스 브릿지와 사우스 브릿지는 다양한 장치들을 효율적으로 제어하며, 버스 대역폭 계산을 통해 시스템의 데이터 전송 능력을 파악할 수 있습니다.
