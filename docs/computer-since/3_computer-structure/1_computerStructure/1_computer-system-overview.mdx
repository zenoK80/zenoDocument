# 컴퓨터 시스템 개요

## 컴퓨터 시스템의 구성

컴퓨터 시스템은 크게 하드웨어(Hardware)와 소프트웨어(Software)로 구성됩니다.  
이 두 요소는 서로 유기적으로 연결되어 컴퓨터가 작동할 수 있도록 합니다.

### 하드웨어와 소프트웨어의 관계

하드웨어는 데이터를 처리하기 위한 물리적인 장치들을 의미하며, 소프트웨어는 이러한 하드웨어를 제어하고 작동시키는 프로그램들을 의미합니다. 두 요소는 다음과 같이 상호작용합니다.

```
소프트웨어 명령 → 하드웨어 실행 → 결과 출력
```

## 하드웨어의 기본 구성

컴퓨터 하드웨어는 다음과 같은 핵심 구성 요소로 이루어집니다.

### 1. 중앙처리장치 (CPU)

CPU(Central Processing Unit)는 컴퓨터의 두뇌 역할을 담당하는 장치로, 연산과 제어를 수행합니다.

**주요 특징:**
- 데이터 처리 및 연산 수행
- 명령어 해석 및 실행
- 프로세서(Processor) 또는 마이크로프로세서라고도 불림
- 예: Intel Core i7, AMD Ryzen 등

### 2. 메모리 시스템

메모리는 데이터를 임시로 저장하는 장치입니다.

#### 주기억장치 (Main Memory)
- **RAM (Random Access Memory)**: 휘발성 메모리, 전원이 꺼지면 데이터 소실
- **ROM (Read-Only Memory)**: 비휘발성 메모리, 전원이 꺼져도 데이터 유지
- **특징**: 고속 액세스 가능, 상대적으로 고가

#### 보조기억장치 (Secondary Storage)
- **종류**: HDD, SSD, USB, CD-ROM, DVD 등
- **특징**: 
  - 저장 밀도가 높음 (대용량 저장 가능)
  - 주기억장치에 비해 저가
  - 데이터 영구 보관 가능
  - 상대적으로 느린 액세스 속도

:::info 메모리의 이해
컴퓨터에서 "메모리"라고 하면 일반적으로 주기억장치(RAM)를 의미합니다.  
보조기억장치는 별도로 "저장장치" 또는 "스토리지"라고 부릅니다.
:::

### 3. 입출력 장치 (I/O Devices)

**입력 장치**: 키보드, 마우스, 스캐너, 마이크 등  
**출력 장치**: 모니터, 프린터, 스피커 등

입출력 장치가 없다면 사용자는 컴퓨터와 상호작용할 수 없습니다.

### 4. 버스 시스템 (Bus)

버스는 컴퓨터 내부의 각 구성 요소들을 연결하는 데이터 통로입니다.

**버스의 역할:**
- CPU, 메모리, 입출력 장치 간 데이터 전송
- 한 번에 많은 양의 데이터 전송 가능
- 시스템 버스를 통해 모든 하드웨어 구성요소가 통신

:::tip 버스의 비유
버스는 택시처럼 소수의 데이터를 운반하는 것이 아니라, 버스처럼 많은 데이터를 한꺼번에 운반하는 역할을 합니다.
:::

## 소프트웨어의 분류

컴퓨터 소프트웨어는 시스템 소프트웨어와 응용 소프트웨어로 구분됩니다.

### 시스템 소프트웨어

하드웨어를 관리하고 응용 소프트웨어를 실행하기 위한 기본 소프트웨어입니다.

#### 1. 운영체제 (Operating System)
- **역할**: CPU, 메모리, I/O 장치, 프로세스 제어
- **예시**: Windows, macOS, Linux, UNIX, iOS, Android

#### 2. 장치 드라이버 (Device Driver)
- 하드웨어 장치를 제어하기 위한 소프트웨어
- 운영체제와 하드웨어 간의 인터페이스 역할

#### 3. 언어 번역 프로그램
- **컴파일러**: 고급 언어를 기계어로 번역 (C, C++, Java 등)
- **어셈블러**: 어셈블리 언어를 기계어로 번역
- **인터프리터**: 소스 코드를 한 줄씩 해석하여 실행

#### 4. 링커와 로더
- **링커(Linker)**: 오브젝트 파일과 라이브러리를 연결하여 실행 파일 생성
- **로더(Loader)**: 프로그램을 메모리에 적재

#### 5. 유틸리티
- 시스템 운영을 보조하는 프로그램
- 예: 백신 프로그램, 압축 프로그램, FTP, Telnet 등

### 응용 소프트웨어

사용자가 특정 목적을 위해 사용하는 프로그램입니다.

**종류:**
- 사무용: MS Office, 한글, 엑셀 등
- 그래픽: Photoshop, Illustrator 등
- 통신/네트워크: 웹 브라우저, 이메일 클라이언트 등
- 게임 프로그램
- 기타 다운로드하여 설치하는 대부분의 프로그램

:::note 유틸리티 vs 응용 소프트웨어
**유틸리티**는 컴퓨터 시스템이 원활히 작동하도록 돕는 보조 프로그램이며, **응용 소프트웨어**는 사용자가 특정 작업을 수행하기 위해 직접 사용하는 프로그램입니다.
:::

## 컴퓨터 시스템의 동작 원리

### 기본 처리 흐름

```
1. 입력장치로부터 데이터 입력
   ↓
2. CPU가 메모리에서 명령어를 가져옴 (Fetch)
   ↓
3. CPU가 명령어를 해석하고 실행 (Decode & Execute)
   ↓
4. 결과를 메모리에 저장 또는 출력장치로 전달
```

### 프로그램 실행 과정

1. **프로그램 로딩**: 보조기억장치(HDD/SSD)에 저장된 프로그램을 주기억장치(RAM)로 적재
2. **명령어 인출**: CPU가 메모리에서 프로그램 코드(명령어)를 읽어옴
3. **명령어 실행**: CPU가 명령어를 해석하고 연산 수행
4. **결과 저장**: 실행 결과를 메모리나 출력장치로 전달

## 마이크로컨트롤러

마이크로컨트롤러는 특정 기능을 수행하기 위해 설계된 소형 CPU입니다.

**특징:**
- 간단한 CPU에 필요한 하드웨어 기능 통합
- 저렴한 비용 (수천 원 수준)
- IoT 기기, 가전제품(세탁기, 냉장고, 밥솥 등)에 사용
- 예: Arduino, ATmega 시리즈

**용도:**
- 임베디드 시스템 (Embedded System)
- 특정 기능만 수행하므로 고성능 CPU 불필요
- 타이머, 센서 제어 등 간단한 연산에 최적화

## 정리

컴퓨터 시스템은 하드웨어와 소프트웨어가 유기적으로 결합되어 작동합니다.  
CPU는 연산을 담당하고, 메모리는 데이터를 저장하며, 입출력 장치는 사용자와의 상호작용을 가능하게 합니다.  
이 모든 구성 요소는 버스를 통해 연결되며, 소프트웨어는 이러한 하드웨어를 제어하여 사용자가 원하는 작업을 수행하도록 합니다.
