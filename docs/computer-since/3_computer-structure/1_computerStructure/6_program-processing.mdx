# 프로그램 처리 과정

프로그램이 실행되기까지의 변환 과정을 단계별로 알아봅니다.  
고급 언어로 작성된 소스 코드가 컴퓨터가 이해할 수 있는 기계어로 변환되는 과정은 컴퓨터 시스템의 핵심 원리입니다.

## 언어의 계층 구조

프로그래밍 언어는 추상화 수준에 따라 계층적으로 분류됩니다.

### 고급 언어 (High-Level Language)

사람이 이해하기 쉬운 형태로 작성된 언어입니다. C, Java, Python 등이 대표적입니다.

```c
// 고급 언어 예시 (C 언어)
int x = 10;
int y = 11;
int z = x + y;
```

위 코드는 간단하지만, 컴퓨터는 이를 직접 실행할 수 없습니다.

### 어셈블리어 (Assembly Language)

기계어와 거의 일대일 대응되는 저수준 언어로, 기호화된 명령어를 사용합니다.

```assembly
LOAD  A, [10]    ; 메모리 10번지의 내용을 레지스터 A에 로드
ADD   A, [11]    ; 메모리 11번지의 값을 A에 더함
STORE [12], A    ; 결과를 메모리 12번지에 저장
```

### 기계어 (Machine Language)

컴퓨터가 직접 이해하고 실행할 수 있는 0과 1로 이루어진 이진 코드입니다.

```
001010101010
010111000001
011001010011
```

## 명령어 구조

모든 명령어는 크게 두 부분으로 구성됩니다.

### 연산 코드 (Opcode)

수행할 연산의 종류를 나타냅니다. LOAD, ADD, STORE 등의 명령어가 여기에 해당합니다.

### 오퍼랜드 (Operand)

연산 대상이 되는 데이터 또는 메모리 주소를 나타냅니다.

```
| 연산 코드 | 오퍼랜드 |
|    3bit   |   5bit   |
```

예를 들어 `LOAD A, [10]` 명령어는 다음과 같이 구성됩니다:

- 연산 코드: LOAD (특정 비트 패턴)
- 오퍼랜드: A (레지스터), 10 (메모리 주소)

## 프로그램 변환 과정

소스 코드에서 실행 파일까지의 변환은 여러 단계를 거칩니다.

### 1. 소스 코드 작성

개발자가 고급 언어로 프로그램을 작성합니다.

```c
// source.c
int main() {
    int x = 10;
    int y = 11;
    int z = x + y;
    return 0;
}
```

### 2. 컴파일 (Compile)

컴파일러가 소스 코드를 기계어 또는 어셈블리어로 변환하여 오브젝트 파일(.obj, .o)을 생성합니다.

```
source.c → [컴파일러] → source.obj
```

### 3. 링크 (Link)

링커가 오브젝트 파일과 라이브러리를 연결하여 실행 파일(.exe)을 생성합니다.

```
source.obj + 라이브러리 → [링커] → program.exe
```

### 4. 로드 (Load)

로더가 실행 파일을 메모리에 적재합니다. 이제 CPU가 프로그램을 실행할 수 있습니다.

## 어셈블리어의 역할

어셈블리어는 기계어와 고급 언어의 중간 다리 역할을 합니다.

### 장점

- 하드웨어 자원을 직접 제어할 수 있습니다
- 성능 최적화가 가능합니다
- 시스템 프로그래밍에 필수적입니다

### 사용 사례

- 운영체제 커널 개발
- 디바이스 드라이버 작성
- 임베디드 시스템 프로그래밍
- 성능이 중요한 특정 루틴 최적화

## 컴파일러의 역할

컴파일러는 고급 언어를 기계어로 변환하는 핵심 시스템 소프트웨어입니다.

### 주요 기능

1. **구문 분석**: 소스 코드의 문법적 정확성 검사
2. **의미 분석**: 변수 타입, 선언 등의 의미적 정확성 검사
3. **최적화**: 코드의 성능과 효율성 개선
4. **코드 생성**: 목적 코드(오브젝트 파일) 생성

## 링커의 역할

링커는 여러 오브젝트 파일과 라이브러리를 연결하여 최종 실행 파일을 만듭니다.

### 주요 작업

- 외부 참조 해결: 다른 파일의 함수나 변수 연결
- 주소 할당: 각 코드와 데이터에 최종 메모리 주소 배정
- 라이브러리 통합: 필요한 라이브러리 함수 포함

```
main.obj + util.obj + stdlib.lib → program.exe
```

## 로더의 역할

로더는 실행 파일을 메모리에 적재하고 실행을 시작합니다.

### 주요 기능

1. 실행 파일을 메모리에 복사
2. 프로그램 카운터(PC) 초기화
3. 스택과 힙 영역 설정
4. 제어권을 프로그램 시작점으로 이전

## 정리

프로그램 처리 과정은 다음과 같이 요약됩니다:

```
소스 코드 (.c, .java)
    ↓ [컴파일러]
오브젝트 파일 (.obj, .o)
    ↓ [링커]
실행 파일 (.exe)
    ↓ [로더]
메모리 적재 및 실행
    ↓ [CPU]
프로그램 실행
```

이러한 과정을 이해하면 프로그램이 어떻게 동작하는지,  
그리고 성능 최적화를 위해 어떤 부분에 집중해야 하는지 파악할 수 있습니다.
