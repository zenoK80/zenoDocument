## 🎯 이번 시간의 목표

가상기억장치(Virtual Memory)가 정확히 무엇이고, 어떻게 동작하는지 이해하는 것입니다.

---

## 📚 가상기억장치란?

### 기본 개념

가상기억장치는 **하드디스크의 일부를 메인 메모리(RAM) 처럼 사용하는 기술**입니다.

간단히 말해서:
- 실제 RAM 용량이 4GB라고 해도
- 마치 훨씬 더 큰 메모리를 가진 것처럼 사용할 수 있게 해주는 것
- 이것이 바로 가상기억장치의 핵심입니다

### 왜 필요한가?

**실제 상황을 생각해봅시다:**

당신이 1GB짜리 게임 프로그램을 실행한다고 가정해봅시다. 
- 컴퓨터의 RAM이 2GB밖에 없으면?
- 게임 전체를 한 번에 메모리에 올릴 수 없죠
- 그래서 게임이 부분부분 필요한 것만 메모리에 올렸다 내렸다 하는 것입니다

이 방식을 사용하면:
- 작은 RAM으로도 큰 프로그램을 실행할 수 있습니다
- 사용자는 메모리 용량에 대해 덜 걱정해도 됩니다
- 여러 프로그램을 동시에 실행할 수 있습니다

## 💾 물리적 메모리 vs 가상 메모리

### 물리적 메모리(Physical Memory)

- **실제로 컴퓨터에 설치된 RAM**
- 예: 16GB 메모리를 구입해서 설치한 것
- 하드웨어적으로 존재하는 메모리

### 가상 메모리(Virtual Memory)

- **RAM + 하드디스크의 특정 영역(스왑 영역)**
- 사용자 입장에서는 마치 매우 큰 메모리가 있는 것처럼 보임
- 운영체제가 자동으로 관리하는 메모리

---

## 🔄 스왑(Swap) 개념 이해하기

### 스왑 인(Swap In)

```
하드디스크의 스왑 영역 → 메인 메모리로 가져오기
```

프로그램이 필요한 데이터를 하드디스크에서 RAM으로 불러오는 과정입니다.

### 스왑 아웃(Swap Out)

```
메인 메모리 → 하드디스크의 스왑 영역으로 내보내기
```

현재 사용하지 않는 데이터를 메모리에서 제거하고 하드디스크에 저장하는 과정입니다.

### 실제 동작 흐름

```
[프로세스 A 실행 중] → [메모리 부족] → [프로세스 B를 하드디스크로 swap out]
                              ↓
                    [프로세스 A를 위한 공간 확보]
                              ↓
                    [나중에 프로세스 B 필요] → [하드디스크에서 swap in]
```

## 📊 가상 메모리의 구조

메인 메모리를 살펴보면 이렇게 나뉩니다:

### 메모리 구성 요소

1. **운영체제 영역** (OS Area)
   - 운영체제가 사용하는 영역
   - 사용자가 함부로 접근할 수 없음

2. **사용자 영역** (User Area)
   - 프로세스(실행 중인 프로그램)들이 사용하는 영역
   - 여러 프로세스가 공유

3. **스왑 영역** (Swap Area)
   - 하드디스크의 특정 부분
   - 메모리가 부족할 때 사용하는 백업 저장소

### 전체 메모리 = 물리적 메모리 + 스왑 영역

프로그램은 실제로는 훨씬 더 큰 주소 공간을 사용할 수 있습니다!

## 🖥️ 실제 시스템에서 확인하기

### Windows 가상 메모리 파일

윈도우에서 가상 메모리는 `pagefile.sys`라는 숨김 파일로 관리됩니다.

**위치:** `C:\ (C드라이브 루트 폴더)`

```
일반적인 크기: 40GB ~ 100GB
(시스템마다 다를 수 있음)
```

### 자신의 메모리 상태 확인하기

메모리 상태를 모니터링하려면:

1. **작업 관리자 열기**
   - Windows: Ctrl + Shift + Esc
   - "성능" 탭에서 메모리 정보 확인

2. **메모리 상세 정보**
   - Commit: 현재 예약된 메모리
   - Paging: 스왑 사용량
   - Cache: 캐시 메모리 사용량

## 🎓 핵심 정리

| 항목 | 설명 |
|------|------|
| **가상 메모리** | RAM과 하드디스크를 함께 사용하여 더 큰 메모리처럼 작동 |
| **물리적 메모리** | 실제 설치된 RAM |
| **스왑** | 메모리와 하드디스크 간의 데이터 이동 |
| **스왑 인** | 하드디스크 → 메모리 |
| **스왑 아웃** | 메모리 → 하드디스크 |
| **주소 공간** | 프로그램이 사용할 수 있는 가상의 메모리 주소 범위 |

## 💡 자주 묻는 질문(FAQ)

**Q: 가상 메모리를 많이 사용하면 컴퓨터가 느려지나요?**

A: 네, 맞습니다. 하드디스크는 RAM보다 훨씬 느리기 때문에, 스왑을 자주 사용하면 성능이 저하됩니다. 이것을 "스래싱(Thrashing)"이라고 부릅니다.

**Q: 가상 메모리를 비활성화할 수 있나요?**

A: 이론상 가능하지만, 많은 프로그램들이 가상 메모리를 기대하고 설계되어 있어서 비활성화하면 문제가 생길 수 있습니다.

**Q: 모두가 가상 메모리를 사용하나요?**

A: 네, Unix, Linux, Windows 등 현대의 모든 운영체제가 가상 메모리를 기본으로 사용합니다.
