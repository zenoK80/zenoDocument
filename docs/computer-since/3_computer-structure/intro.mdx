# 컴퓨터 구조(Computer Architecture)란?

## 개요

컴퓨터 구조는 **컴퓨터 시스템의 하드웨어 설계와 동작 원리**를 다루는 핵심 과목입니다.  
프로그램이 실제로 하드웨어에서 어떻게 실행되는지, CPU와 메모리가 어떻게 협력하는지를 이해하는 학문입니다.

:::tip 핵심 포인트
하드웨어 이해 = 효율적인 소프트웨어 개발 + 성능 최적화
:::

---

## 왜 배워야 하나요?

컴퓨터 구조를 배우는 이유는 다음과 같습니다:

- **성능 최적화**: 하드웨어의 특성을 이해하면 더 빠르고 효율적인 프로그램을 작성할 수 있습니다
- **시스템 설계 능력**: 임베디드 시스템, IoT 기기 등을 설계하고 개발할 수 있는 기초를 다집니다
- **문제 해결**: 병목 현상이나 성능 문제의 근본 원인을 파악하고 해결할 수 있습니다

## 주요 학습 주제

컴퓨터 구조 과목에서는 다음과 같은 내용을 배웁니다:

### 기본 개념
- 컴퓨터 시스템의 구성 요소
- 폰 노이만 구조
- 성능 평가 방법

### 명령어와 프로세서
- 명령어 집합 구조 (ISA)
- RISC vs CISC
- 프로세서 설계 (데이터패스와 제어)

### 프로세서 최적화
- 파이프라이닝 (Pipelining)
- 해저드 (Hazard)와 해결 방법
- 슈퍼스칼라와 비순차 실행

### 메모리 시스템
- 캐시 메모리
- 가상 메모리
- 메모리 계층 구조

### 병렬 처리
- 멀티코어 프로세서
- 병렬 프로그래밍 기초
- GPU 구조
