# 프로그래밍 언어의 진화와 분류

프로그래밍 언어는 시대에 따라 계속 진화하고 있습니다. 이번 시간에는 언어의 분류와 발전 역사를 살펴보겠습니다.

## 프로그래밍 언어의 분류

### 1. 저급 언어 vs 고급 언어

```
프로그래밍 언어
│
├─ 저급 언어
│  ├─ 기계어 (Machine Language)
│  └─ 어셈블리어 (Assembly Language)
│
└─ 고급 언어
   ├─ 컴파일 기반 언어 (Compiled Languages)
   │  └─ C, C++, Java
   │
   └─ 인터프리터 언어 (Interpreted Languages)
      └─ Python, JavaScript, PHP
```

### 저급 언어란?

**저급 언어는 컴퓨터와 직접 대화하는 언어입니다.**

- **기계어** 👾
  - 0과 1로 이루어진 이진수
  - 컴퓨터가 직접 이해하는 언어
  - 인간이 읽기는 거의 불가능

- **어셈블리어** 🔧
  - 기계어를 좀 더 읽기 쉽게 만든 언어
  - 하드웨어와 직접 제어
  - 매우 복잡하고 어려움

:::info
요즘에는 저급 언어를 거의 사용하지 않습니다. 대신 고급 언어를 사용하고, 컴파일러가 자동으로 변환해줍니다.
:::

### 고급 언어란?

**고급 언어는 인간이 이해하기 쉽게 만든 언어입니다.**

#### 컴파일 기반 언어
- 실행 전에 **전체 코드를 한 번에 기계어로 변환**
- 변환 후 실행 속도가 빠름
- 예: C, C++, Java

#### 인터프리터 언어
- 코드를 **한 줄 한 줄 읽으면서 실행**
- 배우기 쉽고 빠르게 테스트 가능
- 실행 속도는 느림
- 예: Python, JavaScript

## 프로그래밍 언어의 역사

### 1950-1960년대: 컴퓨터의 탄생

컴퓨터가 처음 나왔을 때는 **기계어와 어셈블리어**만 사용했습니다.

**FORTRAN (포트란)** 📊
- IBM에서 개발한 최초의 고급 프로그래밍 언어
- 과학 계산과 수치 계산에 특화
- 아직도 과학 분야에서 사용됨

**ALGOL (알골)** 🏗️
- 알고리즘을 표현하기 위한 고급 언어
- 프로그래밍 언어의 기초를 만들어냄
- 현대 언어에 큰 영향을 미침

**COBOL (코볼)** 💼
- 비즈니스를 위한 언어 (Common Business)
- 회사의 재고관리, 성적관리 같은 사무 처리용
- 은행, 보험 같은 금융권에서 여전히 사용

### 1960-1970년대: 기초 언어들

**BASIC (베이직)** 🎓
- 초보자를 위한 가장 쉬운 언어
- 인터프리터 방식
- "Beginner's All-purpose Symbolic Instruction Code"의 약자

**C 언어** ⭐ (가장 중요!)
- 1970년대 Bell Labs에서 Dennis Ritchie와 Ken Thompson이 개발
- **모든 프로그래밍의 기초가 되는 언어**
- 이식성(다양한 시스템에서 사용 가능)과 효율성이 뛰어남
- 하드웨어 제어부터 이미지 처리까지 다양한 분야에서 사용
- Unix 운영체제 개발에 사용되면서 널리 보급됨

:::success
C 언어가 없었다면 현대의 모든 프로그래밍 언어가 존재하지 않았을 것입니다!
:::

### 1980-1990년대: 객체지향의 시대

**C++ (씨플러스플러스)** 🎯
- C 언어에 **객체지향 개념을 추가**
- C 언어의 확장판 (Bjarne Stroustrup 개발)
- 더 복잡하지만 더 강력한 기능 제공

**Pascal (파스칼)** 📚
- 교육과 소프트웨어 개발에 주로 사용
- 간단하고 구조적인 프로그래밍에 최적화
- 초보자가 배우기 좋음

**Java (자바)** ☕ (웹의 대표 언어!)
- 1990년대에 Sun Microsystems에서 개발
- 웹 개발의 대표 언어로 자리 잡음
- 객체지향 프로그래밍 기반
- **플랫폼 독립성** = 어느 운영체제에서나 동일하게 작동
- 웹 애플리케이션, 기업용 소프트웨어에 널리 사용

:::tip
Java는 "한 번 작성하면, 모든 곳에서 작동한다(Write Once, Run Anywhere)"는 철학으로 개발되었습니다.
:::

### 2000년대: 웹과 스크립트 언어의 부상

**JavaScript (자바스크립트)** 🌐
- 웹 브라우저에서 동적인 웹사이트를 만드는 언어
- 프론트엔드 개발의 표준 언어
- Java를 기반으로 만들어졌지만 완전히 다른 언어

**C# (씨샾)** 🪟
- Microsoft에서 개발한 언어
- .NET 프레임워크 기반
- Windows 애플리케이션 개발에 주로 사용

**PHP (피에이치피)** 🌍
- "Personal Home Page Tools"의 약자
- 웹사이트 개발 언어
- 서버 측 웹 개발에 자주 사용

**HTML (에이치티엠엘)** 📄
- "HyperText Markup Language"
- 웹페이지를 만드는 언어
- 우리가 보는 모든 웹사이트의 기초

### 2010년대 이후: 현대의 다양한 언어들

**Python (파이썬)** 🐍 (최신 트렌드!)
- **가장 배우기 쉬운 언어로 평가받음**
- 초보 개발자들 중에 가장 인기 있는 언어
- 웹 개발, 데이터 분석, 인공지능에 널리 사용
- 간결하고 직관적인 문법

**Ruby (루비)** 💎
- 간결하고 직관적인 문법
- 웹 개발과 스크립트 언어로 사용
- 동적 프로그래밍에 최적화

**Swift (스위프트)** 🍎
- Apple에서 개발한 iOS 앱 개발 언어
- iPhone, iPad 앱 개발에 사용
- 현대적이고 안전한 문법

**Go (고)** 🔵
- Google에서 만든 언어
- 클라우드 기반 웹 환경에서 많이 사용
- 빠르고 효율적인 성능

**Kotlin (코틀린)** 🤖
- Java 기반의 안드로이드 앱 개발 언어
- Android 개발의 공식 언어
- Java보다 간결한 문법

**Rust (러스트)** ⚙️
- 고성능과 메모리 안전성을 추구
- 분산 환경과 멀티스레드 처리에 최적화
- 시스템 프로그래밍에 사용

## 프로그래밍 언어 선택 가이드

### 목적에 따른 언어 선택

| 목적 | 추천 언어 |
|------|---------|
| **기초 학습** | C, Python |
| **웹 개발 (서버)** | Java, Python, PHP, Go |
| **웹 개발 (클라이언트)** | JavaScript |
| **모바일 (iOS)** | Swift |
| **모바일 (Android)** | Kotlin, Java |
| **데이터 분석** | Python |
| **게임 개발** | C++, C# |
| **시스템 프로그래밍** | C, Rust |
| **빠른 프로토타이핑** | Python, Ruby |

### 초보자라면?

```
첫 번째 선택: Python 또는 C
├─ Python: 쉽고 재미있음 (데이터, 웹 분야 좋음)
└─ C: 어렵지만 기초가 탄탄함 (모든 분야 기초가 됨)
```

:::tip
**추천**: Python으로 시작해서 기초를 다진 후, C를 배우면 됩니다!
:::

## 정리하며 🎯

### 언어의 분류
- **저급 언어**: 기계어, 어셈블리어 (거의 사용 안 함)
- **고급 언어**: 컴파일 방식, 인터프리터 방식

### 시대별 대표 언어
- **1950-60년대**: FORTRAN, ALGOL, COBOL
- **1970년대**: C 언어 (가장 중요!)
- **1980-90년대**: C++, Java
- **2000년대**: JavaScript, PHP
- **2010년대 이후**: Python, Swift, Go, Kotlin

### 현재 가장 인기 있는 언어들
1. **Python** - 배우기 쉬움, 다양한 분야
2. **JavaScript** - 웹 개발의 표준
3. **Java** - 기업용 소프트웨어
4. **C/C++** - 시스템 프로그래밍
5. **Go, Rust** - 클라우드/성능 중심

:::success
이제 여러분은 프로그래밍 언어의 역사와 특징을 알게 되었습니다. 다음 단계는 언어를 선택해서 직접 배우는 것입니다!
:::
