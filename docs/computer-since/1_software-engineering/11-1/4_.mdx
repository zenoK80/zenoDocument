# 코딩의 원리와 실행 과정

프로그래밍의 **전체 흐름**을 이해하면, 더 나은 코드를 만들 수 있습니다. 이번 시간에는 설계부터 실행까지의 과정을 따라가 봅시다!

## 코딩의 정의

### 코딩(Coding)이란?

**코딩 = 설계명세서를 코드로 바꾸는 작업**

```
설계명세서 (Design Specification)
        ↓
    코딩 (Coding)
        ↓
    소스 코드 (Source Code)
        ↓
  컴파일 및 실행
        ↓
    프로그램 실행
```

:::tip
코딩은 단순히 "코드를 작성하는 것"이 아니라, **설계를 실제 구현으로 변환하는 작업**입니다!
:::

---

## 프로그래밍의 전체 흐름

### 1단계: 요구사항 분석 📋

**질문:** 사용자가 원하는 게 뭐지?

```
사용자 요구사항
├─ 입력: 두 개의 숫자
├─ 처리: 더하기
└─ 출력: 합계 표시
```

### 2단계: 설계 📐

**설계에서 결정할 것:**

```
입력(Input) → 처리(Process) → 출력(Output)

어떻게 입력받을까?
├─ 키보드 입력
├─ 파일 읽기
└─ 네트워크 수신

어떻게 처리할까?
├─ 어떤 공식 사용?
├─ 어떤 순서로?
└─ 예외 상황은?

어떻게 출력할까?
├─ 화면 표시
├─ 파일 저장
└─ 네트워크 전송
```

### 3단계: 코딩 💻

**소스 코드 작성:**

```cpp
#include <stdio.h>

int main() {
    // 1. 입력
    int num1 = 10;
    int num2 = 20;
    
    // 2. 처리
    int sum = num1 + num2;
    
    // 3. 출력
    printf("합계: %d\n", sum);
    
    return 0;
}
```

### 4단계: 컴파일 🔍

컴파일러가 하는 일:
- ✅ 문법 검사 (Syntax Check)
- ✅ 의미 검사 (Semantic Check)
- ✅ 기계어로 변환

### 5단계: 링킹 🔗

목적 프로그램을 실행 가능한 형태로 변환

### 6단계: 실행 ▶️

프로그램 실행해서 결과 출력

---

## 상세한 실행 프로세스

### 소스 코드 → 기계어로 변환되는 과정

```
┌─────────────────────────────────────┐
│   원시 프로그램 (Source Code)        │
│   C, C++, Java, Python 등           │
└─────────────────────────────────────┘
              ↓
      [컴파일러 (Compiler)]
      문법 검사 및 변환
              ↓
┌─────────────────────────────────────┐
│   목적 프로그램 (Object Program)      │
│   기계어로 변환된 중간 형태          │
└─────────────────────────────────────┘
              ↓
        [링커 (Linker)]
    실행 가능한 형태로 연결
              ↓
┌─────────────────────────────────────┐
│   실행 파일 (Executable File)        │
│   .exe, .out 등                     │
└─────────────────────────────────────┘
              ↓
        [로더 (Loader)]
    메모리에 로드해서 실행
              ↓
      프로그램 실행 결과!
```

:::info
우리는 **소스 코드만 문법적으로 정확하게 작성**하면, 컴파일러가 나머지를 모두 처리해줍니다!
:::

### 각 단계별 역할

#### 컴파일러 (Compiler) 🔍

**하는 일:**
- 문법적 오류 확인
- 의미적 오류 확인
- 기계어로 변환

**예시:**
```cpp
// ❌ 오류 코드
int x = "문자열";  // int 변수에 문자열 대입
printf("합계 %d", "오류");  // 형식 불일치

// 컴파일러가 이런 오류들을 찾아줍니다!
```

#### 링커 (Linker) 🔗

**하는 일:**
- 여러 개의 목적 파일 연결
- 라이브러리 연결
- 메모리 주소 할당

**예시:**
```
파일1.o + 파일2.o + 라이브러리 
    ↓ (링커)
실행 파일 (program.exe)
```

#### 로더 (Loader) ▶️

**하는 일:**
- 실행 파일을 메모리에 로드
- 프로그램 실행

## 컴파일러가 언어를 이해하는 방식

### 약속된 규칙 (Syntax Rules)

프로그래밍 언어는 **엄격한 문법 규칙**을 가지고 있습니다.

```java
// Java 문법 규칙 예시

// ✅ 올바른 문법
public class Hello {
    public static void main(String[] args) {
        System.out.println("Hello");
    }
}

// ❌ 잘못된 문법
public class Hello {
    public static void main() {  // String[] args 필수
        System.out.println("Hello")  // 세미콜론 필요
    }
}
```

### 문법 규칙을 지켜야 하는 이유

컴퓨터는 **정확히 약속된 규칙대로만 이해**합니다.

```
예: "안녕하세요"라고 인사하는 것
├─ 한국어 규칙을 지킨 문장
└─ 상대방이 이해 가능 ✅

"안녕 하 세요"라고 띄워쓰거나
"아녕하세요"라고 오타 내면
└─ 규칙을 어겼으니 이해하기 어려움 ❌
```

## 우리가 신경써야 할 것과 자동으로 처리되는 것

### 우리가 해야 할 일 👨‍💻

```cpp
#include <stdio.h>

int main() {
    int a = 10;
    int b = 20;
    int sum = a + b;
    printf("%d", sum);
    return 0;
}
```

**우리가 신경쓸 것:**
- ✅ **소스 코드를 정확하게 작성** (문법에 맞게)
- ✅ **논리적으로 올바르게 구현**
- ✅ **예외 상황 처리**

### 컴파일러가 자동으로 처리하는 것 🤖

```
우리가 만든 코드
        ↓
컴파일러 (자동으로)
├─ 문법 검사
├─ 기계어로 변환
├─ 메모리 할당
└─ 최적화
        ↓
실행 가능한 파일
```

:::success
**핵심:** 우리는 **"무엇을"(What) 해야 할지** 코드로 표현하면, 컴파일러가 **"어떻게"(How) 실행할지** 처리합니다!
:::

## 실제 프로그래밍 예시

### 소수 찾기 프로그램

```cpp
#include <stdio.h>

// 소수인지 판단하는 함수
int isPrime(int num) {
    if (num < 2) return 0;  // 2보다 작으면 소수 아님
    
    for (int i = 2; i * i <= num; i++) {
        if (num % i == 0) {
            return 0;  // 나누어떨어지면 소수 아님
        }
    }
    return 1;  // 소수임
}

int main() {
    printf("1부터 20까지의 소수:\n");
    
    for (int i = 1; i <= 20; i++) {
        if (isPrime(i)) {
            printf("%d ", i);
        }
    }
    
    printf("\n");
    return 0;
}
```

**코드 라인별 설명:**

| 코드 | 설명 |
|------|------|
| `#include <stdio.h>` | 입출력 함수들을 사용하기 위해 라이브러리 포함 |
| `isPrime(int num)` | 소수 판정 함수 정의 |
| `if (num < 2)` | 2보다 작으면 소수가 아니므로 0 반환 |
| `for (int i = 2; i*i <= num; i++)` | 2부터 √num까지 나누어떨어지는지 확인 |
| `if (num % i == 0)` | 나누어떨어지면 소수 아님 |
| `return 1;` | 나누어떨어지지 않으면 소수 |
| `for (int i = 1; i <= 20; i++)` | 1부터 20까지 반복 |
| `printf("%d ", i);` | 소수인 경우 출력 |

**실행 결과:**
```
1부터 20까지의 소수:
2 3 5 7 11 13 17 19
```

## 정리하며 🎯

### 프로그래밍의 전체 흐름

```
요구사항 분석 → 설계 → 코딩 → 컴파일 → 링킹 → 실행
   (뭘?)    (어떻게?) (코드) (검사) (연결) (결과)
```

### 컴파일 과정

```
소스 코드 (Source Code)
    ↓ [컴파일러]
목적 프로그램 (Object Program)
    ↓ [링커]
실행 파일 (Executable)
    ↓ [로더]
프로그램 실행
```

### 우리의 책임

| 항목 | 우리가 할 일 | 컴파일러가 할 일 |
|------|-------------|-----------------|
| **소스 코드 작성** | ✅ | |
| **문법 검사** | | ✅ |
| **기계어 변환** | | ✅ |
| **실행 파일 생성** | | ✅ |
| **프로그램 논리** | ✅ | |
